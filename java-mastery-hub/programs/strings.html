1. Creating Strings (Literal vs New)


public class StringCreation {
    public static void main(String[] args) {
        // String Literal (Stored in Pool)
        String s1 = "Hello";
        
        // New Keyword (Stored in Heap)
        String s2 = new String("Hello");
        
        System.out.println("Literal: " + s1);
        System.out.println("New Object: " + s2);
    }
}
Baby: Two ways to write a name tag: one from the pre-printed pile (pool), one hand-written (new).
Real-world: Buying a generic greeting card vs making a custom one.
Technical: Literals use String Pool for memory efficiency. 'new' forces a new object creation in Heap.

2. Checking Equality (== vs equals)


public class StringEquality {
    public static void main(String[] args) {
        String s1 = "Java";
        String s2 = "Java";
        String s3 = new String("Java");

        System.out.println(s1 == s2);       // true (Same reference)
        System.out.println(s1 == s3);       // false (Different reference)
        System.out.println(s1.equals(s3));  // true (Same content)
    }
}
Baby: '==' checks if they are the SAME paper. 'equals' checks if they have the SAME words.
Real-world: Two identical books. '==' asks "Is this the exact same physical copy?". 'equals' asks "Is the story the same?".
Technical: '==' compares memory addresses. 'equals()' compares character sequences.

3. Demonstration of String Pool


public class StringPoolDemo {
    public static void main(String[] args) {
        String s1 = "Apple";
        String s2 = "Apple";
        
        // HashCode proves they are same object
        System.out.println("s1 Hash: " + System.identityHashCode(s1));
        System.out.println("s2 Hash: " + System.identityHashCode(s2));
    }
}
Technical: System.identityHashCode() reveals the memory address hash. Identical hashes confirm they point to the same object in the Pool.

4. Using substring()


public class SubstringDemo {
    public static void main(String[] args) {
        String text = "Hello World";
        
        String sub1 = text.substring(6);     // "World"
        String sub2 = text.substring(0, 5);  // "Hello" (end index is exclusive)
        
        System.out.println(sub1);
        System.out.println(sub2);
    }
}
Baby: Cutting a piece of the ribbon.
Technical: Creates a new String from the specified range. Remember: endIndex is exclusive (not included).

5. String Concatenation


public class ConcatDemo {
    public static void main(String[] args) {
        String s1 = "Hello";
        String s2 = "World";
        
        String res1 = s1 + " " + s2;
        String res2 = s1.concat(" ").concat(s2);
        
        System.out.println(res1);
        System.out.println(res2);
    }
}
Technical: '+' is syntactic sugar (often uses StringBuilder internally). 'concat()' is a method of String class.

6. Using StringBuilder (Mutable)


public class StringBuilderDemo {
    public static void main(String[] args) {
        StringBuilder sb = new StringBuilder("Hello");
        
        sb.append(" World");
        sb.replace(0, 5, "Hi");
        sb.delete(2, 3); // Remove space
        
        System.out.println(sb.toString()); // "HiWorld"
    }
}
Baby: Like writing on a whiteboard - you can erase and change things easily.
Technical: Modifies the internal character array directly without creating new objects. Efficient!

public class StringBufferDemo {

8. Performance Comparison (Loop)


public class PerformanceTest {
    public static void main(String[] args) {
        long startTime = System.currentTimeMillis();
        
        // BAD WAY
        String s = "";
        for(int i=0; i<10000; i++) {
            s += i;
        }
        System.out.println("String time: " + (System.currentTimeMillis() - startTime) + "ms");
        
        startTime = System.currentTimeMillis();
        
        // GOOD WAY
        StringBuilder sb = new StringBuilder();
        for(int i=0; i<10000; i++) {
            sb.append(i);
        }
        System.out.println("StringBuilder time: " + (System.currentTimeMillis() - startTime) + "ms");
    }
}
Real-world: String is like building a new house every time you add a brick. StringBuilder is adding bricks to the same house.
Technical: String creates 10,000 objects. StringBuilder modifies one. Huge performance difference!

9. Reverse a String


public class ReverseString {
    public static void main(String[] args) {
        String original = "Hello";
        
        // Method 1: Using StringBuilder
        String rev1 = new StringBuilder(original).reverse().toString();
        
        // Method 2: Manual loop
        String rev2 = "";
        for(int i=original.length()-1; i>=0; i--) {
            rev2 += original.charAt(i);
        }
        
        System.out.println(rev1);
        System.out.println(rev2);
    }
}

10. Palindrome Check


public class PalindromeCheck {
    public static void main(String[] args) {
        String str = "Madam";
        String cleanStr = str.toLowerCase(); // Case insensitive
        
        String rev = new StringBuilder(cleanStr).reverse().toString();
        
        if(cleanStr.equals(rev)) {
            System.out.println(str + " is a Palindrome");
        } else {
            System.out.println(str + " is NOT a Palindrome");
        }
    }
}
Baby: A word that reads the same backwards, like "MOM".

11. Word Count using split()


public class WordCount {
    public static void main(String[] args) {
        String sentence = "Java is fun to learn";
        
        String[] words = sentence.split(" ");
        
        System.out.println("Number of words: " + words.length);
        for(String w : words) {
            System.out.println(w);
        }
    }
}
Technical: split() takes a regex delimiter. Here we split by space.

12. Using format()


public class StringFormat {
    public static void main(String[] args) {
        String name = "Alice";
        int age = 25;
        double salary = 50000.50;
        
        String formatted = String.format("Name: %s, Age: %d, Salary: $%.2f", name, age, salary);
        
        System.out.println(formatted);
    }
}
Technical: Similar to printf in C. %s for String, %d for integer, %.2f for float with 2 decimals.

13. Regex Usage (Validation)


public class RegexDemo {
    public static void main(String[] args) {
        String phone = "123-456-7890";
        
        // Pattern: 3 digits, hyphen, 3 digits, hyphen, 4 digits
        boolean isValid = phone.matches("\\d{3}-\\d{3}-\\d{4}");
        
        System.out.println("Is valid phone: " + isValid);
    }
}
Technical: \\d means digit. {n} means n times. matches() returns true if the whole string matches the pattern.

14. Using intern()


public class InternDemo {
    public static void main(String[] args) {
        String s1 = new String("Java");
        String s2 = s1.intern(); // Moves to pool/Gets from pool
        String s3 = "Java";
        
        System.out.println(s1 == s3); // false (Heap vs Pool)
        System.out.println(s2 == s3); // true (Both in Pool)
    }
}
Technical: intern() ensures you get the canonical representation from the String Pool, saving memory.

15. Real-World: Email Validation


public class EmailValidator {
    public static void main(String[] args) {
        String email = "user@example.com";
        
        if(isValidEmail(email)) {
            System.out.println("Email is valid");
        } else {
            System.out.println("Invalid email");
        }
    }
    
    public static boolean isValidEmail(String email) {
        if(email == null || email.isEmpty()) return false;
        
        // Simple regex for demonstration
        return email.matches("^[\\w-\\.]+@([\\w-]+\\.)+[\\w-]{2,4}$");
    }
}
Real-world: Used in every login form to check if you typed a real email address format.
