Hands-On Java Programs

Welcome to the practical section! These 12 programs demonstrate every aspect of variables and data types in Java. Each program is heavily commented to explain not just WHAT the code does, but WHY it works that way.

Program 1: Variable Declaration and Initialization

This program demonstrates the basics of declaring and initializing variables in Java.

‚òï Java 
/**
 * VariableBasicsDemo - Understanding Variable Declaration and Initialization
 */
public class VariableBasicsDemo {
    
    public static void main(String[] args) {
        // Declaration only
        int age;
        
        // Initialization
        age = 25;
        System.out.println("Age: " + age);
        
        // Declaration + Initialization (most common)
        int score = 100;
        System.out.println("Score: " + score);
        
        // Multiple variables
        int x = 10, y = 20, z = 30;
        System.out.println("x=" + x + ", y=" + y + ", z=" + z);
    }
}
Explanation: This program covers variable fundamentals including declaration, initialization, and naming rules.

Program 2: All Primitive Data Types

Demonstrates all 8 primitive types with ranges and default values.

‚òï Java 
/**
 * PrimitiveTypesDemo - All 8 Primitive Types
 */
public class PrimitiveTypesDemo {
    
    public static void main(String[] args) {
        // Integer types
        byte b = 127;           // 8 bits: -128 to 127
        short s = 32000;        // 16 bits: -32,768 to 32,767
        int i = 2000000;        // 32 bits: -2 billion to 2 billion
        long l = 9000000000L;   // 64 bits: very large numbers
        
        // Floating-point types
        float f = 3.14f;        // 32 bits: ~6-7 decimal digits
        double d = 3.14159265;  // 64 bits: ~15-16 decimal digits
        
        // Character and boolean
        char c = 'A';           // 16 bits: Unicode character
        boolean bool = true;    // true or false
        
        System.out.println("byte: " + b);
        System.out.println("short: " + s);
        System.out.println("int: " + i);
        System.out.println("long: " + l);
        System.out.println("float: " + f);
        System.out.println("double: " + d);
        System.out.println("char: " + c);
        System.out.println("boolean: " + bool);
    }
}
Explanation: Shows all 8 primitive types with their ranges, sizes, and use cases.

Program 3: Type Casting (Widening and Narrowing)

Demonstrates implicit and explicit type casting.

‚òï Java 
/**
 * TypeCastingDemo - Widening and Narrowing Conversions
 */
public class TypeCastingDemo {
    
    public static void main(String[] args) {
        // WIDENING (Implicit - Automatic)
        byte byteVal = 10;
        int intVal = byteVal;  // Automatic conversion
        double doubleVal = intVal;  // Automatic conversion
        
        System.out.println("Widening: byte " + byteVal + " ‚Üí int " + intVal + " ‚Üí double " + doubleVal);
        
        // NARROWING (Explicit - Manual)
        double price = 19.99;
        int dollars = (int) price;  // Explicit cast required
        System.out.println("Narrowing: double " + price + " ‚Üí int " + dollars);
        System.out.println("Lost decimal part: " + (price - dollars));
        
        // Overflow example
        int largeInt = 130;
        byte smallByte = (byte) largeInt;
        System.out.println("Overflow: int " + largeInt + " ‚Üí byte " + smallByte);
    }
}
Explanation: Widening is automatic (safe), narrowing requires explicit cast (may lose data).

Program 4: Default Values

Shows default values for instance variables.

‚òï Java 
/**
 * DefaultValuesDemo - Instance vs Local Variables
 */
public class DefaultValuesDemo {
    
    // Instance variables - have default values
    static int instanceInt;
    static double instanceDouble;
    static boolean instanceBoolean;
    static String instanceString;
    
    public static void main(String[] args) {
        System.out.println("Instance variable defaults:");
        System.out.println("int: " + instanceInt);           // 0
        System.out.println("double: " + instanceDouble);     // 0.0
        System.out.println("boolean: " + instanceBoolean);   // false
        System.out.println("String: " + instanceString);     // null
        
        // Local variables - NO default values
        int localInt = 0;  // Must initialize!
        System.out.println("\nLocal variable: " + localInt);
    }
}
Explanation: Instance variables have defaults (0, false, null). Local variables must be initialized.

Program 5: Wrapper Classes and Autoboxing

Demonstrates wrapper classes and autoboxing/unboxing.

‚òï Java 
/**
 * WrapperClassesDemo - Autoboxing and Unboxing
 */
import java.util.ArrayList;

public class WrapperClassesDemo {
    
    public static void main(String[] args) {
        // Autoboxing: primitive ‚Üí wrapper
        int primitive = 42;
        Integer wrapper = primitive;  // Automatic
        System.out.println("Autoboxing: int " + primitive + " ‚Üí Integer " + wrapper);
        
        // Unboxing: wrapper ‚Üí primitive
        Integer wrapperNum = 100;
        int primitiveNum = wrapperNum;  // Automatic
        System.out.println("Unboxing: Integer " + wrapperNum + " ‚Üí int " + primitiveNum);
        
        // Collections require wrappers
        ArrayList numbers = new ArrayList<>();
        numbers.add(10);  // Autoboxing
        numbers.add(20);
        System.out.println("\nArrayList: " + numbers);
        
        // Utility methods
        int parsed = Integer.parseInt("123");
        String str = Integer.toString(456);
        System.out.println("Parsed: " + parsed);
        System.out.println("To String: " + str);
        
        // Constants
        System.out.println("\nInteger.MAX_VALUE: " + Integer.MAX_VALUE);
        System.out.println("Integer.MIN_VALUE: " + Integer.MIN_VALUE);
    }
}
Explanation: Wrappers enable primitives in collections, provide utility methods, and can be null.

Program 6: Integer Overflow

Demonstrates what happens when numbers exceed type limits.

‚òï Java 
/**
 * OverflowDemo - Integer Overflow and Underflow
 */
public class OverflowDemo {
    
    public static void main(String[] args) {
        // Byte overflow
        byte maxByte = 127;
        System.out.println("Max byte: " + maxByte);
        maxByte++;
        System.out.println("After increment: " + maxByte);  // Wraps to -128
        
        // Int overflow
        int maxInt = Integer.MAX_VALUE;
        System.out.println("\nMax int: " + maxInt);
        int overflow = maxInt + 1;
        System.out.println("Max + 1: " + overflow);  // Wraps to MIN_VALUE
        
        // Multiplication overflow
        int large1 = 1000000;
        int large2 = 1000000;
        int product = large1 * large2;
        System.out.println("\n" + large1 + " √ó " + large2 + " = " + product);  // Overflow!
        
        // Correct way: use long
        long correctProduct = (long) large1 * large2;
        System.out.println("Using long: " + correctProduct);  // Correct!
    }
}
Explanation: Overflow wraps around to minimum value. Use appropriate types and validate ranges.

Program 7: Floating-Point Precision

Demonstrates precision differences between float and double.

‚òï Java 
/**
 * PrecisionDemo - Float vs Double Precision
 */
public class PrecisionDemo {
    
    public static void main(String[] args) {
        // Float precision (~6-7 digits)
        float floatNum = 123456.789f;
        System.out.println("Float: " + floatNum);  // Loses precision
        
        // Double precision (~15-16 digits)
        double doubleNum = 123456.789123456;
        System.out.println("Double: " + doubleNum);  // More precise
        
        // Floating-point arithmetic issues
        double a = 0.1;
        double b = 0.2;
        double sum = a + b;
        System.out.println("\n0.1 + 0.2 = " + sum);  // Not exactly 0.3!
        
        // Integer division vs floating division
        int x = 7;
        int y = 2;
        System.out.println("\nInteger division: " + x + " / " + y + " = " + (x / y));  // 3
        System.out.println("Floating division: " + x + " / " + y + " = " + ((double) x / y));  // 3.5
    }
}
Explanation: Double is more precise than float. Never use float/double for currency (use BigDecimal).

Program 8: Character and ASCII

Demonstrates char type and ASCII values.

‚òï Java 
/**

public class CharDemo {

    }
}
Explanation: char stores Unicode characters. Can convert between char and int (ASCII values).

Program 9: Boolean Operations

Demonstrates boolean type and logical operations.

‚òï Java 
/**
 * BooleanDemo - Boolean Type and Logical Operations
 */
public class BooleanDemo {
    
    public static void main(String[] args) {
        // Boolean literals
        boolean isJavaFun = true;
        boolean isPythonBetter = false;
        
        System.out.println("Is Java fun? " + isJavaFun);
        System.out.println("Is Python better? " + isPythonBetter);

        boolean b = false;

    }
}
Explanation: Boolean stores true/false. Used in conditions, flags, and logical operations.

Program 10: Final Variables (Constants)

Demonstrates final keyword for constants.

‚òï Java 
/**
 * FinalVariablesDemo - Constants using final keyword
 */
public class FinalVariablesDemo {
    
    // Class-level constants
    public static final double PI = 3.14159265359;
    public static final int MAX_USERS = 100;
    public static final String COMPANY_NAME = "TechCorp";
    
    public static void main(String[] args) {
        // Using constants
        double radius = 5.0;
        double circumference = 2 * PI * radius;
        
        System.out.println("PI: " + PI);
        System.out.println("Radius: " + radius);
        System.out.println("Circumference: " + circumference);
        
        System.out.println("\nMAX_USERS: " + MAX_USERS);
        System.out.println("COMPANY_NAME: " + COMPANY_NAME);
        
        // Local final variable
        final int MAX_ATTEMPTS = 3;
        System.out.println("\nMax login attempts: " + MAX_ATTEMPTS);
        
        // Attempting to change final variable causes compile error
        // PI = 3.14;  // ‚ùå Error: cannot assign a value to final variable
        // MAX_ATTEMPTS = 5;  // ‚ùå Error
        
        System.out.println("\n‚úÖ Final variables cannot be changed after initialization");
    }
}
Explanation: final makes variables immutable. Use UPPER_SNAKE_CASE for constants.

Program 11: Stack vs Heap Memory

Demonstrates memory allocation for primitives and objects.

‚òï Java 
/**
 * MemoryDemo - Stack vs Heap Memory Allocation
 */
public class MemoryDemo {
    
    public static void main(String[] args) {
        // STACK: Primitives and references
        int age = 25;              // Stored on stack (value)
        double salary = 50000.0;   // Stored on stack (value)
        
        // HEAP: Objects
        String name = "Alice";     // Reference on stack, object on heap
        int[] numbers = {1, 2, 3}; // Reference on stack, array on heap
        
        System.out.println("=== STACK (Primitives) ===");
        System.out.println("age (value): " + age);
        System.out.println("salary (value): " + salary);
        
        System.out.println("\n=== HEAP (Objects) ===");
        System.out.println("name (reference ‚Üí \"" + name + "\")");
        System.out.println("numbers (reference ‚Üí array)");
        
        // Demonstrating reference behavior
        String s1 = "Hello";
        String s2 = s1;  // Both reference same object
        
        System.out.println("\n=== Reference Behavior ===");
        System.out.println("s1: " + s1);
        System.out.println("s2: " + s2);
        System.out.println("s1 == s2: " + (s1 == s2));  // true (same reference)
        
        // Creating new object
        String s3 = new String("Hello");
        System.out.println("s1 == s3: " + (s1 == s3));  // false (different reference)
        System.out.println("s1.equals(s3): " + s1.equals(s3));  // true (same content)
    }
}
Explanation: Primitives store values on stack. Objects store data on heap with references on stack.

Program 12: Practical Example - Temperature Converter

A practical program using variables and data types.

‚òï Java 
/**
 * TemperatureConverter - Practical Application
 * Demonstrates variables, casting, and arithmetic
 */
public class TemperatureConverter {
    
    // Constants
    public static final double FREEZING_POINT_F = 32.0;
    public static final double CONVERSION_FACTOR = 5.0 / 9.0;
    
    public static void main(String[] args) {
        System.out.println("=== Temperature Converter ===\n");
        
        // Fahrenheit to Celsius
        double fahrenheit = 98.6;  // Normal body temperature
        double celsius = (fahrenheit - FREEZING_POINT_F) * CONVERSION_FACTOR;
        
        System.out.println("Fahrenheit: " + fahrenheit + "¬∞F");
        System.out.println("Celsius: " + celsius + "¬∞C");
        
        // Celsius to Fahrenheit
        double celsiusTemp = 37.0;
        double fahrenheitTemp = (celsiusTemp / CONVERSION_FACTOR) + FREEZING_POINT_F;
        
        System.out.println("\nCelsius: " + celsiusTemp + "¬∞C");
        System.out.println("Fahrenheit: " + fahrenheitTemp + "¬∞F");
        
        // Temperature ranges
        final int FREEZING_C = 0;
        final int BOILING_C = 100;
        
        System.out.println("\n=== Water Properties ===");
        System.out.println("Freezing point: " + FREEZING_C + "¬∞C");
        System.out.println("Boiling point: " + BOILING_C + "¬∞C");
        
        // Check temperature status
        double currentTemp = 25.0;
        boolean isFreezing = currentTemp <= FREEZING_C;
        boolean isBoiling = currentTemp >= BOILING_C;
        boolean isComfortable = currentTemp >= 18 && currentTemp <= 25;
        
        System.out.println("\nCurrent temperature: " + currentTemp + "¬∞C");
        System.out.println("Is freezing? " + isFreezing);
        System.out.println("Is boiling? " + isBoiling);
        System.out.println("Is comfortable? " + isComfortable);
    }
}
Explanation: Practical application using double for precision, final for constants, boolean for conditions, and proper variable naming.
