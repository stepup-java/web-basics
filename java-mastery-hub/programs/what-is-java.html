First Steps with Java Programs

Welcome to your first Java programs! These examples will help you understand the fundamentals of Java programming, from the classic "Hello World" to understanding how the JVM works. Each program is carefully commented to explain what's happening and why.

üí° TIP

Read the code comments carefully! They explain not just WHAT the code does, but WHY it works that way. Understanding the "why" is key to mastering Java.

Program 1: Hello World - The Starting Point

Every Java journey begins with "Hello World". This simple program demonstrates the basic structure of a Java application.

‚òï Java 
/**
 * HelloWorld - Your First Java Program
 * This demonstrates the minimum structure required for a Java application
 */
public class HelloWorld {
    
    // This is the main method - the entry point of every Java application
    // The JVM looks for this exact method signature to start execution
    public static void main(String[] args) {
        
        // Print a message to the console
        System.out.println("Hello, Java World!");
        
        // Let's print a few more things to understand output
        System.out.println("Welcome to Java programming!");
        System.out.println("Java runs on " + System.getProperty("os.name"));
    }
}

/* 
 * COMPILATION AND EXECUTION:
 * 
 * Step 1: Save this file as HelloWorld.java
 * Step 2: Compile it using: javac HelloWorld.java
 *         This creates HelloWorld.class (bytecode)
 * Step 3: Run it using: java HelloWorld
 *         The JVM executes the bytecode
 * 
 * OUTPUT:
 * Hello, Java World!
 * Welcome to Java programming!
 * Java runs on [Your Operating System]
 */
Explanation: This program demonstrates several key Java concepts:
  • Class Declaration: Every Java program must have at least one class. The class name must match the filename.
  • main() Method: This is the entry point. The JVM looks for public static void main(String[] args) to start execution.
  • System.out.println(): Prints text to the console. "System" is a class, "out" is a static field, "println" is a method.
  • Comments: // for single-line, /* */ for multi-line, /** */ for documentation comments.
Why this matters: Understanding the basic structure is crucial. Every Java application, no matter how complex, starts with this foundation.

Program 2: Variables and Data Types Demo

Java is a strongly-typed language. This program demonstrates different data types and how to use variables.

‚òï Java 
/**
 * VariablesDemo - Understanding Java Data Types
 * Demonstrates primitive types and basic variable usage
 */
public class VariablesDemo {
    
    public static void main(String[] args) {
        
        // INTEGER TYPES
        byte smallNumber = 127;           // 8-bit: -128 to 127
        short mediumNumber = 32000;       // 16-bit: -32,768 to 32,767
        int regularNumber = 2147483647;   // 32-bit: most commonly used
        long bigNumber = 9223372036854775807L; // 64-bit: note the 'L' suffix
        
        // FLOATING-POINT TYPES
        float decimalNumber = 3.14f;      // 32-bit: note the 'f' suffix
        double preciseDecimal = 3.14159265359; // 64-bit: more precision
        
        // CHARACTER TYPE
        char letter = 'A';                // 16-bit Unicode character
        char symbol = '‚òï';                // Java supports Unicode!
        
        // BOOLEAN TYPE
        boolean isJavaFun = true;         // Only two values: true or false
        boolean isPythonBetter = false;   // Obviously false! üòÑ
        
        // STRING (Reference Type, not primitive)
        String message = "Java is platform-independent!";
        
        // PRINTING ALL VARIABLES
        System.out.println("=== Java Data Types Demo ===\n");
        
        System.out.println("Integer Types:");
        System.out.println("byte: " + smallNumber);
        System.out.println("short: " + mediumNumber);
        System.out.println("int: " + regularNumber);
        System.out.println("long: " + bigNumber);
        
        System.out.println("\nFloating-Point Types:");
        System.out.println("float: " + decimalNumber);
        System.out.println("double: " + preciseDecimal);
        
        System.out.println("\nCharacter Type:");
        System.out.println("char: " + letter);
        System.out.println("symbol: " + symbol);
        
        System.out.println("\nBoolean Type:");
        System.out.println("Is Java fun? " + isJavaFun);
        System.out.println("Is Python better? " + isPythonBetter);
        
        System.out.println("\nString Type:");
        System.out.println(message);
        
        // VARIABLE OPERATIONS
        int sum = regularNumber + mediumNumber;
        double product = decimalNumber * preciseDecimal;
        String fullMessage = message + " That's why we love it!";
        
        System.out.println("\nOperations:");
        System.out.println("Sum: " + sum);
        System.out.println("Product: " + product);
        System.out.println("Concatenation: " + fullMessage);
    }
}
Explanation: This program introduces Java's type system:
  • Primitive Types: byte, short, int, long, float, double, char, boolean - stored directly in memory
  • Reference Types: String and objects - stored as references to memory locations
  • Type Suffixes: 'L' for long, 'f' for float - tells compiler the exact type
  • Strong Typing: You must declare the type of every variable
Why this matters: Understanding data types is fundamental. Choosing the right type affects memory usage, precision, and program behavior.

Program 4: Dissecting the main() Method

Let's understand every part of the main method signature and why it's written that way.

‚òï Java 
/**
 * MainMethodExplained - Understanding the Entry Point
 * This program explains every keyword in the main method signature
 */
public class MainMethodExplained {
    
    /*
     * Let's break down: public static void main(String[] args)
     * 
     * public:
     *   - Access modifier
     *   - Makes the method accessible from anywhere
     *   - JVM needs to call this method from outside the class
     * 
     * static:
     *   - Means this method belongs to the class, not to instances
     *   - JVM can call it without creating an object
     *   - No need to do: MainMethodExplained obj = new MainMethodExplained();
     * 
     * void:
     *   - Return type
     *   - This method doesn't return any value
     *   - It just executes and finishes
     * 
     * main:
     *   - Method name
     *   - JVM specifically looks for a method named "main"
     *   - This is the entry point of the application
     * 
     * String[] args:
     *   - Parameter (array of strings)
     *   - Allows passing command-line arguments
     *   - "args" is just a variable name (can be anything, but "args" is convention)
     */
    
    public static void main(String[] args) {
        
        System.out.println("=== Understanding the main() Method ===\n");
        
        // Demonstrating command-line arguments
        System.out.println("Number of arguments passed: " + args.length);
        
        if (args.length > 0) {
            System.out.println("\nArguments received:");
            for (int i = 0; i < args.length; i++) {
                System.out.println("  Argument " + i + ": " + args[i]);
            }
        } else {
            System.out.println("\nNo arguments passed.");
            System.out.println("Try running: java MainMethodExplained Hello World 123");
        }
        
        // Calling other methods from main
        System.out.println("\nCalling other methods from main:");
        greetUser();
        displayJavaVersion();
        
        System.out.println("\nProgram execution complete!");
    }
    
    // Another static method (can be called from main without creating object)
    public static void greetUser() {
        System.out.println("  ‚Üí Hello from greetUser() method!");
    }
    
    // Method to display Java version
    public static void displayJavaVersion() {
        String javaVersion = System.getProperty("java.version");
        System.out.println("  ‚Üí You are running Java version: " + javaVersion);
    }
}

/*
 * HOW TO RUN WITH ARGUMENTS:
 * 
 * javac MainMethodExplained.java
 * java MainMethodExplained Hello World 123
 * 
 * This will pass three arguments: "Hello", "World", "123"
 * 
 * WHY MAIN IS STATIC:
 * When JVM starts your program, no objects exist yet!
 * JVM needs to call a method without creating an object first.
 * That's why main() must be static.
 */
Explanation: This program demystifies the main method:
  • public: JVM must access it from outside the class
  • static: JVM calls it without creating an object instance
  • void: Doesn't return a value (execution just ends)
  • String[] args: Accepts command-line arguments as an array
Why this matters: Understanding why main() is written this way helps you understand Java's execution model and the role of the JVM.

public class ClassAndObjectDemo {
    
    public static void main(String[] args) {
        
        System.out.println("=== Classes and Objects in Java ===\n");
        
        // Creating objects using the "new" keyword
        // "new" allocates memory in the heap for the object
        Book book1 = new Book();
        Book book2 = new Book();
        
        // Setting properties for book1
        book1.title = "Effective Java";
        book1.author = "Joshua Bloch";
        book1.pages = 416;

        book2.title = "Clean Code";
        book2.author = "Robert C. Martin";
        book2.pages = 464;
        book2.price = 42.99;
        
        // Calling methods on objects
        book1.displayInfo();
        book2.displayInfo();
        
        // Using methods that return values
        System.out.println("Is '" + book1.title + "' lengthy? " + book1.isLengthy());
        System.out.println("Is '" + book2.title + "' lengthy? " + book2.isLengthy());
        
        // Demonstrating multiple objects
        System.out.println("\n=== Understanding Objects ===");
        System.out.println("book1 and book2 are TWO DIFFERENT objects");
        System.out.println("They are created from the SAME class (Book)");
        System.out.println("Each has its own copy of instance variables");
        System.out.println("\nThink of:");
        System.out.println("  Class = Blueprint/Template");
        System.out.println("  Object = Actual instance created from blueprint");
    }
}

/*
 * KEY CONCEPTS:
 * 
 * 1. Class:
 *    - Blueprint or template for creating objects
 *    - Defines properties (variables) and behaviors (methods)
 * 
 * 2. Object:
 *    - Instance of a class
 *    - Created using "new" keyword
 *    - Has its own copy of instance variables
 * 
 * 3. "new" Keyword:
 *    - Allocates memory in the heap
 *    - Creates a new object instance
 *    - Returns reference to the object
 * 
 * 4. Instance Variables:
 *    - Variables declared inside class but outside methods
 *    - Each object has its own copy
 * 
 * 5. Methods:
 *    - Functions defined inside a class
 *    - Define the behavior of objects
 * 
 * ANALOGY:
 * Class = Cookie cutter (template)
 * Object = Actual cookies made from the cutter
 * You can make many cookies (objects) from one cutter (class)!
 */
Explanation: This program introduces Object-Oriented Programming:
  • Class: Blueprint defining properties and behaviors
  • Object: Instance created from a class using "new" keyword
  • Instance Variables: Each object has its own copy
  • Methods: Define what objects can do
Why this matters: Java is object-oriented. Understanding classes and objects is fundamental to writing Java programs.

/**

public class StringOperationsDemo {

        String language = "Java";

        String str1 = "Java";

/**

public class ArithmeticDemo {
    
    public static void main(String[] args) {
        
        System.out.println("=== Java Arithmetic Operations ===\n");
        
        // BASIC ARITHMETIC OPERATORS
        int a = 20;
        int b = 6;
        
        System.out.println("=== Basic Operators ===");
        System.out.println("a = " + a + ", b = " + b);
        System.out.println("Addition (a + b): " + (a + b));
        System.out.println("Subtraction (a - b): " + (a - b));
        System.out.println("Multiplication (a * b): " + (a * b));
        System.out.println("Division (a / b): " + (a / b));  // Integer division!
        System.out.println("Modulus (a % b): " + (a % b));   // Remainder
        
        // INTEGER VS FLOATING-POINT DIVISION
        System.out.println("\n=== Integer vs Floating-Point Division ===");
        System.out.println("20 / 6 (int): " + (20 / 6));           // 3 (truncated)
        System.out.println("20.0 / 6 (double): " + (20.0 / 6));    // 3.333...
        System.out.println("20 / 6.0 (mixed): " + (20 / 6.0));     // 3.333...
        
        // TYPE CASTING
        System.out.println("\n=== Type Casting ===");
        int num1 = 20;
        int num2 = 6;
        double result = (double) num1 / num2;  // Cast to double
        System.out.println("(double)20 / 6: " + result);
        
        // INCREMENT AND DECREMENT
        System.out.println("\n=== Increment/Decrement ===");
        int counter = 10;
        System.out.println("Initial counter: " + counter);
        System.out.println("counter++: " + counter++);  // Post-increment (use then increment)
        System.out.println("After post-increment: " + counter);
        System.out.println("++counter: " + ++counter);  // Pre-increment (increment then use)
        System.out.println("After pre-increment: " + counter);
        
        // COMPOUND ASSIGNMENT OPERATORS
        System.out.println("\n=== Compound Assignment ===");
        int value = 100;
        System.out.println("Initial value: " + value);
        value += 50;  // Same as: value = value + 50
        System.out.println("After value += 50: " + value);
        value -= 30;  // Same as: value = value - 30
        System.out.println("After value -= 30: " + value);
        value *= 2;   // Same as: value = value * 2
        System.out.println("After value *= 2: " + value);
        value /= 4;   // Same as: value = value / 4
        System.out.println("After value /= 4: " + value);
        
        // OPERATOR PRECEDENCE
        System.out.println("\n=== Operator Precedence ===");
        int result1 = 10 + 5 * 2;           // Multiplication first
        int result2 = (10 + 5) * 2;         // Parentheses first
        System.out.println("10 + 5 * 2 = " + result1);
        System.out.println("(10 + 5) * 2 = " + result2);
        
        // PRACTICAL EXAMPLE: CALCULATOR
        System.out.println("\n=== Simple Calculator ===");
        double num1Double = 25.5;
        double num2Double = 4.2;
        
        System.out.println("Number 1: " + num1Double);
        System.out.println("Number 2: " + num2Double);
        System.out.println("Sum: " + (num1Double + num2Double));
        System.out.println("Difference: " + (num1Double - num2Double));
        System.out.println("Product: " + (num1Double * num2Double));
        System.out.println("Quotient: " + (num1Double / num2Double));

        System.out.println("Math.floor(3.8): " + Math.floor(3.8));      // Floor

    public static double calculateDiscount(double amount) {

        System.out.println("Purchase amount: $" + amount);
        System.out.println("Discount: " + (discount * 100) + "%");
        System.out.println("Savings: $" + (amount * discount));
        
        return discount;
    }

  • Multi-Line Comments (/* */): For longer explanations
  • Javadoc Comments (/** */): For API documentation with tags
  • Best Practices: Comment WHY not WHAT, keep updated, avoid obvious comments
Why this matters: Well-documented code is maintainable code. Comments help you and others understand the code months or years later.
