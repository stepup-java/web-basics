Program 1: Encapsulation (Getters & Setters)

Demonstrates data hiding and controlled access using private fields and public methods.

â˜• Java
/**
 * EncapsulationDemo - Protecting data
 * Baby-level: "Treasure chest with a key"
 */
class BankAccount {
    // Private fields - hidden from outside
    private String accountNumber;
    private double balance;
    
    public BankAccount(String accNum, double initialBalance) {
        this.accountNumber = accNum;
        if (initialBalance >= 0) {
            this.balance = initialBalance;
        }
    }
    
    // Public getter - read access
    public double getBalance() {
        return balance;
    }
    
    // Public setter with validation - write access
    public void deposit(double amount) {
        if (amount > 0) {
            balance += amount;
            System.out.println("Deposited: $" + amount);
        } else {
            System.out.println("Invalid amount!");
        }
    }
    
    public void withdraw(double amount) {
        if (amount > 0 && amount <= balance) {
            balance -= amount;
            System.out.println("Withdrawn: $" + amount);
        } else {
            System.out.println("Transaction failed: Invalid amount or insufficient funds");
        }
    }
}

public class EncapsulationDemo {
    public static void main(String[] args) {
        BankAccount account = new BankAccount("ACC123", 1000);
        
        // account.balance = 5000; // ERROR! Private access
        
        account.deposit(500);
        account.withdraw(200);
        account.withdraw(2000); // Should fail
        
        System.out.println("Final Balance: $" + account.getBalance());
    }
}
Explanation: The balance field is private, so it cannot be modified directly. The deposit and withdraw methods ensure that only valid transactions occur. This protects the data integrity.

Program 2: Single Inheritance

Demonstrates basic parent-child relationship.

â˜• Java
/**
 * InheritanceDemo - Parent and Child
 * Baby-level: "Child inherits traits from parent"
 */
// Parent Class
class Vehicle {
    String brand = "Generic Vehicle";
    
    void honk() {
        System.out.println("Tuut, tuut!");
    }
}

// Child Class - inherits from Vehicle
class Car extends Vehicle {
    String modelName = "Mustang";
    
    void turboBoost() {
        System.out.println("Turbo Boost activated!");
    }
}

public class InheritanceDemo {
    public static void main(String[] args) {
        Car myCar = new Car();
        
        // Accessing parent properties
        System.out.println("Brand: " + myCar.brand);
        myCar.honk();
        
        // Accessing child properties
        System.out.println("Model: " + myCar.modelName);
        myCar.turboBoost();
    }
}
Explanation: Car extends Vehicle, meaning it automatically gets the brand field and honk() method. It also adds its own features like modelName and turboBoost().

Program 3: Multilevel Inheritance

Demonstrates a chain of inheritance.

â˜• Java
/**
 * MultilevelInheritance - Chain of classes
 * Baby-level: "Grandparent -> Parent -> Child"
 */
class Animal {
    void eat() { System.out.println("Eating..."); }
}

class Dog extends Animal {
    void bark() { System.out.println("Barking..."); }
}

class BabyDog extends Dog {
    void weep() { System.out.println("Weeping..."); }
}

public class MultilevelInheritance {
    public static void main(String[] args) {
        BabyDog puppy = new BabyDog();
        
        puppy.eat();  // From Animal (Grandparent)
        puppy.bark(); // From Dog (Parent)
        puppy.weep(); // From BabyDog (Self)
    }
}
Explanation: BabyDog inherits from Dog, which inherits from Animal. So BabyDog has access to methods from all levels of the hierarchy.

Program 4: Hierarchical Inheritance

Demonstrates multiple children from one parent.

â˜• Java
/**
 * HierarchicalInheritance - One parent, many children
 * Baby-level: "Siblings sharing same parent"
 */
class Shape {
    void draw() { System.out.println("Drawing a shape..."); }
}

class Circle extends Shape {
    void roll() { System.out.println("Rolling circle..."); }
}

class Square extends Shape {
    void stack() { System.out.println("Stacking square..."); }
}

public class HierarchicalInheritance {
    public static void main(String[] args) {
        Circle c = new Circle();
        c.draw(); // Inherited
        c.roll(); // Specific
        
        Square s = new Square();
        s.draw(); // Inherited
        s.stack(); // Specific
        
        // s.roll(); // Error! Square doesn't have roll()
    }
}
Explanation: Both Circle and Square inherit from Shape. They share common code but are independent of each other.

Program 5: Method Overriding

Demonstrates changing parent behavior in child class.

â˜• Java
/**
 * OverridingDemo - Changing inherited behavior
 * Baby-level: "Doing the same thing, but differently"
 */
class Bank {
    int getInterestRate() { return 0; }
}

class SBI extends Bank {
    @Override
    int getInterestRate() { return 5; }
}

class ICICI extends Bank {
    @Override
    int getInterestRate() { return 6; }
}

class AXIS extends Bank {
    @Override
    int getInterestRate() { return 7; }
}

public class OverridingDemo {
    public static void main(String[] args) {
        Bank sbi = new SBI();
        Bank icici = new ICICI();
        Bank axis = new AXIS();
        
        System.out.println("SBI Rate: " + sbi.getInterestRate() + "%");
        System.out.println("ICICI Rate: " + icici.getInterestRate() + "%");
        System.out.println("AXIS Rate: " + axis.getInterestRate() + "%");
    }
}
Explanation: Each bank has a getInterestRate() method, but they implement it differently. The @Override annotation ensures we are correctly overriding the parent method.

Program 6: Method Overloading

Demonstrates same method name with different parameters.

â˜• Java
/**
 * OverloadingDemo - Same name, different inputs
 * Baby-level: "Different tools for different jobs"
 */
class Calculator {
    // Add two integers
    int add(int a, int b) {
        return a + b;
    }
    
    // Add three integers
    int add(int a, int b, int c) {
        return a + b + c;
    }
    
    // Add two doubles
    double add(double a, double b) {
        return a + b;
    }
}

public class OverloadingDemo {
    public static void main(String[] args) {
        Calculator calc = new Calculator();
        
        System.out.println("2 params: " + calc.add(10, 20));
        System.out.println("3 params: " + calc.add(10, 20, 30));
        System.out.println("Doubles: " + calc.add(10.5, 20.5));
    }
}
Explanation: The add method is overloaded to handle different numbers and types of arguments. Java decides which one to call at compile time.

Program 7: Abstract Class

Demonstrates partial abstraction where some details are hidden.

â˜• Java
/**
 * AbstractClassDemo - Partial blueprint
 * Baby-level: "Template with some blanks to fill"
 */
abstract class Bike {
    Bike() {
        System.out.println("Bike is created");
    }
    
    // Abstract method - must be implemented by child
    abstract void run();
    
    // Concrete method - ready to use
    void changeGear() {
        System.out.println("Gear changed");
    }
}

class Honda extends Bike {
    @Override
    void run() {
        System.out.println("Honda running safely...");
    }
}

public class AbstractClassDemo {
    public static void main(String[] args) {
        Bike obj = new Honda();
        obj.run();
        obj.changeGear();
    }
}
Explanation: Bike is abstract and cannot be instantiated. It defines a contract (run()) that Honda must implement. It also provides common functionality (changeGear()).

Program 8: Interface

Demonstrates full abstraction and multiple inheritance.

â˜• Java
/**
 * InterfaceDemo - Pure contract
 * Baby-level: "Rules you must follow"
 */
interface Printable {
    void print();
}

interface Showable {
    void show();
}

// Implementing multiple interfaces
class Document implements Printable, Showable {
    public void print() {
        System.out.println("Printing document...");
    }
    
    public void show() {
        System.out.println("Showing document...");
    }
}

public class InterfaceDemo {
    public static void main(String[] args) {
        Document doc = new Document();
        doc.print();
        doc.show();
    }
}
Explanation: Document implements two interfaces, achieving multiple inheritance of type. It must provide implementations for all methods defined in the interfaces.

Program 9: Constructor Chaining

Demonstrates how constructors work in inheritance.

â˜• Java
/**
 * SuperConstructorDemo - Parent first
 * Baby-level: "Building foundation before the house"
 */
class Person {
    Person() {
        System.out.println("Person created");
    }
}

class Employee extends Person {
    Employee() {
        super(); // Implicitly called if not written
        System.out.println("Employee created");
    }
}

public class SuperConstructorDemo {
    public static void main(String[] args) {
        new Employee();
    }
}
Explanation: When creating an Employee, the Person constructor runs first. This ensures the parent part of the object is initialized before the child part.

Program 10: 'this' Keyword

Demonstrates referring to the current object instance.

â˜• Java
/**
 * ThisKeywordDemo - Me, myself, and I
 * Baby-level: "Pointing to yourself"
 */
class Student {
    int rollno;
    String name;
    
    Student(int rollno, String name) {
        // Distinguish field from parameter
        this.rollno = rollno;
        this.name = name;
    }
    
    void display() {
        System.out.println(rollno + " " + name);
    }
}

public class ThisKeywordDemo {
    public static void main(String[] args) {
        Student s1 = new Student(111, "Ankit");
        Student s2 = new Student(112, "Sumit");
        s1.display();
        s2.display();
    }
}
Explanation: this.rollno refers to the class field, while rollno refers to the constructor parameter. It resolves naming conflicts.

Program 11: 'super' Keyword

Demonstrates accessing parent class members.

â˜• Java
/**
 * SuperKeywordDemo - Accessing parent
 * Baby-level: "Asking mom/dad for something"
 */
class Animal {
    String color = "white";
    
    void eat() {
        System.out.println("eating...");
    }
}

class Dog extends Animal {
    String color = "black";
    
    void printColor() {
        System.out.println(color); // prints black (current class)
        System.out.println(super.color); // prints white (parent class)
    }
    
    void eat() {
        System.out.println("eating bread...");
    }
    
    void work() {
        super.eat(); // calls parent's eat()
        eat();       // calls current class's eat()
    }
}

public class SuperKeywordDemo {
    public static void main(String[] args) {
        Dog d = new Dog();
        d.printColor();
        d.work();
    }
}
Explanation: super allows access to the parent class's fields and methods, even if they are hidden or overridden in the child class.

Program 12: Final Keyword

Demonstrates constants and preventing inheritance/overriding.

â˜• Java
/**
 * FinalDemo - The unchangeable
 * Baby-level: "Written in stone"
 */
class Bike {
    final int speedLimit = 90; // final variable
    
    final void run() { // final method
        System.out.println("running");
    }
}

class Honda extends Bike {
    // void run() { System.out.println("running safely"); } // Compile Error!
}

public class FinalDemo {
    public static void main(String[] args) {
        Bike obj = new Bike();
        // obj.speedLimit = 400; // Compile Error!
        System.out.println("Speed limit: " + obj.speedLimit);
    }
}
Explanation: final variable cannot be changed. final method cannot be overridden. final class cannot be extended.

Program 13: Static vs Instance Blocks

Demonstrates initialization order.

â˜• Java
/**
 * BlockDemo - Initialization order
 * Baby-level: "Setting up the stage"
 */
public class BlockDemo {
    // Static block - runs once when class loads
    static {
        System.out.println("Static block invoked");
    }
    
    // Instance block - runs before every constructor
    {
        System.out.println("Instance initializer block invoked");
    }
    
    BlockDemo() {
        System.out.println("Constructor invoked");
    }
    
    public static void main(String[] args) {
        System.out.println("Main method started");
        new BlockDemo();
        new BlockDemo();
    }
}
Explanation: Static blocks run once. Instance blocks run for every object creation, before the constructor.

Program 14: Real-world Polymorphism

A practical example of polymorphism in a payment system.

â˜• Java
/**
 * PaymentSystem - Polymorphism in action
 * Baby-level: "Paying with different cards"
 */
interface PaymentStrategy {
    void pay(int amount);
}

class CreditCard implements PaymentStrategy {
    public void pay(int amount) {
        System.out.println("Paid " + amount + " using Credit Card");
    }
}

class PayPal implements PaymentStrategy {
    public void pay(int amount) {
        System.out.println("Paid " + amount + " using PayPal");
    }
}

class ShoppingCart {
    void checkout(int amount, PaymentStrategy paymentMethod) {
        paymentMethod.pay(amount);
    }
}

public class PaymentDemo {
    public static void main(String[] args) {
        ShoppingCart cart = new ShoppingCart();
        
        // Pay using Credit Card
        cart.checkout(100, new CreditCard());
        
        // Pay using PayPal
        cart.checkout(200, new PayPal());
    }
}
Explanation: The ShoppingCart doesn't care how the payment is made. It just calls pay() on whatever PaymentStrategy is passed. This is the power of loose coupling!

Program 15: Composition (HAS-A)

Demonstrates object ownership.

â˜• Java
/**
 * CompositionDemo - Building complex objects
 * Baby-level: "Car HAS-A Engine"
 */
class Engine {
    void start() {
        System.out.println("Engine started");
    }
}

class Car {
    private Engine engine; // Car HAS-A Engine
    
    Car() {
        engine = new Engine();
    }
    
    void startCar() {
        engine.start();
        System.out.println("Car is moving");
    }
}

public class CompositionDemo {
    public static void main(String[] args) {
        Car car = new Car();
        car.startCar();
    }
}
Explanation: The Car class contains an Engine object. This is composition. The car delegates the starting logic to the engine.
