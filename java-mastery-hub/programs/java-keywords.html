Hands-On Java Programs

Welcome to the practical section! These 12+ programs demonstrate every major Java keyword category. Each program is heavily commented to explain not just WHAT the code does, but WHY and HOW each keyword works.

Try running these programs yourself! Experiment with different keywords, observe compilation errors, and understand how keywords control program behavior. Hands-on practice is essential!

Program 1: Access Modifiers (public, private, protected)

Demonstrates how access modifiers control visibility of class members.

‚òï Java 
/**
 * AccessModifiersDemo - Understanding public, private, protected
 * 
 * Baby-level: Like different types of rooms - some anyone can enter,
 * some only family, some only you!
 */
public class AccessModifiersDemo {
    
    // PUBLIC: Accessible from anywhere
    public String publicName = "Everyone can see me!";
    
    // PRIVATE: Only accessible within this class
    private String privateSecret = "Only I can see this!";
    
    // PROTECTED: Accessible in same package and subclasses
    protected String protectedData = "Family can see me!";
    
    // DEFAULT (no modifier): Accessible only in same package
    String defaultValue = "Package friends can see me!";
    
    // PUBLIC method - anyone can call
    public void publicMethod() {
        System.out.println("Public method called!");
        // Can access all members inside the class
        System.out.println("Private secret: " + privateSecret);
    }
    
    // PRIVATE method - only this class can call
    private void privateMethod() {
        System.out.println("Private method - only internal use!");
    }
    
    // PROTECTED method - same package + subclasses
    protected void protectedMethod() {
        System.out.println("Protected method - for family!");
    }
    
    public static void main(String[] args) {
        AccessModifiersDemo demo = new AccessModifiersDemo();
        
        // ‚úÖ Can access public
        System.out.println(demo.publicName);
        
        // ‚úÖ Can access private (we're inside the class)
        System.out.println(demo.privateSecret);
        
        // ‚úÖ Can call private method (we're inside the class)
        demo.privateMethod();
        
        // Real-world analogy: Access modifiers are like security clearances
        // - public: Public information (anyone)
        // - protected: Internal company (employees + partners)
        // - default: Department level (same department)
        // - private: Top secret (specific person only)
    }
}
Explanation: Access modifiers control who can access class members. Use private for encapsulation (hiding internal details), public for the class's API, and protected for inheritance hierarchies. Best practice: Start with private and only make more accessible if needed.

Program 2: static Keyword

Demonstrates static variables and methods that belong to the class, not instances.

‚òï Java 
/**
 * StaticKeywordDemo - Understanding static members
 * 
 * Baby-level: static is like a shared toy that all kids use together,
 * not a personal toy that each kid has their own copy of!
 */
public class StaticKeywordDemo {
    
    // STATIC variable - shared by all instances
    static int studentCount = 0;
    
    // INSTANCE variable - each object has its own copy
    String studentName;
    
    // STATIC constant - shared, unchangeable value
    public static final String SCHOOL_NAME = "Java Academy";
    
    // Constructor
    public StaticKeywordDemo(String name) {
        this.studentName = name;
        studentCount++;  // Increment shared counter
    }
    
    // STATIC method - belongs to class, not object
    public static void displaySchoolInfo() {
        System.out.println("School: " + SCHOOL_NAME);
        System.out.println("Total students: " + studentCount);
        // ‚ùå Cannot access instance variables from static method
        // System.out.println(studentName);  // Compilation error!
    }
    
    // INSTANCE method - belongs to object
    public void displayStudentInfo() {
        System.out.println("Student: " + studentName);
        System.out.println("School: " + SCHOOL_NAME);  // ‚úÖ Can access static
        System.out.println("Total students: " + studentCount);  // ‚úÖ Can access static
    }
    
    public static void main(String[] args) {
        // Call static method without creating object
        StaticKeywordDemo.displaySchoolInfo();
        
        // Create objects
        StaticKeywordDemo student1 = new StaticKeywordDemo("Alice");
        StaticKeywordDemo student2 = new StaticKeywordDemo("Bob");
        StaticKeywordDemo student3 = new StaticKeywordDemo("Charlie");
        
        // Static variable is shared - all see same value
        System.out.println("\nAfter creating 3 students:");
        StaticKeywordDemo.displaySchoolInfo();
        
        student1.displayStudentInfo();
        
        // Real-world analogy:
        // - static: Community swimming pool (shared by all)
        // - instance: Personal backyard pool (each house has own)
        
        // Industry usage:
        // - Utility methods: Math.sqrt(), Arrays.sort()
        // - Constants: Math.PI, Integer.MAX_VALUE
        // - Factory methods: LocalDate.now()
    }
}
Explanation: static members belong to the class itself, not to individual objects. They're shared across all instances. Use for utility methods, constants, and counters. Static methods cannot access instance variables because they don't belong to any specific object.

Program 3: final Keyword (Variable, Method, Class)

Demonstrates final for constants, preventing method override, and preventing inheritance.

‚òï Java 
/**
 * FinalKeywordDemo - Understanding final variables, methods, and classes
 * 
 * Baby-level: final is like writing in permanent marker - once written,
 * you cannot erase or change it!
 */

// FINAL CLASS - cannot be extended
final class ImmutableClass {
    // No class can inherit from this
}

class Parent {
    // FINAL METHOD - cannot be overridden
    public final void criticalMethod() {
        System.out.println("This method cannot be changed by children!");
    }
    
    public void normalMethod() {
        System.out.println("This can be overridden");
    }
}

class Child extends Parent {
    // ‚ùå Cannot override final method
    // public void criticalMethod() { }  // Compilation error!
    
    // ‚úÖ Can override normal method
    @Override
    public void normalMethod() {
        System.out.println("Child's version");
    }
}

public class FinalKeywordDemo {
    
    // FINAL VARIABLE - constant (cannot be reassigned)
    public static final double PI = 3.14159265359;
    public static final int MAX_USERS = 100;
    
    // Final instance variable - must be initialized
    private final String id;
    
    public FinalKeywordDemo(String id) {
        this.id = id;  // ‚úÖ Can assign once in constructor
    }
    
    public void demonstrateFinal() {
        // Final local variable
        final int MAX_ATTEMPTS = 3;
        
        System.out.println("Max attempts: " + MAX_ATTEMPTS);
        
        // ‚ùå Cannot reassign final variable
        // MAX_ATTEMPTS = 5;  // Compilation error!
        // PI = 3.14;  // Compilation error!
        
        // Final reference - reference cannot change, but object can
        final StringBuilder sb = new StringBuilder("Hello");
        sb.append(" World");  // ‚úÖ Can modify object
        System.out.println(sb);
        
        // ‚ùå Cannot reassign reference
        // sb = new StringBuilder("New");  // Compilation error!
    }
    
    public static void main(String[] args) {
        System.out.println("PI: " + PI);
        System.out.println("MAX_USERS: " + MAX_USERS);
        
        FinalKeywordDemo demo = new FinalKeywordDemo("USER123");
        demo.demonstrateFinal();
        
        // Real-world analogy:
        // - final variable: Permanent tattoo (cannot change)
        // - final method: Company policy (cannot be altered)
        // - final class: Sealed vault (cannot be extended)
        
        // Best practices:
        // - Use final for constants: public static final
        // - Use final for parameters that shouldn't change
        // - Use final for security-critical classes (String, Integer)
    }
}
Explanation: final prevents modification. For variables, it creates constants. For methods, it prevents overriding (useful for security). For classes, it prevents inheritance. Note: final reference means the reference cannot change, but the object it points to can still be modified.

Program 4: abstract Keyword (Class and Method)

Demonstrates abstract classes and methods for creating templates.

‚òï Java 
/**
 * AbstractKeywordDemo - Understanding abstract classes and methods
 * 
 * Baby-level: abstract is like a coloring book outline - it shows the
 * shape but you need to fill in the colors yourself!
 */

// ABSTRACT CLASS - cannot be instantiated directly
abstract class Animal {
    
    String name;
    
    // Constructor in abstract class
    public Animal(String name) {
        this.name = name;
    }
    
    // ABSTRACT METHOD - no implementation (must be overridden)
    public abstract void makeSound();
    
    // ABSTRACT METHOD
    public abstract void move();
    
    // CONCRETE METHOD - has implementation
    public void sleep() {
        System.out.println(name + " is sleeping... Zzz");
    }
    
    public void eat() {
        System.out.println(name + " is eating");
    }
}

// Concrete class - must implement all abstract methods
class Dog extends Animal {
    
    public Dog(String name) {
        super(name);
    }
    
    // ‚úÖ Must implement abstract method
    @Override
    public void makeSound() {
        System.out.println(name + " says: Woof! Woof!");
    }
    
    @Override
    public void move() {
        System.out.println(name + " runs on four legs");
    }
}

class Bird extends Animal {
    
    public Bird(String name) {
        super(name);
    }
    
    @Override
    public void makeSound() {
        System.out.println(name + " says: Tweet! Tweet!");
    }
    
    @Override
    public void move() {
        System.out.println(name + " flies in the sky");
    }
}

public class AbstractKeywordDemo {
    
    public static void main(String[] args) {
        // ‚ùå Cannot instantiate abstract class
        // Animal animal = new Animal("Generic");  // Compilation error!
        
        // ‚úÖ Can create concrete subclass instances
        Animal dog = new Dog("Buddy");
        Animal bird = new Bird("Tweety");
        
        // Call methods
        dog.makeSound();   // Woof! Woof!
        dog.move();        // runs on four legs
        dog.sleep();       // is sleeping...
        
        System.out.println();
        
        bird.makeSound();  // Tweet! Tweet!
        bird.move();       // flies in the sky
        bird.eat();        // is eating
        
        // Real-world analogy:
        // - abstract class: Car blueprint (shows structure, not actual car)
        // - abstract method: "Add engine" instruction (what to do, not how)
        // - concrete class: Actual car model (fully implemented)
        
        // Use cases:
        // - Template pattern: Define skeleton, let subclasses fill details
        // - Common behavior: Share code among related classes
        // - Enforce contract: Force subclasses to implement specific methods
    }
}
Explanation: abstract classes provide templates. They can have both abstract methods (no implementation) and concrete methods (with implementation). Subclasses must implement all abstract methods. Use abstract classes when you want to share code and enforce a contract for subclasses.

Program 5: synchronized Keyword (Thread Safety)

Demonstrates synchronized methods for thread-safe operations.

‚òï Java 
/**
 * SynchronizedKeywordDemo - Understanding thread synchronization
 * 
 * Baby-level: synchronized is like a bathroom - only ONE person can
 * use it at a time. Others must wait in line!
 */

class BankAccount {
    private int balance = 1000;
    
    // SYNCHRONIZED METHOD - thread-safe
    // Only one thread can execute this at a time
    public synchronized void withdraw(int amount) {
        System.out.println(Thread.currentThread().getName() + 
                          " attempting to withdraw: $" + amount);
        
        if (balance >= amount) {
            System.out.println(Thread.currentThread().getName() + 
                              " checking balance: $" + balance);
            
            // Simulate processing time
            try {
                Thread.sleep(100);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
            
            balance -= amount;
            System.out.println(Thread.currentThread().getName() + 
                              " completed withdrawal. New balance: $" + balance);
        } else {
            System.out.println(Thread.currentThread().getName() + 
                              " insufficient funds!");
        }
    }
    
    // Non-synchronized method for comparison
    public void deposit(int amount) {
        balance += amount;
        System.out.println("Deposited: $" + amount + 
                          ". Balance: $" + balance);
    }
    
    public synchronized int getBalance() {
        return balance;
    }
}

public class SynchronizedKeywordDemo {
    
    public static void main(String[] args) {
        BankAccount account = new BankAccount();
        
        // Create multiple threads trying to withdraw simultaneously
        Thread thread1 = new Thread(() -> {
            account.withdraw(600);
        }, "Thread-1");
        
        Thread thread2 = new Thread(() -> {
            account.withdraw(600);
        }, "Thread-2");
        
        // Start both threads
        thread1.start();
        thread2.start();
        
        // Wait for threads to complete
        try {
            thread1.join();
            thread2.join();
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
        
        System.out.println("\nFinal balance: $" + account.getBalance());
        
        // Real-world analogy:
        // - synchronized: Single-occupancy restroom (one at a time)
        // - non-synchronized: Open park (everyone at once)
        
        // Without synchronized:
        // - Both threads might see balance = 1000
        // - Both withdraw 600
        // - Balance becomes -200 (incorrect!)
        
        // With synchronized:
        // - Thread-1 locks method, withdraws 600 (balance = 400)
        // - Thread-2 waits, then sees balance = 400
        // - Thread-2 cannot withdraw (insufficient funds)
        // - Final balance = 400 (correct!)
    }
}
Explanation: synchronized ensures thread safety by allowing only one thread to execute a method at a time. Critical for shared resources like bank accounts, counters, or any data accessed by multiple threads. Without synchronization, race conditions can cause data corruption.

Program 6: volatile Keyword (Thread Visibility)

Demonstrates volatile for ensuring variable visibility across threads.

‚òï Java 
/**
 * VolatileKeywordDemo - Understanding volatile for thread visibility
 * 
 * Baby-level: volatile is like a public announcement board - when you
 * write something, EVERYONE sees it immediately, not just your notes!
 */

class TaskRunner {
    
    // VOLATILE - ensures all threads see the latest value
    // Without volatile, threads might cache the value
    private volatile boolean running = true;
    
    private int counter = 0;
    
    public void start() {
        new Thread(() -> {
            System.out.println("Task started...");
            
            while (running) {
                counter++;
                // Simulate work
                if (counter % 100000000 == 0) {
                    System.out.println("Still running... counter: " + counter);
                }
            }
            
            System.out.println("Task stopped! Final counter: " + counter);
        }, "Worker-Thread").start();
    }
    
    public void stop() {
        System.out.println("Stopping task...");
        running = false;  // All threads see this change immediately
    }
}

public class VolatileKeywordDemo {
    
    public static void main(String[] args) throws InterruptedException {
        TaskRunner task = new TaskRunner();
        
        // Start the task
        task.start();
        
        // Let it run for 2 seconds
        Thread.sleep(2000);
        
        // Stop the task
        task.stop();
        
        // Wait a bit to see the result
        Thread.sleep(500);
        
        System.out.println("\nMain thread finished");
        
        // Real-world analogy:
        // - volatile: Live TV broadcast (everyone sees same thing)
        // - non-volatile: Recorded video (might be outdated)
        
        // Technical explanation:
        // - Without volatile: Thread might cache 'running' value
        // - Thread keeps seeing running = true (cached value)
        // - Loop never stops even after stop() is called!
        
        // - With volatile: Thread always reads from main memory
        // - Thread sees running = false immediately
        // - Loop stops as expected
        
        // Use volatile when:
        // - Variable is accessed by multiple threads
        // - One thread writes, others read
        // - No compound operations (like counter++)
    }
}
Explanation: volatile ensures that variable reads/writes go directly to main memory, not thread-local cache. This guarantees visibility across threads. Use for flags and state variables accessed by multiple threads. Note: volatile doesn't guarantee atomicity for compound operations like counter++.

Program 7: this Keyword

Demonstrates various uses of the this keyword.

‚òï Java 
/**
 * ThisKeywordDemo - Understanding 'this' reference
 * 
 * Baby-level: 'this' is like pointing to yourself and saying "ME!"
 * when someone asks "who wants ice cream?"
 */

class Person {
    private String name;
    private int age;
    
    // USE 1: Distinguish instance variable from parameter
    public Person(String name, int age) {
        this.name = name;  // this.name = instance variable
        this.age = age;    // name = parameter
    }
    
    // USE 2: Call another constructor (constructor chaining)
    public Person(String name) {
        this(name, 0);  // Calls Person(String, int) constructor
        System.out.println("Single-parameter constructor called");
    }
    
    // USE 3: Return current object (method chaining)
    public Person setName(String name) {
        this.name = name;
        return this;  // Returns current object
    }
    
    public Person setAge(int age) {
        this.age = age;
        return this;  // Returns current object
    }
    
    // USE 4: Pass current object as parameter
    public void display() {
        System.out.println("Name: " + this.name + ", Age: " + this.age);
        this.printDetails(this);  // Passing current object
    }
    
    private void printDetails(Person person) {
        System.out.println("Printing details for: " + person.name);
    }
    
    // USE 5: Explicit reference to instance members
    public void compareWith(Person other) {
        if (this.age > other.age) {
            System.out.println(this.name + " is older than " + other.name);
        } else {
            System.out.println(other.name + " is older than " + this.name);
        }
    }
}

public class ThisKeywordDemo {
    
    public static void main(String[] args) {
        // Constructor chaining
        Person person1 = new Person("Alice");
        person1.display();
        
        System.out.println();
        
        // Method chaining (fluent interface)
        Person person2 = new Person("Bob", 25)
                            .setName("Robert")
                            .setAge(30);
        person2.display();
        
        System.out.println();
        
        // Comparing objects
        Person person3 = new Person("Charlie", 35);
        person2.compareWith(person3);
        
        // Real-world analogy:
        // - this: Your name tag at a conference (identifies YOU)
        // - this.name: Your specific name (not someone else's)
        
        // Industry usage:
        // - Builder pattern: return this for chaining
        // - Distinguishing parameters from fields
        // - Passing current object to other methods
        // - Constructor chaining for code reuse
    }
}
Explanation: this refers to the current object. Use it to distinguish instance variables from parameters, call other constructors, enable method chaining, and pass the current object as a parameter. Essential for clean, readable code and design patterns like Builder.

Program 8: super Keyword

Demonstrates super for accessing parent class members.

‚òï Java 
/**
 * SuperKeywordDemo - Understanding 'super' reference
 * 
 * Baby-level: 'super' is like saying "my parent" or "my mom/dad"
 * when you want to refer to your parent's stuff!
 */

class Vehicle {
    String type = "Vehicle";
    int maxSpeed = 100;
    
    public Vehicle() {
        System.out.println("Vehicle constructor called");
    }
    
    public Vehicle(int maxSpeed) {
        this.maxSpeed = maxSpeed;
        System.out.println("Vehicle constructor with speed: " + maxSpeed);
    }
    
    public void start() {
        System.out.println("Vehicle is starting...");
    }
    
    public void displayInfo() {
        System.out.println("Type: " + type + ", Max Speed: " + maxSpeed);
    }
}

class Car extends Vehicle {
    String type = "Car";  // Shadows parent's type
    String brand;
    
    // USE 1: Call parent constructor
    public Car(String brand, int maxSpeed) {
        super(maxSpeed);  // Must be first line! Calls Vehicle(int)
        this.brand = brand;
        System.out.println("Car constructor called");
    }
    
    // USE 2: Access parent's variable (when shadowed)
    public void showTypes() {
        System.out.println("Child type: " + this.type);    // "Car"
        System.out.println("Parent type: " + super.type);  // "Vehicle"
    }
    
    // USE 3: Call parent's method (when overridden)
    @Override
    public void start() {
        super.start();  // Call parent's start() first
        System.out.println("Car engine starting with key...");
        System.out.println(brand + " is ready to drive!");
    }
    
    // USE 4: Access parent's method
    public void displayFullInfo() {
        super.displayInfo();  // Call parent's method
        System.out.println("Brand: " + brand);
    }
}

class SportsCar extends Car {
    boolean turboMode;
    
    public SportsCar(String brand, int maxSpeed, boolean turboMode) {
        super(brand, maxSpeed);  // Call Car constructor
        this.turboMode = turboMode;
        System.out.println("SportsCar constructor called");
    }
    
    @Override
    public void start() {
        super.start();  // Call Car's start (which calls Vehicle's start)
        if (turboMode) {
            System.out.println("TURBO MODE ACTIVATED! üöÄ");
        }
    }
}

public class SuperKeywordDemo {
    
    public static void main(String[] args) {
        System.out.println("=== Creating Car ===");
        Car car = new Car("Toyota", 180);
        car.showTypes();
        
        System.out.println("\n=== Starting Car ===");
        car.start();
        
        System.out.println("\n=== Car Info ===");
        car.displayFullInfo();
        
        System.out.println("\n=== Creating SportsCar ===");
        SportsCar sportsCar = new SportsCar("Ferrari", 320, true);
        
        System.out.println("\n=== Starting SportsCar ===");
        sportsCar.start();
        
        // Real-world analogy:
        // - super: Calling your parent for help
        // - super(): "Mom, Dad, I need you!" (constructor)
        // - super.method(): "How did you do this, Mom?" (method)
        
        // Key rules:
        // - super() must be first line in constructor
        // - Cannot use both this() and super() in same constructor
        // - Use super to access parent's overridden methods
        // - Use super to access parent's shadowed variables
    }
}
Explanation: super refers to the parent class. Use it to call parent constructors, access parent methods (especially when overridden), and access parent variables (when shadowed). Essential for inheritance and code reuse. Remember: super() must be the first line in a constructor.

Program 9: interface and implements Keywords

Demonstrates interfaces and implementation.

‚òï Java 
/**
 * InterfaceImplementsDemo - Understanding interfaces
 * 
 * Baby-level: interface is like a list of chores you MUST do.
 * implements means "I promise to do all these chores!"
 */

// INTERFACE - contract defining what must be done
interface Flyable {
    // Abstract methods (public abstract by default)
    void fly();
    void land();
    
    // Default method (Java 8+) - has implementation
    default void checkWeather() {
        System.out.println("Weather is clear for flying");
    }
    
    // Static method (Java 8+)
    static void displayInfo() {
        System.out.println("Flyable interface - for flying objects");
    }
    
    // Constant (public static final by default)
    int MAX_ALTITUDE = 40000;
}

interface Swimmable {
    void swim();
    void dive();
}

// Class implementing single interface
class Airplane implements Flyable {
    
    // MUST implement all abstract methods
    @Override
    public void fly() {
        System.out.println("Airplane flying with engines");
    }
    
    @Override
    public void land() {
        System.out.println("Airplane landing on runway");
    }
}

// Class implementing multiple interfaces
class Duck implements Flyable, Swimmable {
    
    // Implement Flyable methods
    @Override
    public void fly() {
        System.out.println("Duck flying with wings");
    }
    
    @Override
    public void land() {
        System.out.println("Duck landing on water");
    }
    
    // Implement Swimmable methods
    @Override
    public void swim() {
        System.out.println("Duck swimming on water");
    }
    
    @Override
    public void dive() {
        System.out.println("Duck diving for food");
    }
    
    // Can override default method
    @Override
    public void checkWeather() {
        System.out.println("Duck checking weather with instinct");
    }
}

public class InterfaceImplementsDemo {
    
    public static void main(String[] args) {
        // Using interface reference
        Flyable airplane = new Airplane();
        airplane.fly();
        airplane.land();
        airplane.checkWeather();  // Default method
        
        System.out.println("Max altitude: " + Flyable.MAX_ALTITUDE);
        Flyable.displayInfo();  // Static method
        
        System.out.println("\n=== Duck Demo ===");
        Duck duck = new Duck();
        
        // Duck can do everything
        duck.fly();
        duck.land();
        duck.swim();
        duck.dive();
        duck.checkWeather();
        
        // Polymorphism with interfaces
        System.out.println("\n=== Polymorphism ===");
        Flyable flyingDuck = duck;
        flyingDuck.fly();
        
        Swimmable swimmingDuck = duck;
        swimmingDuck.swim();
        
        // Real-world analogy:
        // - interface: Job description (what you must do)
        // - implements: Accepting the job (promising to do it)
        
        // Key points:
        // - Interface = 100% abstraction (before Java 8)
        // - Class can implement multiple interfaces
        // - Interface methods are public abstract by default
        // - Interface variables are public static final by default
        // - Use interfaces for "CAN-DO" relationships (Duck CAN fly, CAN swim)
    }
}
Explanation: interface defines a contract that classes must follow. implements means the class promises to provide implementations for all abstract methods. Interfaces enable multiple inheritance and define capabilities (CAN-DO relationships). Modern interfaces can have default and static methods.

Program 10: class and extends Keywords

Demonstrates class definition and inheritance using extends.

‚òï Java 
/**
 * ClassExtendsDemo - Understanding class and extends
 * 
 * Baby-level: class is like a recipe, extends is like saying
 * "I'll use mom's recipe but add my own special ingredient!"
 */

// BASE CLASS (Parent/Superclass)
class Animal {
    protected String name;
    protected int age;
    
    public Animal(String name, int age) {
        this.name = name;
        this.age = age;
        System.out.println("Animal constructor: " + name);
    }
    
    public void eat() {
        System.out.println(name + " is eating");
    }
    
    public void sleep() {
        System.out.println(name + " is sleeping");
    }
    
    public void makeSound() {
        System.out.println(name + " makes a sound");
    }
}

// DERIVED CLASS (Child/Subclass) - EXTENDS Animal
class Dog extends Animal {
    private String breed;
    
    public Dog(String name, int age, String breed) {
        super(name, age);  // Call parent constructor
        this.breed = breed;
        System.out.println("Dog constructor: " + breed);
    }
    
    // OVERRIDE parent method
    @Override
    public void makeSound() {
        System.out.println(name + " barks: Woof! Woof!");
    }
    
    // NEW method (specific to Dog)
    public void fetch() {
        System.out.println(name + " is fetching the ball");
    }
    
    public void displayInfo() {
        System.out.println("Name: " + name + ", Age: " + age + 
                          ", Breed: " + breed);
    }
}

// MULTI-LEVEL INHERITANCE
class Puppy extends Dog {
    private boolean isVaccinated;
    
    public Puppy(String name, int age, String breed, boolean isVaccinated) {
        super(name, age, breed);
        this.isVaccinated = isVaccinated;
        System.out.println("Puppy constructor");
    }
    
    @Override
    public void makeSound() {
        System.out.println(name + " makes puppy sounds: Yip! Yip!");
    }
    
    public void play() {
        System.out.println(name + " is playing with toys");
    }
}

public class ClassExtendsDemo {
    
    public static void main(String[] args) {
        System.out.println("=== Creating Dog ===");
        Dog dog = new Dog("Buddy", 5, "Golden Retriever");
        dog.displayInfo();
        
        // Inherited methods from Animal
        dog.eat();
        dog.sleep();
        
        // Overridden method
        dog.makeSound();
        
        // Dog-specific method
        dog.fetch();
        
        System.out.println("\n=== Creating Puppy ===");
        Puppy puppy = new Puppy("Max", 1, "Labrador", true);
        
        // Inherited from Animal
        puppy.eat();
        
        // Inherited from Dog
        puppy.fetch();
        
        // Overridden in Puppy
        puppy.makeSound();
        
        // Puppy-specific
        puppy.play();
        
        // Polymorphism
        System.out.println("\n=== Polymorphism ===");
        Animal animal1 = new Dog("Charlie", 3, "Beagle");
        Animal animal2 = new Puppy("Lucy", 1, "Poodle", true);
        
        animal1.makeSound();  // Dog's version
        animal2.makeSound();  // Puppy's version
        
        // Real-world analogy:
        // - class: Blueprint for building something
        // - extends: "Based on existing blueprint, with modifications"
        
        // IS-A relationship:
        // - Dog IS-A Animal ‚úÖ
        // - Puppy IS-A Dog ‚úÖ
        // - Puppy IS-A Animal ‚úÖ
        
        // Inheritance benefits:
        // - Code reuse (don't repeat eat(), sleep())
        // - Polymorphism (treat Dog as Animal)
        // - Extensibility (add new animal types easily)
    }
}
Explanation: class defines a blueprint for objects. extends creates an inheritance relationship (IS-A). Child class inherits all non-private members from parent. Use inheritance for code reuse and polymorphism. Java supports single inheritance (one parent) but multi-level inheritance (grandparent ‚Üí parent ‚Üí child).

Program 11: try-catch-finally (Exception Handling)

Demonstrates exception handling with try, catch, and finally.

‚òï Java 
/**
 * TryCatchFinallyDemo - Understanding exception handling
 * 
 * Baby-level: try is like "I'll try to do this", catch is like
 * "If something goes wrong, I'll handle it", finally is like
 * "No matter what, I'll do this at the end"
 */

import java.io.*;

public class TryCatchFinallyDemo {
    
    // Example 1: Basic try-catch
    public static void example1() {
        System.out.println("=== Example 1: Basic try-catch ===");
        
        try {
            int result = 10 / 0;  // ArithmeticException!
            System.out.println("Result: " + result);
        } catch (ArithmeticException e) {
            System.out.println("Error: Cannot divide by zero!");
            System.out.println("Exception: " + e.getMessage());
        }
        
        System.out.println("Program continues...\n");
    }
    
    // Example 2: Multiple catch blocks
    public static void example2() {
        System.out.println("=== Example 2: Multiple catch blocks ===");
        
        try {
            String str = null;
            System.out.println(str.length());  // NullPointerException!
        } catch (NullPointerException e) {
            System.out.println("Error: String is null!");
        } catch (Exception e) {
            System.out.println("Error: Something else went wrong!");
        }
        
        System.out.println("Program continues...\n");
    }
    
    // Example 3: try-catch-finally
    public static void example3() {
        System.out.println("=== Example 3: try-catch-finally ===");
        
        FileReader reader = null;
        
        try {
            reader = new FileReader("nonexistent.txt");
            System.out.println("File opened");
        } catch (FileNotFoundException e) {
            System.out.println("Error: File not found!");
        } finally {
            // ALWAYS executes, even if exception occurs
            System.out.println("Finally block: Cleanup code");
            
            if (reader != null) {
                try {
                    reader.close();
                } catch (IOException e) {
                    System.out.println("Error closing file");
                }
            }
        }
        
        System.out.println("Program continues...\n");
    }
    
    // Example 4: finally executes even with return
    public static int example4() {
        System.out.println("=== Example 4: finally with return ===");
        
        try {
            System.out.println("In try block");
            return 1;
        } catch (Exception e) {
            System.out.println("In catch block");
            return 2;
        } finally {
            // Executes BEFORE return!
            System.out.println("In finally block - executes before return!");
        }
    }
    
    // Example 5: Nested try-catch
    public static void example5() {
        System.out.println("=== Example 5: Nested try-catch ===");
        
        try {
            System.out.println("Outer try");
            
            try {
                int[] arr = new int[5];
                arr[10] = 50;  // ArrayIndexOutOfBoundsException!
            } catch (ArrayIndexOutOfBoundsException e) {
                System.out.println("Inner catch: Array index out of bounds!");
            }
            
            // This will still throw exception
            int result = 10 / 0;
            
        } catch (ArithmeticException e) {
            System.out.println("Outer catch: Arithmetic error!");
        }
        
        System.out.println("Program continues...\n");
    }
    
    public static void main(String[] args) {
        example1();
        example2();
        example3();
        
        int returnValue = example4();
        System.out.println("Returned value: " + returnValue + "\n");
        
        example5();
        
        // Real-world analogy:
        // - try: Attempting to cross a bridge
        // - catch: Safety net if bridge breaks
        // - finally: Checking your equipment after crossing
        
        // Key points:
        // - try: Code that might throw exception
        // - catch: Handles specific exceptions
        // - finally: Always executes (cleanup code)
        // - Multiple catch blocks: Most specific first
        // - finally executes even with return statement
        
        // Best practices:
        // - Catch specific exceptions, not generic Exception
        // - Use finally for cleanup (close files, connections)
        // - Don't swallow exceptions - at least log them
        // - Don't use exceptions for flow control
    }
}
Explanation: try contains code that might throw exceptions. catch handles specific exceptions. finally always executes, even if exception occurs or return is called. Use finally for cleanup (closing files, connections). Catch specific exceptions for better error handling.

Program 12: throw and throws Keywords

Demonstrates throwing exceptions explicitly and declaring exceptions.

‚òï Java 
/**
 * ThrowThrowsDemo - Understanding throw and throws
 * 
 * Baby-level: throw is like actually throwing a ball,
 * throws is like warning "I might throw a ball, watch out!"
 */

import java.io.*;

class BankAccount {
    private double balance;
    
    public BankAccount(double initialBalance) {
        this.balance = initialBalance;
    }
    
    // THROWS - declares that method might throw exception
    public void withdraw(double amount) throws InsufficientFundsException {
        if (amount > balance) {
            // THROW - actually throws the exception
            throw new InsufficientFundsException(
                "Insufficient funds! Balance: $" + balance + 
                ", Requested: $" + amount
            );
        }
        
        balance -= amount;
        System.out.println("Withdrew: $" + amount + 
                          ". New balance: $" + balance);
    }
    
    public void deposit(double amount) {
        if (amount <= 0) {
            // Throw unchecked exception (no throws needed)
            throw new IllegalArgumentException(
                "Deposit amount must be positive!"
            );
        }
        
        balance += amount;
        System.out.println("Deposited: $" + amount + 
                          ". New balance: $" + balance);
    }
    
    public double getBalance() {
        return balance;
    }
}

// Custom checked exception
class InsufficientFundsException extends Exception {
    public InsufficientFundsException(String message) {
        super(message);
    }
}

class AgeValidator {
    
    // Method that throws multiple exceptions
    public static void validateAge(int age) 
            throws IllegalArgumentException, InvalidAgeException {
        
        if (age < 0) {
            throw new IllegalArgumentException("Age cannot be negative!");
        }
        
        if (age > 150) {
            throw new InvalidAgeException("Age seems invalid: " + age);
        }
        
        System.out.println("Age " + age + " is valid");
    }
}

class InvalidAgeException extends Exception {
    public InvalidAgeException(String message) {
        super(message);
    }
}

public class ThrowThrowsDemo {
    
    public static void main(String[] args) {
        // Example 1: Using throw and throws
        System.out.println("=== Example 1: Bank Account ===");
        BankAccount account = new BankAccount(1000);
        
        try {
            account.deposit(500);
            account.withdraw(800);
            account.withdraw(1000);  // Will throw exception
        } catch (InsufficientFundsException e) {
            System.out.println("Caught: " + e.getMessage());
        }
        
        // Example 2: Unchecked exception (no try-catch required)
        System.out.println("\n=== Example 2: Invalid Deposit ===");
        try {
            account.deposit(-100);  // Will throw IllegalArgumentException
        } catch (IllegalArgumentException e) {
            System.out.println("Caught: " + e.getMessage());
        }
        
        // Example 3: Multiple exceptions
        System.out.println("\n=== Example 3: Age Validation ===");
        try {
            AgeValidator.validateAge(25);
            AgeValidator.validateAge(-5);
        } catch (IllegalArgumentException e) {
            System.out.println("Caught IllegalArgument: " + e.getMessage());
        } catch (InvalidAgeException e) {
            System.out.println("Caught InvalidAge: " + e.getMessage());
        }
        
        try {
            AgeValidator.validateAge(200);
        } catch (Exception e) {
            System.out.println("Caught: " + e.getMessage());
        }
        
        // Real-world analogy:
        // - throw: Fire alarm going off (actual alert)
        // - throws: "Warning: Fire alarm may sound" (declaration)
        
        // Key differences:
        // THROW:
        // - Used inside method body
        // - Actually throws an exception
        // - Followed by exception object
        // - Example: throw new Exception();
        
        // THROWS:
        // - Used in method signature
        // - Declares potential exceptions
        // - Followed by exception class names
        // - Example: void method() throws IOException
        
        // Checked vs Unchecked:
        // - Checked (extends Exception): Must declare with throws
        // - Unchecked (extends RuntimeException): Optional to declare
        
        // Best practices:
        // - Use custom exceptions for domain-specific errors
        // - Provide meaningful error messages
        // - Document exceptions in JavaDoc
        // - Don't catch exceptions you can't handle
    }
}
Explanation: throw actually throws an exception (used inside method). throws declares that a method might throw exceptions (used in method signature). Checked exceptions must be declared with throws. Use custom exceptions for domain-specific errors with meaningful messages.
