Program 1: Checking if File Exists

Demonstrates how to check if a file exists before performing operations.

â˜• Java
import java.io.File;

public class FileExistsCheck {
    public static void main(String[] args) {
        File file = new File("data.txt");
        
        if (file.exists()) {
            System.out.println("File exists!");
            System.out.println("Is file: " + file.isFile());
            System.out.println("Is directory: " + file.isDirectory());
            System.out.println("Size: " + file.length() + " bytes");
        } else {
            System.out.println("File does not exist.");
        }
    }
}

Baby: Like checking if your toy is in the toy box before playing with it.

Real-world: Checking if a document exists in a filing cabinet before trying to read it.

Technical: Always check file existence before reading to avoid FileNotFoundException. The File class provides metadata methods without actually opening the file.

public class AppendToFile {

Baby: Like adding a new page to your storybook instead of erasing the old pages.

Technical: The second parameter (true) in FileWriter constructor enables append mode. Without it, the file is overwritten. Useful for logs and incremental data.

Program 5: Reading File using FileReader

Demonstrates basic file reading character by character.

â˜• Java
import java.io.FileReader;
import java.io.IOException;

public class ReadFile {
    public static void main(String[] args) {
        try {
            FileReader reader = new FileReader("output.txt");
            int character;
            
            while ((character = reader.read()) != -1) {
                System.out.print((char) character);
            }

            System.out.println("An error occurred.");
            e.printStackTrace();
        }
    }
}

Baby: Like reading a book letter by letter.

Real-world: Opening a text file and reading its contents.

Technical: FileReader reads one character at a time. Returns -1 when end of file is reached. Inefficient for large files - use BufferedReader instead.

Program 6: Reading using BufferedReader (Recommended)

import java.io.BufferedReader;
import java.io.FileReader;
import java.io.IOException;

public class BufferedReaderExample {
    public static void main(String[] args) {
        try {
            BufferedReader reader = new BufferedReader(new FileReader("output.txt"));

Baby: Like reading a book line by line instead of letter by letter.

Real-world: Reading a document paragraph by paragraph.

Technical: BufferedReader is much faster than FileReader for large files. It reads data in chunks (buffer) and provides readLine() method. Always prefer this for text files.

Program 9: Using try-with-resources (Best Practice)

Demonstrates automatic resource management with try-with-resources.

â˜• Java
import java.io.BufferedReader;
import java.io.FileReader;
import java.io.IOException;

public class TryWithResourcesExample {
    public static void main(String[] args) {
        // Resources are automatically closed!
        try (BufferedReader reader = new BufferedReader(new FileReader("output.txt"))) {
            String line;
            while ((line = reader.readLine()) != null) {
                System.out.println(line);
            }
        } catch (IOException e) {
            System.out.println("An error occurred.");
            e.printStackTrace();
        }
        // No need to call reader.close() - it's automatic!
    }
}

Baby: Like a toy that puts itself back in the box when you're done.

Real-world: Automatic doors that close themselves.

Technical: try-with-resources (Java 7+) automatically closes resources that implement AutoCloseable. Prevents resource leaks even if exceptions occur. Always use this for file operations!

Program 10: Using NIO Files.readAllLines (Modern Approach)

Demonstrates reading entire file with one method call using NIO.

â˜• Java
import java.nio.file.Files;
import java.nio.file.Paths;
import java.nio.charset.StandardCharsets;
import java.io.IOException;
import java.util.List;

public class NIOReadFile {
    public static void main(String[] args) {
        try {
            // Read all lines in one shot!
            List<String> lines = Files.readAllLines(
                Paths.get("output.txt"), 
                StandardCharsets.UTF_8
            );
            
            for (String line : lines) {
                System.out.println(line);
            }
            
            System.out.println("\nTotal lines: " + lines.size());
        } catch (IOException e) {
            System.out.println("An error occurred.");
            e.printStackTrace();
        }
    }
}

Baby: Like grabbing all your toys at once instead of one by one.

Real-world: Downloading an entire document at once.

Technical: NIO Files.readAllLines() reads entire file into a List. Very convenient but loads entire file into memory. Not suitable for very large files (GB+). Use BufferedReader for huge files.

Program 14: Listing Files in a Directory

Demonstrates how to list all files and folders in a directory.

â˜• Java
import java.io.File;

public class ListFiles {
    public static void main(String[] args) {
        File directory = new File(".");
        
        if (directory.isDirectory()) {
            String[] files = directory.list();
            
            System.out.println("Files and folders in current directory:");
            for (String fileName : files) {
                File file = new File(directory, fileName);
                String type = file.isDirectory() ? "[DIR]" : "[FILE]";
                System.out.println(type + " " + fileName);
            }
        } else {
            System.out.println("Not a directory.");
        }
    }
}

Baby: Like looking at all the toys in your toy box.

Real-world: Opening a folder to see what files are inside.

Technical: list() returns array of file/directory names. listFiles() returns array of File objects (more useful). "." represents current directory. Can filter results using FilenameFilter.

Program 15: Real-World Example - Writing Log File

A practical application combining file operations for logging.

â˜• Java
import java.io.BufferedWriter;
import java.io.FileWriter;
import java.io.IOException;
import java.time.LocalDateTime;
import java.time.format.DateTimeFormatter;

public class LogFileWriter {
    private static final String LOG_FILE = "application.log";
    
    public static void main(String[] args) {
        log("INFO", "Application started");
        log("DEBUG", "Processing user request");
        log("WARNING", "Low memory detected");
        log("ERROR", "Failed to connect to database");
        log("INFO", "Application shutdown");
        
        System.out.println("Logs written to " + LOG_FILE);
    }
    
    public static void log(String level, String message) {
        try (BufferedWriter writer = new BufferedWriter(
                new FileWriter(LOG_FILE, true))) { // Append mode
            
            String timestamp = LocalDateTime.now().format(
                DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm:ss")
            );
            
            String logEntry = String.format("[%s] [%s] %s%n", 
                timestamp, level, message);
            
            writer.write(logEntry);
            
        } catch (IOException e) {
            System.err.println("Failed to write log: " + e.getMessage());
        }
    }
}

Baby: Like keeping a diary of everything that happens.

Real-world: Recording all events in an application for debugging and monitoring.

Technical: Combines BufferedWriter (performance), append mode (incremental logs), try-with-resources (safety), and timestamps. Real applications use logging frameworks (Log4j, SLF4J), but this demonstrates the underlying concept.

Program 16: Writing and Reading with NIO (Complete Example)

Demonstrates complete NIO workflow for writing and reading files.

â˜• Java
import java.nio.file.*;
import java.nio.charset.StandardCharsets;
import java.io.IOException;
import java.util.Arrays;
import java.util.List;

public class NIOCompleteExample {
    public static void main(String[] args) {
        Path filePath = Paths.get("nio_example.txt");
        
        try {
            // Write data
            List<String> lines = Arrays.asList(
                "Java NIO is powerful!",
                "It simplifies file operations.",
                "Use it for modern applications."
            );
            
            Files.write(filePath, lines, StandardCharsets.UTF_8);
            System.out.println("File written successfully!");
            
            // Read data
            List<String> readLines = Files.readAllLines(filePath, StandardCharsets.UTF_8);
            System.out.println("\nFile contents:");
            readLines.forEach(System.out::println);
            
            // File metadata
            System.out.println("\nFile size: " + Files.size(filePath) + " bytes");
            System.out.println("Is readable: " + Files.isReadable(filePath));
            System.out.println("Is writable: " + Files.isWritable(filePath));
            
        } catch (IOException e) {
            System.out.println("An error occurred.");
            e.printStackTrace();
        }
    }
}

Baby: Like using a magic wand to write and read from your notebook instantly.

Real-world: Modern document management system.

Technical: NIO provides clean, concise APIs. Files.write() and Files.readAllLines() handle all the complexity. Always specify charset (UTF-8) for consistency. Perfect for small to medium files.
