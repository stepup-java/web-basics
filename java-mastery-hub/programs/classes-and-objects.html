Program 1: Basic Class and Object Creation

Demonstrates the simplest form of class and object creation.

â˜• Java 
/**
 * BasicClassDemo - Understanding basic class and object creation
 * 
 * Baby-level: Like creating a recipe (class) and then making actual cookies (objects)!
 */
class Dog {
    // Fields - what a dog has
    String name;
    String breed;
    int age;
    
    // Method - what a dog can do
    void bark() {
        System.out.println(name + " says: Woof! Woof!");
    }
    
    void displayInfo() {
        System.out.println("Name: " + name + ", Breed: " + breed + ", Age: " + age);
    }
}

public class BasicClassDemo {
    public static void main(String[] args) {
        // Creating first dog object
        Dog dog1 = new Dog();
        dog1.name = "Buddy";
        dog1.breed = "Golden Retriever";
        dog1.age = 3;
        
        // Creating second dog object
        Dog dog2 = new Dog();
        dog2.name = "Max";
        dog2.breed = "German Shepherd";
        dog2.age = 5;
        
        // Using the objects
        dog1.displayInfo();
        dog1.bark();
        
        dog2.displayInfo();
        dog2.bark();
        
        // Real-world analogy: Each dog is a separate pet with its own name and breed
        // Industry usage: Objects represent real entities in your application
    }
}
Explanation: This demonstrates the fundamental concept: a class is a blueprint, objects are instances. Each object has its own copy of fields but shares the same methods.

Program 2: Using Fields and Methods

Shows how to work with object fields and call methods.

â˜• Java 
/**
 * FieldsMethodsDemo - Working with fields and methods
 * 
 * Baby-level: Fields are like characteristics (name, color), methods are like actions (run, jump)
 */
class Car {
    // Fields
    String brand;
    String model;
    int year;
    String color;
    int speed;
    
    // Method to accelerate
    void accelerate(int increment) {
        speed += increment;
        System.out.println(brand + " accelerating... Current speed: " + speed + " km/h");
    }
    
    // Method to brake
    void brake(int decrement) {
        speed -= decrement;
        if (speed < 0) speed = 0;
        System.out.println(brand + " braking... Current speed: " + speed + " km/h");
    }
    
    // Method to display car info
    void displayInfo() {
        System.out.println("\n=== Car Information ===");
        System.out.println("Brand: " + brand);
        System.out.println("Model: " + model);
        System.out.println("Year: " + year);
        System.out.println("Color: " + color);
        System.out.println("Current Speed: " + speed + " km/h");
    }
}

public class FieldsMethodsDemo {
    public static void main(String[] args) {
        // Create car object
        Car myCar = new Car();
        
        // Set fields
        myCar.brand = "Honda";
        myCar.model = "Civic";
        myCar.year = 2022;
        myCar.color = "Red";
        myCar.speed = 0;
        
        // Display initial info
        myCar.displayInfo();
        
        // Call methods
        myCar.accelerate(30);
        myCar.accelerate(20);
        myCar.brake(10);
        
        // Display final info
        myCar.displayInfo();
    }
}
Explanation: Fields store object state, methods define behavior. Methods can access and modify the object's fields to perform operations.

Program 5: Constructor Overloading

Demonstrates multiple constructors with different parameters.

â˜• Java
/**
 * ConstructorOverloadingDemo
 * Baby-level: Like ordering food - you can order just a burger, burger with fries, or full meal!
 */
class Product {
    String name;
    double price;
    int quantity;
    
    // Constructor 1: No parameters
    Product() {
        this.name = "Unknown";
        this.price = 0.0;
        this.quantity = 0;
    }
    
    // Constructor 2: Name and price
    Product(String name, double price) {
        this.name = name;
        this.price = price;
        this.quantity = 1;  // Default quantity
    }
    
    // Constructor 3: All parameters
    Product(String name, double price, int quantity) {
        this.name = name;
        this.price = price;
        this.quantity = quantity;
    }
    
    void displayInfo() {
        System.out.println(name + " - $" + price + " x " + quantity + " = $" + (price * quantity));
    }
}

public class ConstructorOverloadingDemo {
    public static void main(String[] args) {
        Product p1 = new Product();
        Product p2 = new Product("Laptop", 999.99);
        Product p3 = new Product("Mouse", 25.50, 3);
        
        p1.displayInfo();
        p2.displayInfo();
        p3.displayInfo();
    }
}
Explanation: Constructor overloading provides flexibility - users can create objects with different amounts of information. Java chooses the right constructor based on the arguments provided.

Program 6: Using 'this' Keyword

Demonstrates all uses of the 'this' keyword.

â˜• Java
/**
 * ThisKeywordDemo
 * Baby-level: 'this' is like pointing to yourself saying "ME!" or "MY OWN!"
 */
class Employee {
    String name;
    int id;
    double salary;
    
    // Use 1: Distinguish instance variables from parameters
    Employee(String name, int id, double salary) {
        this.name = name;      // this.name = instance variable
        this.id = id;          // name = parameter
        this.salary = salary;
    }
    
    // Use 2: Return current object for method chaining
    Employee setSalary(double salary) {
        this.salary = salary;
        return this;  // Returns current object
    }
    
    Employee setName(String name) {
        this.name = name;
        return this;
    }
    
    void displayInfo() {
        System.out.println("ID: " + this.id + ", Name: " + this.name + ", Salary: $" + this.salary);
    }
}

public class ThisKeywordDemo {
    public static void main(String[] args) {
        Employee emp = new Employee("John", 1001, 50000);
        emp.displayInfo();
        
        // Method chaining using 'this'
        emp.setName("John Doe").setSalary(55000);
        emp.displayInfo();
    }
}
Explanation: 'this' refers to the current object. It's essential for distinguishing instance variables from parameters and enables method chaining for fluent APIs.

Program 7: Multiple Objects Demo

Creating and managing multiple independent objects.

â˜• Java
/**
 * MultipleObjectsDemo
 * Baby-level: Like having multiple toy cars - each is separate and independent!
 */
class BankAccount {
    String accountNumber;
    String holderName;
    double balance;
    
    BankAccount(String accountNumber, String holderName, double initialBalance) {
        this.accountNumber = accountNumber;
        this.holderName = holderName;
        this.balance = initialBalance;
    }
    
    void deposit(double amount) {
        balance += amount;
        System.out.println(holderName + " deposited $" + amount + ". New balance: $" + balance);
    }
    
    void withdraw(double amount) {
        if (balance >= amount) {
            balance -= amount;
            System.out.println(holderName + " withdrew $" + amount + ". New balance: $" + balance);
        } else {
            System.out.println("Insufficient funds for " + holderName);
        }
    }
}

public class MultipleObjectsDemo {
    public static void main(String[] args) {
        // Create multiple independent bank accounts
        BankAccount acc1 = new BankAccount("ACC001", "Alice", 1000);
        BankAccount acc2 = new BankAccount("ACC002", "Bob", 500);
        BankAccount acc3 = new BankAccount("ACC003", "Charlie", 2000);
        
        // Each account operates independently
        acc1.deposit(500);
        acc2.withdraw(200);
        acc3.deposit(1000);
        
        acc1.withdraw(300);
        acc2.deposit(100);
    }
}
Explanation: Each object is completely independent with its own memory space. Modifying one object doesn't affect others, even though they're from the same class.

Program 8: Real-World Example - Car Management

Practical example of a Car class with complete functionality.

â˜• Java
/**
 * CarManagementDemo
 * Real-world: Car dealership management system
 */
class Car {
    private String brand;
    private String model;
    private int year;
    private double price;
    private boolean sold;
    
    Car(String brand, String model, int year, double price) {
        this.brand = brand;
        this.model = model;
        this.year = year;
        this.price = price;
        this.sold = false;
    }
    
    void displayDetails() {
        System.out.println("\n" + year + " " + brand + " " + model);
        System.out.println("Price: $" + price);
        System.out.println("Status: " + (sold ? "SOLD" : "AVAILABLE"));
    }
    
    void sellCar() {
        if (!sold) {
            sold = true;
            System.out.println(brand + " " + model + " has been sold!");
        } else {
            System.out.println("This car is already sold.");
        }
    }
    
    void applyDiscount(double percentage) {
        if (!sold) {
            double discount = price * (percentage / 100);
            price -= discount;
            System.out.println("Discount applied! New price: $" + price);
        }
    }
}

public class CarManagementDemo {
    public static void main(String[] args) {
        Car car1 = new Car("Honda", "Civic", 2022, 25000);
        Car car2 = new Car("Toyota", "Camry", 2023, 28000);
        Car car3 = new Car("Ford", "Mustang", 2021, 35000);
        
        car1.displayDetails();
        car1.applyDiscount(10);
        car1.sellCar();
        car1.displayDetails();
        
        car2.displayDetails();
        car3.displayDetails();
    }
}
Explanation: This real-world example shows how classes model actual business entities. Private fields ensure data protection, and methods provide controlled access to object state.

Program 10: Memory Reference Demo

Understanding object references and memory behavior.

â˜• Java
/**
 * MemoryReferenceDemo
 * Baby-level: References are like TV remotes - multiple remotes can control the same TV!
 */
class Box {
    int value;
    
    Box(int value) {
        this.value = value;
    }
    
    void display() {
        System.out.println("Box value: " + value);
    }
}

public class MemoryReferenceDemo {
    public static void main(String[] args) {
        // Create one object
        Box box1 = new Box(10);
        box1.display();
        
        // box2 references the SAME object as box1
        Box box2 = box1;
        box2.display();
        
        // Modify through box2
        box2.value = 20;
        
        // Both show the change (same object!)
        System.out.println("\nAfter modifying through box2:");
        box1.display();  // Shows 20!
        box2.display();  // Shows 20!
        
        // Create a NEW object
        Box box3 = new Box(30);
        box3.display();
        
        // Now box1 and box3 are different objects
        box1.value = 100;
        System.out.println("\nAfter modifying box1:");
        box1.display();  // 100
        box3.display();  // Still 30 (different object)
        
        // Real-world: Understanding references prevents bugs!
    }
}
Explanation: Reference variables store memory addresses, not actual objects. Multiple references can point to the same object, so changes through one reference affect all others pointing to that object.

Program 11: Null Object Reference

Understanding null references and NullPointerException.

â˜• Java
/**
 * NullReferenceDemo
 * Baby-level: null is like having a TV remote with no batteries - it exists but doesn't work!
 */
class Person {
    String name;
    int age;
    
    Person(String name, int age) {
        this.name = name;
        this.age = age;
    }
    
    void greet() {
        System.out.println("Hello, I'm " + name);
    }
}

public class NullReferenceDemo {
    public static void main(String[] args) {
        // Valid object
        Person person1 = new Person("Alice", 25);
        person1.greet();  // Works fine
        
        // Null reference
        Person person2 = null;
        System.out.println("person2 is: " + person2);
        
        // Attempting to use null reference causes NullPointerException
        // person2.greet();  // âŒ This would crash!
        
        // Safe way: Check for null before using
        if (person2 != null) {
            person2.greet();
        } else {
            System.out.println("person2 is null, cannot call methods!");
        }
        
        // Assigning null makes object eligible for garbage collection
        person1 = null;
        System.out.println("person1 is now null");
        
        // Real-world: Always check for null to prevent crashes!
    }
}
Explanation: null means "no object". Attempting to call methods on null references causes NullPointerException. Always check for null before using references, especially when receiving objects from other methods.

Program 12: Garbage Collection Concept

Understanding when objects become eligible for garbage collection.

â˜• Java
/**
 * GarbageCollectionDemo
 * Baby-level: GC is like a cleanup crew that removes abandoned toys (objects) from the playground (memory)
 */
class Resource {
    String name;
    
    Resource(String name) {
        this.name = name;
        System.out.println("Resource created: " + name);
    }
    
    // Called before garbage collection (deprecated but shown for concept)
    protected void finalize() {
        System.out.println("Resource being garbage collected: " + name);
    }
}

public class GarbageCollectionDemo {
    public static void main(String[] args) {
        // Scenario 1: Object becomes unreachable when reference is set to null
        Resource r1 = new Resource("R1");
        r1 = null;  // R1 object is now eligible for GC
        
        // Scenario 2: Object becomes unreachable when reference is reassigned
        Resource r2 = new Resource("R2");
        r2 = new Resource("R3");  // R2 object is now eligible for GC
        
        // Scenario 3: Anonymous object (no reference)
        new Resource("R4");  // Immediately eligible for GC
        
        // Scenario 4: Reference goes out of scope
        {
            Resource r5 = new Resource("R5");
        }  // r5 goes out of scope, R5 object eligible for GC
        
        // Suggest garbage collection (not guaranteed to run immediately)
        System.out.println("\nSuggesting garbage collection...");
        System.gc();
        
        // Give GC time to run
        try {
            Thread.sleep(1000);
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
        
        System.out.println("\nProgram ending...");
        
        // Real-world: Java automatically manages memory, but understanding GC helps write efficient code
    }
}
Explanation: Objects become eligible for garbage collection when no references point to them. Java's GC automatically frees memory, preventing memory leaks. You can suggest GC with System.gc() but cannot force it.

Program 14: Rectangle Class - Complete OOP Example

Demonstrates encapsulation, validation, and computed properties.

â˜• Java
/**
 * RectangleDemo
 * Real-world: Geometric calculations with proper encapsulation
 */
class Rectangle {
    private double length;
    private double width;
    
    // Constructor with validation
    Rectangle(double length, double width) {
        if (length > 0 && width > 0) {
            this.length = length;
            this.width = width;
        } else {
            System.out.println("Invalid dimensions! Using default 1x1");
            this.length = 1.0;
            this.width = 1.0;
        }
    }
    
    // Getters
    double getLength() {
        return length;
    }
    
    double getWidth() {
        return width;
    }
    
    // Setters with validation
    void setLength(double length) {
        if (length > 0) {
            this.length = length;
        } else {
            System.out.println("Length must be positive!");
        }
    }
    
    void setWidth(double width) {
        if (width > 0) {
            this.width = width;
        } else {
            System.out.println("Width must be positive!");
        }
    }
    
    // Computed properties
    double calculateArea() {
        return length * width;
    }
    
    double calculatePerimeter() {
        return 2 * (length + width);
    }
    
    boolean isSquare() {
        return length == width;
    }
    
    void displayInfo() {
        System.out.println("\nRectangle: " + length + " x " + width);
        System.out.println("Area: " + calculateArea());
        System.out.println("Perimeter: " + calculatePerimeter());
        System.out.println("Is Square: " + (isSquare() ? "Yes" : "No"));
    }
}

public class RectangleDemo {
    public static void main(String[] args) {
        Rectangle rect1 = new Rectangle(5, 3);
        Rectangle rect2 = new Rectangle(4, 4);
        Rectangle rect3 = new Rectangle(-2, 5);  // Invalid, will use default
        
        rect1.displayInfo();
        rect2.displayInfo();
        rect3.displayInfo();
        
        // Modify rectangle
        System.out.println("\n--- Modifying rect1 ---");
        rect1.setLength(10);
        rect1.setWidth(6);
        rect1.displayInfo();
        
        // Try invalid modification
        rect1.setLength(-5);  // Will be rejected
    }
}
Explanation: This example demonstrates encapsulation (private fields with getters/setters), validation (preventing invalid data), and computed properties (area, perimeter calculated on demand). This is how professional Java code is structured.
