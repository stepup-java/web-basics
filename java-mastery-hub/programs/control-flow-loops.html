Hands-On Java Programs

Welcome to the practical section! These 12+ programs demonstrate every control flow statement and loop type. Each program is heavily commented to explain not just WHAT the code does, but WHY and HOW each statement works.

Program 1: Simple if Statement

Demonstrates basic if statement for conditional execution.

‚òï Java 
/**
 * SimpleIfDemo - Understanding basic if statement
 * 
 * Baby-level: "If it's raining, take an umbrella"
 */
public class SimpleIfDemo {
    
    public static void main(String[] args) {
        int age = 20;
        
        // Simple if - executes ONLY if condition is true
        if (age >= 18) {
            System.out.println("You are an adult!");
            System.out.println("You can vote!");
        }
        
        System.out.println("Program continues...");
        
        // Example 2: Checking even/odd
        int number = 10;
        if (number % 2 == 0) {
            System.out.println(number + " is even");
        }
        
        // Example 3: Multiple conditions with AND
        int score = 85;
        if (score >= 80 && score <= 100) {
            System.out.println("Excellent score!");
        }
        
        // Real-world analogy:
        // - if: Like a bouncer at a club - "If you're 21+, you can enter"
        // - If condition is false, the code inside is simply skipped
        
        // Common mistake: Using = instead of ==
        // if (age = 18) { }  ‚ùå This assigns 18 to age!
        // if (age == 18) { } ‚úÖ This compares age with 18
    }
}
Explanation: The if statement checks a boolean condition. If true, the code block executes; if false, it's skipped entirely. Use && for AND (both must be true) and || for OR (at least one must be true). Always use == for comparison, not = (assignment).

Program 2: if-else Statement

Demonstrates if-else for binary decisions.

‚òï Java 
/**
 * IfElseDemo - Understanding if-else decisions
 * 
 * Baby-level: "If it's sunny, go to park. Otherwise, stay home"
 */
public class IfElseDemo {
    
    public static void main(String[] args) {
        // Example 1: Pass/Fail
        int marks = 45;
        
        if (marks >= 60) {
            System.out.println("Result: PASS ‚úÖ");
            System.out.println("Congratulations!");
        } else {
            System.out.println("Result: FAIL ‚ùå");
            System.out.println("Better luck next time!");
        }
        
        // Example 2: Even/Odd
        int num = 7;
        if (num % 2 == 0) {
            System.out.println(num + " is EVEN");
        } else {
            System.out.println(num + " is ODD");
        }
        
        // Example 3: Positive/Negative
        int value = -5;
        if (value >= 0) {
            System.out.println(value + " is positive or zero");
        } else {
            System.out.println(value + " is negative");
        }
        
        // Real-world analogy:
        // - Like a fork in the road: you MUST take one path
        // - if path (condition true) OR else path (condition false)
        // - Exactly ONE block will execute, never both
        
        // Industry usage:
        // - User authentication: if (password correct) login else show error
        // - File operations: if (file exists) read else create
        // - API responses: if (success) process data else handle error
    }
}
Explanation: if-else provides two paths: one for true, one for false. Exactly ONE block executes - never both, never neither. Use for binary decisions like pass/fail, yes/no, valid/invalid. The else block is optional but recommended for clarity.

Program 3: if-else-if Ladder (Grade Calculator)

Demonstrates multiple condition checking with if-else-if ladder.

‚òï Java 
/**
 * GradeCalculator - if-else-if ladder demonstration
 * 
 * Baby-level: "If score is 90+, grade A. If 80+, grade B. If 70+, grade C..."
 */
public class GradeCalculator {
    
    public static void main(String[] args) {
        int score = 85;
        String grade;
        
        // if-else-if ladder - checks conditions in order
        // First TRUE condition executes, rest are skipped
        if (score >= 90) {
            grade = "A";
            System.out.println("Outstanding! üåü");
        } else if (score >= 80) {
            grade = "B";
            System.out.println("Excellent! üëè");
        } else if (score >= 70) {
            grade = "C";
            System.out.println("Good! üëç");
        } else if (score >= 60) {
            grade = "D";
            System.out.println("Pass üìù");
        } else {
            grade = "F";
            System.out.println("Fail ‚ùå");
        }
        
        System.out.println("Your grade: " + grade);
        
        // Example 2: Temperature classification
        int temp = 25;
        if (temp > 35) {
            System.out.println("Extremely Hot üî•");
        } else if (temp > 25) {
            System.out.println("Hot ‚òÄÔ∏è");
        } else if (temp > 15) {
            System.out.println("Warm üå§Ô∏è");
        } else if (temp > 5) {
            System.out.println("Cold ‚ùÑÔ∏è");
        } else {
            System.out.println("Freezing üßä");
        }
        
        // Real-world analogy:
        // - Like a decision tree: check first condition, if false, check next
        // - Similar to multiple-choice: first matching answer wins
        
        // IMPORTANT: Order matters!
        // if (score >= 60) { grade = "D"; }
        // else if (score >= 90) { grade = "A"; }  ‚Üê This will NEVER execute!
        // Because if score is 90, it's also >= 60, so first condition is true
        
        // Best practice: Order from most specific to least specific
    }
}
Explanation: The if-else-if ladder checks multiple conditions sequentially. The FIRST true condition executes, and all subsequent conditions are skipped. Order matters - always check from most specific to least specific. Use for range-based decisions like grading, categorization, or tiered pricing.

Program 4: switch-case Statement (with break)

Demonstrates switch-case for multiple constant value checks.

‚òï Java 
/**
 * SwitchCaseDemo - Understanding switch-case with break
 * 
 * Baby-level: "Pick a number 1-7, I'll tell you the day of the week"
 */
public class SwitchCaseDemo {
    
    public static void main(String[] args) {
        // Example 1: Day of week
        int day = 3;
        String dayName;
        
        switch (day) {
            case 1:
                dayName = "Monday";
                break;  // Exit switch, don't check other cases
            case 2:
                dayName = "Tuesday";
                break;
            case 3:
                dayName = "Wednesday";
                break;
            case 4:
                dayName = "Thursday";
                break;
            case 5:
                dayName = "Friday";
                break;
            case 6:
                dayName = "Saturday";
                break;
            case 7:
                dayName = "Sunday";
                break;
            default:
                dayName = "Invalid day";
        }
        
        System.out.println("Day " + day + " is " + dayName);
        
        // Example 2: Menu system
        char choice = 'B';
        
        switch (choice) {
            case 'A':
                System.out.println("You selected: Add");
                break;
            case 'B':
                System.out.println("You selected: Browse");
                break;
            case 'C':
                System.out.println("You selected: Create");
                break;
            case 'D':
                System.out.println("You selected: Delete");
                break;
            default:
                System.out.println("Invalid choice");
        }
        
        // Example 3: HTTP status codes
        int statusCode = 404;
        
        switch (statusCode) {
            case 200:
                System.out.println("OK - Success");
                break;
            case 404:
                System.out.println("Not Found");
                break;
            case 500:
                System.out.println("Internal Server Error");
                break;
            default:
                System.out.println("Unknown status code");
        }
        
        // Real-world analogy:
        // - Like a vending machine: press button A for chips, B for soda, etc.
        // - Each case is a specific button
        
        // When to use switch vs if-else:
        // ‚úÖ switch: Checking exact constant values (1, 2, 3 or 'A', 'B', 'C')
        // ‚úÖ if-else: Range checks (age > 18), boolean conditions, complex logic
    }
}
Explanation: switch-case is efficient for checking a variable against multiple constant values. Each case represents a specific value. The break statement exits the switch; without it, execution "falls through" to the next case. default handles values that don't match any case.

Program 5: switch-case Fall-through (Without break)

Demonstrates intentional fall-through behavior in switch-case.

‚òï Java 
/**
 * SwitchFallThroughDemo - Understanding fall-through behavior
 * 
 * Baby-level: "Without break, it keeps going to the next case!"
 */
public class SwitchFallThroughDemo {
    
    public static void main(String[] args) {
        // Example 1: Intentional fall-through for grouping
        int month = 2;
        int days;
        
        switch (month) {
            case 1:   // January
            case 3:   // March
            case 5:   // May
            case 7:   // July
            case 8:   // August
            case 10:  // October
            case 12:  // December
                days = 31;
                break;
            case 4:   // April
            case 6:   // June
            case 9:   // September
            case 11:  // November
                days = 30;
                break;
            case 2:   // February
                days = 28;  // Ignoring leap year for simplicity
                break;
            default:
                days = 0;
        }
        
        System.out.println("Month " + month + " has " + days + " days");
        
        // Example 2: Demonstrating accidental fall-through (BUG!)
        int score = 1;
        
        System.out.println("Without break (fall-through):");
        switch (score) {
            case 1:
                System.out.println("Bronze medal ü•â");
                // Missing break! Falls through to next case
            case 2:
                System.out.println("Silver medal ü•à");
                // Missing break! Falls through to next case
            case 3:
                System.out.println("Gold medal ü•á");
                break;
            default:
                System.out.println("No medal");
        }
        // Output: Bronze, Silver, AND Gold! (all three print!)
        
        // Correct version with break:
        System.out.println("\nWith break (correct):");
        switch (score) {
            case 1:
                System.out.println("Bronze medal ü•â");
                break;  // Exit switch
            case 2:
                System.out.println("Silver medal ü•à");
                break;
            case 3:
                System.out.println("Gold medal ü•á");
                break;
            default:
                System.out.println("No medal");
        }
        // Output: Only Bronze (correct!)
        
        // Real-world analogy:
        // - Fall-through is like a waterfall: once you start, you keep falling
        // - break is like a platform: stops the fall
        
        // When fall-through is useful:
        // ‚úÖ Grouping multiple cases with same logic (like months with 31 days)
        // ‚ùå Usually it's a bug if unintentional!
    }
}
Explanation: Without break, execution continues to the next case (fall-through). This is useful for grouping cases with the same logic but is usually a bug if unintentional. Always use break unless you specifically want fall-through behavior. Comment your code if fall-through is intentional.

Program 6: Standard for Loop

Demonstrates the classic for loop with initialization, condition, and update.

‚òï Java 
/**
 * ForLoopDemo - Understanding for loops
 * 
 * Baby-level: "Count from 1 to 10" - you know exactly how many times
 */
public class ForLoopDemo {
    
    public static void main(String[] args) {
        // Example 1: Print 1 to 5
        System.out.println("Counting 1 to 5:");
        for (int i = 1; i <= 5; i++) {
            System.out.println(i);
        }
        
        // Breakdown:
        // 1. int i = 1     ‚Üí Initialize: Start at 1
        // 2. i <= 5        ‚Üí Condition: Continue while i is 5 or less
        // 3. i++           ‚Üí Update: Increase i by 1 after each iteration
        
        // Example 2: Print even numbers 0-10
        System.out.println("\nEven numbers 0-10:");
        for (int i = 0; i <= 10; i += 2) {
            System.out.println(i);
        }
        
        // Example 3: Countdown
        System.out.println("\nCountdown:");
        for (int i = 5; i >= 1; i--) {
            System.out.println(i);
        }
        System.out.println("Blast off! üöÄ");
        
        // Example 4: Calculate sum of 1 to 10
        int sum = 0;
        for (int i = 1; i <= 10; i++) {
            sum += i;  // sum = sum + i
        }
        System.out.println("\nSum of 1-10: " + sum);  // 55
        
        // Example 5: Print multiplication table
        int number = 5;
        System.out.println("\nMultiplication table of " + number + ":");
        for (int i = 1; i <= 10; i++) {
            System.out.println(number + " x " + i + " = " + (number * i));
        }
        
        // Example 6: Nested for loop (pattern printing)
        System.out.println("\nPattern:");
        for (int i = 1; i <= 5; i++) {
            for (int j = 1; j <= i; j++) {
                System.out.print("* ");
            }
            System.out.println();
        }
        
        // Real-world analogy:
        // - Like climbing stairs: you know there are 10 steps
        // - Count each step as you climb
        
        // Common mistakes:
        // ‚ùå for (int i = 0; i < 10; i++);  ‚Üê Semicolon creates empty loop!
        // ‚ùå for (int i = 1; i <= 10; i--) ‚Üê Infinite loop! i never reaches 10
        // ‚ùå Forgetting to increment: for (int i = 0; i < 10; ) ‚Üê Infinite loop!
    }
}
Explanation: The for loop is perfect when you know the exact number of iterations. It has three parts: initialization (runs once), condition (checked before each iteration), and update (runs after each iteration). Use for counting, iterating arrays, and repeating actions a fixed number of times.

Program 7: while Loop

Demonstrates while loop for condition-based iteration.

‚òï Java 
/**
 * WhileLoopDemo - Understanding while loops
 * 
 * Baby-level: "Keep doing something WHILE a condition is true"
 */
public class WhileLoopDemo {
    
    public static void main(String[] args) {
        // Example 1: Print 1 to 5
        System.out.println("Counting 1 to 5:");
        int i = 1;
        while (i <= 5) {
            System.out.println(i);
            i++;  // CRITICAL: Must update i, or infinite loop!
        }
        
        // Example 2: Sum until user enters 0 (simulated)
        int[] inputs = {5, 10, 15, 20, 0};  // Simulating user input
        int index = 0;
        int sum = 0;
        int num = inputs[index++];
        
        while (num != 0) {
            sum += num;
            num = inputs[index++];
        }
        System.out.println("\nSum: " + sum);  // 50
        
        // Example 3: Find first power of 2 greater than 100
        int power = 1;
        while (power <= 100) {
            power *= 2;  // power = power * 2
        }
        System.out.println("\nFirst power of 2 > 100: " + power);  // 128
        
        // Example 4: Reverse a number
        int number = 12345;
        int reversed = 0;
        
        while (number > 0) {
            int digit = number % 10;      // Get last digit
            reversed = reversed * 10 + digit;  // Add to reversed
            number = number / 10;         // Remove last digit
        }
        System.out.println("\nReversed: " + reversed);  // 54321
        
        // Example 5: Login attempts (max 3)
        String correctPassword = "java123";
        String[] attempts = {"wrong1", "wrong2", "java123"};  // Simulated
        int attemptCount = 0;
        int attemptIndex = 0;
        boolean loggedIn = false;
        
        while (attemptCount < 3 && !loggedIn) {
            String password = attempts[attemptIndex++];
            attemptCount++;
            
            if (password.equals(correctPassword)) {
                loggedIn = true;
                System.out.println("\nLogin successful! ‚úÖ");
            } else {
                System.out.println("Wrong password. Attempts left: " + (3 - attemptCount));
            }
        }
        
        if (!loggedIn) {
            System.out.println("Account locked! ‚ùå");
        }
        
        // Real-world analogy:
        // - Like filling a bucket: keep adding water WHILE it's not full
        // - You don't know exactly how many cups it takes
        
        // while vs for:
        // ‚úÖ while: Unknown iteration count (user input, file reading)
        // ‚úÖ for: Known iteration count (arrays, counting)
    }
}
Explanation: while loops check the condition BEFORE each iteration. If initially false, the loop never executes. Best for unknown iteration counts like user input validation, file reading, or searching. ALWAYS update the condition variable inside the loop to avoid infinite loops.

Program 8: do-while Loop

Demonstrates do-while loop that guarantees at least one execution.

‚òï Java 
/**
 * DoWhileLoopDemo - Understanding do-while loops
 * 
 * Baby-level: "Do something first, THEN check if you should do it again"
 */
public class DoWhileLoopDemo {
    
    public static void main(String[] args) {
        // Example 1: Print 1 to 5
        System.out.println("Counting 1 to 5:");
        int i = 1;
        do {
            System.out.println(i);
            i++;
        } while (i <= 5);
        
        // Example 2: Menu system (always shows menu at least once)
        int choice = 0;
        int menuIteration = 0;
        int[] choices = {1, 2, 3};  // Simulated user choices
        
        do {
            System.out.println("\n=== Menu ===");
            System.out.println("1. Play Game");
            System.out.println("2. View Scores");
            System.out.println("3. Exit");
            
            // Simulate user choice
            if (menuIteration < choices.length) {
                choice = choices[menuIteration++];
                System.out.println("You selected: " + choice);
            }
            
            switch (choice) {
                case 1:
                    System.out.println("Starting game...");
                    break;
                case 2:
                    System.out.println("Displaying scores...");
                    break;
                case 3:
                    System.out.println("Goodbye!");
                    break;
            }
        } while (choice != 3);
        
        // Example 3: Input validation
        System.out.println("\n=== Age Validation ===");
        int[] ageInputs = {-5, 150, 25};  // Simulated inputs
        int ageIndex = 0;
        int age;
        
        do {
            age = ageInputs[ageIndex++];
            System.out.println("Entered age: " + age);
            
            if (age < 1 || age > 120) {
                System.out.println("Invalid! Age must be 1-120");
            }
        } while (age < 1 || age > 120);
        
        System.out.println("Valid age: " + age);
        
        // Example 4: Demonstrating difference between while and do-while
        System.out.println("\n=== while vs do-while ===");
        
        // while loop - may not execute at all
        int x = 10;
        System.out.println("while loop (x=10, condition x<5):");
        while (x < 5) {
            System.out.println("This never prints!");
            x++;
        }
        System.out.println("while loop executed 0 times");
        
        // do-while loop - always executes at least once
        int y = 10;
        System.out.println("\ndo-while loop (y=10, condition y<5):");
        do {
            System.out.println("This prints once! y = " + y);
            y++;
        } while (y < 5);
        System.out.println("do-while loop executed 1 time");
        
        // Real-world analogy:
        // - while: "Check if door is open, then enter"
        // - do-while: "Enter first, then check if you should stay"
        
        // Use cases for do-while:
        // ‚úÖ Menus (must show at least once)
        // ‚úÖ Input validation (must ask at least once)
        // ‚úÖ "Try at least once" scenarios
    }
}
Explanation: do-while executes the body FIRST, then checks the condition. This guarantees AT LEAST ONE execution, even if the condition is initially false. Perfect for menus, input validation, and any scenario where you must execute at least once. The semicolon after while(condition) is required.

Program 9: Enhanced for Loop (for-each)

Demonstrates the simplified for-each loop for arrays and collections.

‚òï Java 
/**
 * EnhancedForLoopDemo - Understanding for-each loops
 * 
 * Baby-level: "For EACH item in the box, do something"
 */
public class EnhancedForLoopDemo {
    
    public static void main(String[] args) {
        // Example 1: Print array elements
        int[] numbers = {10, 20, 30, 40, 50};
        
        System.out.println("Using enhanced for loop:");
        for (int num : numbers) {
            System.out.println(num);
        }
        
        // Equivalent regular for loop:
        System.out.println("\nUsing regular for loop:");
        for (int i = 0; i < numbers.length; i++) {
            System.out.println(numbers[i]);
        }
        
        // Example 2: Sum of array
        int sum = 0;
        for (int num : numbers) {
            sum += num;
        }
        System.out.println("\nSum: " + sum);  // 150
        
        // Example 3: String array
        String[] fruits = {"Apple", "Banana", "Cherry", "Date"};
        
        System.out.println("\nFruits:");
        for (String fruit : fruits) {
            System.out.println("- " + fruit);
        }
        
        // Example 4: Find maximum
        int[] scores = {85, 92, 78, 95, 88};
        int max = scores[0];
        
        for (int score : scores) {
            if (score > max) {
                max = score;
            }
        }
        System.out.println("\nHighest score: " + max);  // 95
        
        // Example 5: Count specific elements
        int[] grades = {65, 75, 85, 55, 95, 45, 70};
        int passCount = 0;
        
        for (int grade : grades) {
            if (grade >= 60) {
                passCount++;
            }
        }
        System.out.println("\nStudents passed: " + passCount);
        
        // Example 6: 2D array with enhanced for loop
        int[][] matrix = {
            {1, 2, 3},
            {4, 5, 6},
            {7, 8, 9}
        };
        
        System.out.println("\nMatrix:");
        for (int[] row : matrix) {
            for (int element : row) {
                System.out.print(element + " ");
            }
            System.out.println();
        }
        
        // Real-world analogy:
        // - Like checking each item in your shopping cart
        // - You don't care about position, just the item itself
        
        // When to use enhanced for loop:
        // ‚úÖ Reading all elements
        // ‚úÖ Don't need index
        // ‚úÖ Not modifying array
        
        // When NOT to use:
        // ‚ùå Need index position
        // ‚ùå Modifying array elements
        // ‚ùå Iterating in reverse
        // ‚ùå Skipping elements
        
        // Limitation: Cannot modify array elements
        int[] nums = {1, 2, 3};
        for (int n : nums) {
            n = n * 2;  // This modifies the COPY, not the array!
        }
        System.out.println("\nArray unchanged: " + nums[0]);  // Still 1
    }
}
Explanation: The enhanced for loop (for-each) simplifies iteration over arrays and collections. It automatically handles indexing and is more readable. However, you cannot access the index, modify array elements, or iterate in reverse. Use when you need to process every element without modification.

Program 10: break Statement in Loops

Demonstrates using break to exit loops early.

‚òï Java 
/**
 * BreakStatementDemo - Understanding break in loops
 * 
 * Baby-level: "Stop the loop immediately and get out!"
 */
public class BreakStatementDemo {
    
    public static void main(String[] args) {
        // Example 1: Find first even number
        System.out.println("Finding first even number:");
        for (int i = 1; i <= 10; i++) {
            if (i % 2 == 0) {
                System.out.println("First even: " + i);
                break;  // Exit loop immediately
            }
        }
        // Loop stops at i=2, doesn't check 3,4,5...
        
        // Example 2: Search for a value
        int[] numbers = {10, 25, 30, 45, 50};
        int target = 30;
        boolean found = false;
        
        System.out.println("\nSearching for " + target + ":");
        for (int i = 0; i < numbers.length; i++) {
            System.out.println("Checking index " + i + ": " + numbers[i]);
            if (numbers[i] == target) {
                System.out.println("Found at index " + i + "!");
                found = true;
                break;  // No need to check remaining elements
            }
        }
        
        if (!found) {
            System.out.println("Not found");
        }
        
        // Example 3: Password attempts with break
        String correctPassword = "secret123";
        String[] attempts = {"wrong1", "wrong2", "secret123", "wrong3"};
        
        System.out.println("\n=== Login System ===");
        for (int i = 0; i < attempts.length; i++) {
            System.out.println("Attempt " + (i + 1) + ": " + attempts[i]);
            
            if (attempts[i].equals(correctPassword)) {
                System.out.println("Login successful! ‚úÖ");
                break;  // Stop trying, we're in!
            } else {
                System.out.println("Wrong password");
            }
        }
        
        // Example 4: break in nested loops (only exits inner loop)
        System.out.println("\n=== Nested Loop with break ===");
        for (int i = 1; i <= 3; i++) {
            System.out.println("Outer loop: i = " + i);
            
            for (int j = 1; j <= 5; j++) {
                if (j == 3) {
                    System.out.println("  Breaking inner loop at j = " + j);
                    break;  // Only exits inner loop
                }
                System.out.println("  Inner loop: j = " + j);
            }
        }
        
        // Example 5: Labeled break (exits outer loop)
        System.out.println("\n=== Labeled break ===");
        outerLoop:  // Label for outer loop
        for (int i = 1; i <= 3; i++) {
            for (int j = 1; j <= 3; j++) {
                System.out.println("i = " + i + ", j = " + j);
                if (i == 2 && j == 2) {
                    System.out.println("Breaking outer loop!");
                    break outerLoop;  // Exits BOTH loops
                }
            }
        }
        System.out.println("After labeled break");
        
        // Real-world analogy:
        // - Like finding your keys: once found, stop searching!
        // - Emergency exit: get out immediately
        
        // Use cases:
        // ‚úÖ Searching (stop when found)
        // ‚úÖ Validation (stop on first error)
        // ‚úÖ Early termination (condition met)
    }
}
Explanation: break immediately exits the current loop. In nested loops, it only exits the innermost loop unless you use a labeled break. Use break for early termination when a condition is met, like finding a value or detecting an error. Execution continues after the loop.

Program 11: continue Statement in Loops

Demonstrates using continue to skip iterations.

‚òï Java 
/**
 * ContinueStatementDemo - Understanding continue in loops
 * 
 * Baby-level: "Skip this one and move to the next!"
 */
public class ContinueStatementDemo {
    
    public static void main(String[] args) {
        // Example 1: Print only odd numbers
        System.out.println("Odd numbers 1-10:");
        for (int i = 1; i <= 10; i++) {
            if (i % 2 == 0) {
                continue;  // Skip even numbers
            }
            System.out.println(i);
        }
        // When i=2,4,6,8,10, continue skips the print
        
        // Example 2: Skip negative numbers
        int[] numbers = {5, -3, 8, -1, 12, -7, 15};
        int sum = 0;
        
        System.out.println("\nProcessing numbers (skipping negatives):");
        for (int num : numbers) {
            if (num < 0) {
                System.out.println("Skipping " + num);
                continue;  // Skip negative numbers
            }
            sum += num;
            System.out.println("Adding " + num + ", sum = " + sum);
        }
        System.out.println("Final sum: " + sum);  // 40
        
        // Example 3: Skip multiples of 3
        System.out.println("\nNumbers 1-15 (skip multiples of 3):");
        for (int i = 1; i <= 15; i++) {
            if (i % 3 == 0) {
                continue;  // Skip 3, 6, 9, 12, 15
            }
            System.out.print(i + " ");
        }
        System.out.println();
        
        // Example 4: Process only valid data
        String[] data = {"apple", "", "banana", null, "cherry", ""};
        
        System.out.println("\nValid items:");
        for (String item : data) {
            if (item == null || item.isEmpty()) {
                continue;  // Skip null or empty strings
            }
            System.out.println("- " + item);
        }
        
        // Example 5: continue in nested loops
        System.out.println("\n=== Nested loop with continue ===");
        for (int i = 1; i <= 3; i++) {
            System.out.println("Outer: i = " + i);
            
            for (int j = 1; j <= 5; j++) {
                if (j == 3) {
                    System.out.println("  Skipping j = " + j);
                    continue;  // Skip j=3, move to j=4
                }
                System.out.println("  Inner: j = " + j);
            }
        }
        
        // Example 6: break vs continue comparison
        System.out.println("\n=== break vs continue ===");
        
        System.out.println("With break:");
        for (int i = 1; i <= 5; i++) {
            if (i == 3) {
                break;  // Exit loop entirely
            }
            System.out.print(i + " ");
        }
        System.out.println("(stopped at 3)");
        
        System.out.println("\nWith continue:");
        for (int i = 1; i <= 5; i++) {
            if (i == 3) {
                continue;  // Skip only i=3
            }
            System.out.print(i + " ");
        }
        System.out.println("(skipped 3, continued)");
        
        // Real-world analogy:
        // - break: "I'm leaving the party" (exit completely)
        // - continue: "I'll skip this song and wait for the next" (skip one)
        
        // Use cases:
        // ‚úÖ Filtering data (skip invalid entries)
        // ‚úÖ Conditional processing (skip certain conditions)
        // ‚úÖ Optimization (skip unnecessary work)
    }
}
Explanation: continue skips the rest of the current iteration and moves to the next one. The loop continues running, unlike break which exits entirely. Use continue for filtering, skipping invalid data, or conditional processing. It's like saying "skip this one, but keep going."

Program 12: Real-World Example - ATM System

Comprehensive example combining all control flow concepts.

‚òï Java 
/**
 * ATMSystem - Real-world example using all control flow statements
 * 
 * Demonstrates: if-else, switch-case, while, do-while, break, continue
 */
public class ATMSystem {
    
    public static void main(String[] args) {
        // Account setup
        double balance = 1000.0;
        String correctPin = "1234";
        int maxAttempts = 3;
        
        // Simulated user inputs
        String[] pinAttempts = {"wrong", "1234"};
        int[] menuChoices = {1, 2, 3, 4};
        double[] withdrawAmounts = {200.0, 1500.0};
        double[] depositAmounts = {500.0};
        
        int pinIndex = 0;
        int menuIndex = 0;
        int withdrawIndex = 0;
        int depositIndex = 0;
        
        System.out.println("=== Welcome to Java Bank ATM ===\n");
        
        // Step 1: PIN Verification (while loop with break)
        boolean authenticated = false;
        int attempts = 0;
        
        while (attempts < maxAttempts && !authenticated) {
            String enteredPin = pinAttempts[pinIndex++];
            attempts++;
            
            System.out.println("Attempt " + attempts + ": Enter PIN");
            System.out.println("Entered: " + enteredPin);
            
            if (enteredPin.equals(correctPin)) {
                authenticated = true;
                System.out.println("‚úÖ PIN Correct! Access granted.\n");
                break;  // Exit authentication loop
            } else {
                System.out.println("‚ùå Wrong PIN!");
                if (attempts < maxAttempts) {
                    System.out.println("Attempts remaining: " + (maxAttempts - attempts) + "\n");
                }
            }
        }
        
        // Check if authentication failed
        if (!authenticated) {
            System.out.println("üîí Account locked! Too many failed attempts.");
            return;  // Exit program
        }
        
        // Step 2: Main Menu (do-while loop)
        int choice;
        do {
            System.out.println("=== Main Menu ===");
            System.out.println("1. Check Balance");
            System.out.println("2. Withdraw Money");
            System.out.println("3. Deposit Money");
            System.out.println("4. Exit");
            
            choice = menuChoices[menuIndex++];
            System.out.println("Selected: " + choice + "\n");
            
            // Step 3: Process choice (switch-case)
            switch (choice) {
                case 1:
                    // Check Balance
                    System.out.println("üí∞ Current Balance: $" + balance);
                    System.out.println();
                    break;
                
                case 2:
                    // Withdraw Money
                    if (withdrawIndex < withdrawAmounts.length) {
                        double amount = withdrawAmounts[withdrawIndex++];
                        System.out.println("Withdraw amount: $" + amount);
                        
                        // Validation (if-else)
                        if (amount <= 0) {
                            System.out.println("‚ùå Invalid amount!");
                        } else if (amount > balance) {
                            System.out.println("‚ùå Insufficient funds!");
                            System.out.println("Available: $" + balance);
                        } else {
                            balance -= amount;
                            System.out.println("‚úÖ Withdrawal successful!");
                            System.out.println("New balance: $" + balance);
                        }
                    }
                    System.out.println();
                    break;
                
                case 3:
                    // Deposit Money
                    if (depositIndex < depositAmounts.length) {
                        double amount = depositAmounts[depositIndex++];
                        System.out.println("Deposit amount: $" + amount);
                        
                        if (amount <= 0) {
                            System.out.println("‚ùå Invalid amount!");
                        } else {
                            balance += amount;
                            System.out.println("‚úÖ Deposit successful!");
                            System.out.println("New balance: $" + balance);
                        }
                    }
                    System.out.println();
                    break;
                
                case 4:
                    // Exit
                    System.out.println("Thank you for using Java Bank!");
                    System.out.println("Final balance: $" + balance);
                    System.out.println("Goodbye! üëã");
                    break;
                
                default:
                    System.out.println("‚ùå Invalid choice! Please try again.\n");
            }
            
        } while (choice != 4);  // Continue until user exits
        
        // Program ends
        System.out.println("\n=== ATM Session Ended ===");
        
        // This example demonstrates:
        // ‚úÖ while loop: PIN verification with limited attempts
        // ‚úÖ if-else: Input validation and balance checks
        // ‚úÖ do-while: Menu system (show at least once)
        // ‚úÖ switch-case: Menu option processing
        // ‚úÖ break: Exit loops early
        // ‚úÖ return: Exit program on authentication failure
        
        // Real-world lessons:
        // 1. Always validate user input
        // 2. Limit retry attempts for security
        // 3. Provide clear feedback
        // 4. Handle edge cases (negative amounts, insufficient funds)
        // 5. Use appropriate loop types for each scenario
    }
}
Explanation: This ATM system demonstrates real-world usage of all control flow statements. It uses while for PIN verification with limited attempts, do-while for the menu (must show at least once), switch-case for menu processing, if-else for validation, break for early exit, and return to terminate on authentication failure. This is how professional applications combine control flow for robust functionality.
