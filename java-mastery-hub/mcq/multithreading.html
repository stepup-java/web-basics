Q1. Which package contains the Thread class and Runnable interface?

âœ“ Correct Answer: java.lang

The `Thread` class and `Runnable` interface are part of the `java.lang` package, which is automatically imported.

Q2. Which method is used to begin the execution of a thread?

âœ“ Correct Answer: start()

The `start()` method is used to begin the execution of a thread. It calls the `run()` method internally in a separate call stack.

Q3. Which interface should be implemented to create a thread?

âœ“ Correct Answer: Runnable

The `Runnable` interface is the primary interface to implement for creating a thread. `Callable` is also used but returns a result.

Q4. What is the state of a thread after it is created but before the start() method is called?

âœ“ Correct Answer: NEW

A thread is in the `NEW` state when it is created but not yet started.

Q5. Which keyword is used to prevent multiple threads from accessing a block of code simultaneously?

âœ“ Correct Answer: synchronized

The `synchronized` keyword is used to acquire a lock on an object or class, ensuring that only one thread can execute the synchronized block or method at a time.

Q6. Which method causes the current thread to release the lock and wait until another thread invokes notify()?

âœ“ Correct Answer: wait()

The `wait()` method causes the current thread to release the lock and wait until another thread invokes `notify()` or `notifyAll()` on the same object.

Q7. In which class are the wait(), notify(), and notifyAll() methods defined?

âœ“ Correct Answer: Object

These methods are defined in the `Object` class because they are related to the monitor lock associated with every object.

Q8. Which method pauses the execution of the current thread for a specified time without releasing the lock?

âœ“ Correct Answer: sleep()

`Thread.sleep()` pauses the current thread for a specified time but does not release any locks it holds.

Q9. What is the condition called when two or more threads are waiting for each other to release a lock forever?

âœ“ Correct Answer: Deadlock

Deadlock occurs when two or more threads are blocked forever, each waiting for the other to release a resource.

Q10. Which keyword guarantees that the value of a variable is always read from the main memory?

âœ“ Correct Answer: volatile

The `volatile` keyword ensures visibility of changes to variables across threads by guaranteeing reads and writes go directly to main memory.

Q11. Which method allows one thread to wait for the completion of another thread?

âœ“ Correct Answer: join()

The `join()` method allows the calling thread to wait until the thread on which `join()` is called completes its execution.

Q12. Which interface in the java.util.concurrent package represents an asynchronous execution mechanism?

âœ“ Correct Answer: ExecutorService

`ExecutorService` is a higher-level replacement for working directly with threads. It manages a pool of threads and provides methods for submitting tasks.

Q13. Which interface is similar to Runnable but can return a result and throw a checked exception?

âœ“ Correct Answer: Callable

The `Callable` interface is similar to `Runnable`, but its `call()` method can return a value and throw a checked exception.

Q14. Which interface represents the result of an asynchronous computation?

âœ“ Correct Answer: Future

A `Future` represents the result of an asynchronous computation. Methods are provided to check if the computation is complete, to wait for its completion, and to retrieve the result.

Q15. What occurs when multiple threads access shared data and try to change it at the same time?

âœ“ Correct Answer: Race Condition

A race condition occurs when the correctness of a program depends on the relative timing or interleaving of multiple threads.

Q16. Which class provides thread-local variables?

âœ“ Correct Answer: ThreadLocal

`ThreadLocal` provides thread-local variables. Each thread accessing such a variable has its own, independently initialized copy of the variable.

Q17. Which method hints to the scheduler that the current thread is willing to yield its current use of a processor?

âœ“ Correct Answer: yield()

`Thread.yield()` is a hint to the scheduler that the current thread is willing to yield its current use of a processor. The scheduler is free to ignore this hint.

Q18. What type of thread runs in the background and does not prevent the JVM from exiting?

âœ“ Correct Answer: Daemon Thread

Daemon threads are low-priority threads that run in the background to perform tasks such as garbage collection. The JVM exits when the only threads running are all daemon threads.

Q19. Which class provides atomic operations on an int value without using synchronization?

âœ“ Correct Answer: AtomicInteger

`AtomicInteger` (in `java.util.concurrent.atomic`) provides atomic operations on an `int` value, allowing thread-safe updates without explicit synchronization.

Q20. Which class implements the Lock interface and provides more flexible locking operations than synchronized blocks?

âœ“ Correct Answer: ReentrantLock

`ReentrantLock` implements the `Lock` interface and provides extended capabilities compared to the implicit monitor lock accessed using `synchronized` methods and statements.

Q21. Which synchronization aid allows one or more threads to wait until a set of operations being performed in other threads completes?

âœ“ Correct Answer: CountDownLatch

`CountDownLatch` is a synchronization aid that allows one or more threads to wait until a set of operations being performed in other threads completes.

Q22. Which synchronization aid allows a set of threads to all wait for each other to reach a common barrier point?

âœ“ Correct Answer: CyclicBarrier

`CyclicBarrier` is a synchronization aid that allows a set of threads to all wait for each other to reach a common barrier point. It is useful for parallel computations.

Q23. Which class maintains a set of permits and is often used to restrict the number of threads than can access some (physical or logical) resource?

âœ“ Correct Answer: Semaphore

A `Semaphore` maintains a set of permits. It is useful for restricting the number of threads that can access a resource.

Q24. Which ExecutorService implementation is designed for work that can be broken down into smaller pieces recursively?

âœ“ Correct Answer: ForkJoinPool

`ForkJoinPool` is an `ExecutorService` for running `ForkJoinTask`s. It employs a work-stealing algorithm and is ideal for recursive tasks.

Q25. Which exception is thrown when a thread is waiting, sleeping, or otherwise occupied, and the thread is interrupted?

âœ“ Correct Answer: InterruptedException

`InterruptedException` is thrown when a thread is waiting, sleeping, or otherwise occupied, and the thread is interrupted, either before or during the activity.

Q26. Which utility class provides factory methods for creating thread pools?

âœ“ Correct Answer: Executors

The `Executors` class provides factory and utility methods for `Executor`, `ExecutorService`, `ScheduledExecutorService`, `ThreadFactory`, and `Callable`.

Q27. Which interface represents a queue that additionally supports operations that wait for the queue to become non-empty when retrieving an element, and wait for space to become available in the queue when storing an element?

âœ“ Correct Answer: BlockingQueue

`BlockingQueue` implementations are thread-safe and are commonly used in producer-consumer scenarios.

Q28. What occurs when a thread is unable to gain regular access to shared resources and is unable to make progress?

âœ“ Correct Answer: Starvation

Starvation describes a situation where a thread is unable to gain regular access to shared resources and is unable to make progress. This happens when shared resources are made unavailable for long periods by "greedy" threads.

Q29. What occurs when threads are not blocked but are too busy responding to each other to resume work?

âœ“ Correct Answer: Livelock

A livelock is similar to a deadlock, except that the states of the processes involved in the livelock constantly change with regard to one another, none progressing.

Q30. Which class introduced in Java 8 implements Future and CompletionStage, allowing for composable asynchronous programming?

âœ“ Correct Answer: CompletableFuture

`CompletableFuture` is a powerful class for asynchronous programming, allowing you to chain tasks, combine results, and handle errors.

Q31. Which thread-safe List implementation is efficient when reads vastly outnumber writes?

âœ“ Correct Answer: CopyOnWriteArrayList

`CopyOnWriteArrayList` is a thread-safe variant of `ArrayList` in which all mutative operations (add, set, and so on) are implemented by making a fresh copy of the underlying array.

Q32. Which Map implementation allows concurrent access and updates without locking the entire map?

âœ“ Correct Answer: ConcurrentHashMap

`ConcurrentHashMap` allows concurrent read and thread-safe update operations. It uses a more sophisticated locking strategy (bucket-level locking or CAS) than `Hashtable`.

Q33. Which method is used to interrupt a thread?

âœ“ Correct Answer: interrupt()

The `interrupt()` method is used to interrupt a thread. If the thread is blocked in an invocation of `wait()`, `sleep()`, or `join()`, its interrupted status will be cleared and it will receive an `InterruptedException`.

Q34. Which method checks if a thread is still running?

âœ“ Correct Answer: isAlive()

The `isAlive()` method returns true if the thread has been started and has not yet died.

Q35. How do you get a reference to the currently executing thread object?

âœ“ Correct Answer: Thread.currentThread()

`Thread.currentThread()` is a static method that returns a reference to the currently executing thread object.
