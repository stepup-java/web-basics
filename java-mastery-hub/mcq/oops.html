Q1. Which OOP principle involves wrapping code and data together into a single unit?

âœ“ Correct Answer: Encapsulation

Encapsulation is the mechanism of wrapping the data (variables) and code acting on the data (methods) together as a single unit (class). It also involves restricting direct access to some of an object's components.

Q2. Which keyword is used to inherit a class in Java?

âœ“ Correct Answer: extends

The `extends` keyword is used to create a subclass that inherits attributes and methods from a superclass. Example: `class Dog extends Animal`.

Q3. "One interface, multiple methods" describes which OOP concept?

âœ“ Correct Answer: Polymorphism

Polymorphism allows objects to be treated as instances of their parent class. It allows a single interface to map to different underlying forms (methods), such as through method overloading or overriding.

Q4. What is Abstraction?

âœ“ Correct Answer: Hiding implementation details and showing only functionality

Abstraction is the process of hiding the implementation details and showing only the functionality to the user. In Java, this is achieved using abstract classes and interfaces.

Q5. Which type of inheritance is NOT supported by Java classes?

âœ“ Correct Answer: Multiple inheritance of classes

Java does not support multiple inheritance with classes (a class cannot extend more than one class) to avoid the "Diamond Problem". However, it supports multiple inheritance through interfaces.

Q6. Which keyword is used to refer to the immediate parent class object?

âœ“ Correct Answer: super

The `super` keyword is a reference variable used to refer to the immediate parent class object. It can be used to invoke parent class methods or constructors.

Q7. Providing a specific implementation of a method already defined in the parent class is called?

âœ“ Correct Answer: Method Overriding

Method Overriding occurs when a subclass provides a specific implementation for a method that is already defined in its superclass. The method signature must be exactly the same.

Q8. Method overriding is an example of which type of polymorphism?

âœ“ Correct Answer: Runtime Polymorphism

Method overriding is resolved at runtime (Dynamic Method Dispatch), making it an example of Runtime Polymorphism. Method overloading is Compile-time polymorphism.

Q9. Which of the following is true about an abstract class?

âœ“ Correct Answer: Cannot be instantiated

An abstract class cannot be instantiated directly using the `new` keyword. It is meant to be subclassed. It can have constructors, static methods, and both abstract and concrete methods.

Q10. Which keyword is used to implement an interface?

âœ“ Correct Answer: implements

A class uses the `implements` keyword to implement an interface. A class can implement multiple interfaces.

Q11. Variables declared in an interface are implicitly...?

âœ“ Correct Answer: public static final

All variables declared in an interface are implicitly `public`, `static`, and `final` (constants), even if you don't explicitly specify these modifiers.

Q12. Inheritance represents which type of relationship?

âœ“ Correct Answer: IS-A relationship

Inheritance represents an IS-A relationship (e.g., A Car IS-A Vehicle). Composition/Aggregation represents a HAS-A relationship (e.g., A Car HAS-A Engine).

Q13. Which operator is used to check if an object is an instance of a specific class?

âœ“ Correct Answer: instanceof

The `instanceof` operator is used to test whether the object is an instance of the specified type (class or subclass or interface).

Q14. Which access modifier is most restrictive?

âœ“ Correct Answer: private

`private` is the most restrictive access modifier. Members declared as private are accessible only within the same class.

Q15. Where can a protected member be accessed?

âœ“ Correct Answer: Same package and subclasses

`protected` members are accessible within the same package and by subclasses in other packages.

Q16. Which class is the root of the Java class hierarchy?

âœ“ Correct Answer: Object

The `java.lang.Object` class is the root of the class hierarchy. Every class has Object as a superclass.

Q17. How do you call the parent class constructor from a subclass constructor?

âœ“ Correct Answer: super()

`super()` is used to invoke the parent class constructor. It must be the first statement in the subclass constructor.

Q18. What is the relationship between two separate classes which establish communication through their objects?

âœ“ Correct Answer: Association

Association represents a relationship between two classes. It can be one-to-one, one-to-many, etc. Aggregation and Composition are specialized forms of Association.

Q19. What is Composition?

âœ“ Correct Answer: Strong Association (Death relationship)

Composition is a strong form of association where the child object cannot exist without the parent object (e.g., A House and a Room). If the House is destroyed, the Room is also destroyed.

Q20. A 'Department' has 'Teachers'. If the Department is closed, Teachers still exist. This is an example of?

âœ“ Correct Answer: Aggregation

This is Aggregation (Weak Association). The child object (Teacher) can exist independently of the parent object (Department).

Q21. Are constructors inherited in Java?

âœ“ Correct Answer: No, constructors are not inherited

Constructors are not inherited by subclasses. However, the subclass constructor invokes the superclass constructor (either implicitly or explicitly using `super()`).

Q22. Casting a subclass object to a superclass reference is called?

âœ“ Correct Answer: Upcasting

Upcasting is casting a subclass object to a superclass reference (e.g., `Animal a = new Dog();`). It is done implicitly and is safe.

Q23. What exception is thrown if an invalid Downcast is attempted?

âœ“ Correct Answer: ClassCastException

A `ClassCastException` is thrown at runtime if you try to cast an object to a subclass of which it is not an instance.

Q24. An interface with no methods is called?

âœ“ Correct Answer: Marker Interface

A Marker Interface (or Tagging Interface) is an interface with no methods or constants (e.g., `Serializable`, `Cloneable`). It provides run-time type information about objects.

Q25. An interface with exactly one abstract method is called?

âœ“ Correct Answer: Functional Interface

A Functional Interface is an interface that contains exactly one abstract method. They are used with Lambda expressions (e.g., `Runnable`, `Callable`).

Q26. Which keyword allows concrete methods in interfaces (Java 8+)?

âœ“ Correct Answer: default

Since Java 8, interfaces can have methods with implementation using the `default` keyword. This allows adding new methods to interfaces without breaking existing implementations.

Q27. The degree of dependency between components is called?

âœ“ Correct Answer: Coupling

Coupling refers to the degree of direct knowledge that one element has of another. Loose coupling is desirable in OOP design.

Q28. The measure of how strongly related the responsibilities of a class are is called?

âœ“ Correct Answer: Cohesion

Cohesion refers to how closely related the functions within a single class are. High cohesion is desirable (Single Responsibility Principle).

Q29. Can static methods be overridden?

âœ“ Correct Answer: No, static methods cannot be overridden

Static methods cannot be overridden because they are bound to the class, not the instance. However, they can be *hidden* (Method Hiding) if a subclass defines a static method with the same signature.

Q30. Which keyword prevents a class from being subclassed?

âœ“ Correct Answer: final

A `final` class cannot be subclassed (e.g., `String`, `System`). This is used to create immutable classes or for security reasons.

Q31. Runtime polymorphism is achieved through?

âœ“ Correct Answer: Dynamic Binding

Runtime polymorphism (method overriding) is achieved through Dynamic Binding (Late Binding), where the method call is resolved at runtime based on the actual object type.

Q32. Which of the following can have abstract methods?

âœ“ Correct Answer: Both Abstract Class and Interface

Both Abstract Classes and Interfaces can contain abstract methods (methods without a body).

Q33. What is the ambiguity problem in multiple inheritance called?

âœ“ Correct Answer: Diamond Problem

The Diamond Problem occurs in multiple inheritance when a class inherits from two classes that have a common ancestor, leading to ambiguity about which method to inherit.

Q34. Can an abstract class be final?

âœ“ Correct Answer: No

An abstract class cannot be `final` because abstract classes are meant to be subclassed, while `final` prevents subclassing. These keywords are mutually exclusive.

Q35. Which type of object cloning creates a new copy of all objects in the original object's hierarchy?

âœ“ Correct Answer: Deep Copy

A Deep Copy creates a new object and recursively copies all objects referenced by the original object, ensuring that changes to the new object do not affect the original one.
