Q1. What is a method in Java?

âœ“ Correct Answer: A block of code that performs a specific task

A method is a block of code that performs a specific task. Methods provide code reusability, improve organization, and make programs easier to understand and maintain. They can accept inputs (parameters) and return outputs.

Q2. What is the correct syntax for declaring a method?

âœ“ Correct Answer: returnType methodName(parameters) { }

The correct method syntax is: accessModifier returnType methodName(parameters) { method body }. The return type comes first, followed by the method name, then parameters in parentheses, and finally the method body in curly braces.

Q3. Which keyword indicates a method doesn't return any value?

âœ“ Correct Answer: void

The void keyword indicates that a method doesn't return any value. Void methods perform actions but don't send back a result. Example: public void printMessage() { }

Q4. Which keyword is used to return a value from a method?

âœ“ Correct Answer: return

The return keyword is used to exit a method and optionally return a value. For non-void methods, you must return a value of the declared return type. Example: return result;

Q5. What are values passed to a method called?

âœ“ Correct Answer: Both arguments and parameters

Parameters are variables in the method declaration, while arguments are the actual values passed when calling the method. Example: void method(int param) - param is parameter. method(5) - 5 is argument. The terms are often used interchangeably.

Q6. What is it called when multiple methods have the same name but different parameters?

âœ“ Correct Answer: Method overloading

Method overloading allows multiple methods with the same name but different parameter lists (different number, type, or order of parameters). This is compile-time polymorphism. The return type alone cannot differentiate overloaded methods.

Q7. What must be different for method overloading?

âœ“ Correct Answer: Number, type, or order of parameters

For method overloading, the parameter list must differ in number, type, or order. The return type and access modifiers can be different but don't count for overloading. Example: add(int a, int b) and add(int a, int b, int c) are overloaded.

Q8. What is recursion?

âœ“ Correct Answer: A method that calls itself

Recursion is when a method calls itself. Every recursive method needs a base case (stopping condition) to prevent infinite recursion. Recursion is useful for problems like factorial calculation, tree traversal, and divide-and-conquer algorithms.

Q9. What error occurs if a recursive method has no base case?

âœ“ Correct Answer: StackOverflowError

Without a base case, recursion continues infinitely, filling up the call stack until a StackOverflowError is thrown. Always ensure recursive methods have a proper base case that will eventually be reached.

Q10. Which keyword allows a method to be called without creating an object?

âœ“ Correct Answer: static

Static methods belong to the class rather than instances. They can be called using the class name without creating an object. Example: Math.max(5, 10). Static methods cannot access instance variables or call instance methods directly.

Q11. What is the entry point method in a Java program?

âœ“ Correct Answer: main

The main method is the entry point of a Java application. Its signature must be: public static void main(String[] args). The JVM looks for this method to start program execution.

Q12. What is the correct signature for the main method?

âœ“ Correct Answer: public static void main(String[] args)

The main method must be public (accessible to JVM), static (callable without object), void (no return value), and accept String array as parameter. This signature is mandatory for the JVM to recognize it as the entry point.

Q13. What allows a method to accept any number of arguments of the same type?

âœ“ Correct Answer: Variable-length arguments (varargs)

Varargs (variable-length arguments) allow a method to accept zero or more arguments of the same type. Syntax: void method(int... numbers). Internally treated as an array. Only one varargs parameter allowed, and it must be last.

Q14. How are primitive types passed to methods in Java?

âœ“ Correct Answer: Pass by value

Primitive types are passed by value, meaning a copy of the value is passed. Changes to the parameter inside the method don't affect the original variable. Java is always pass-by-value, even for objects (the reference value is passed).

Q15. Which keyword prevents a method from being overridden?

âœ“ Correct Answer: final

The final keyword prevents a method from being overridden in subclasses. This ensures the method's implementation remains unchanged throughout the inheritance hierarchy. Private methods are implicitly final.

Q16. Which keyword declares a method without implementation?

âœ“ Correct Answer: abstract

Abstract methods are declared with the abstract keyword and have no body (no implementation). They must be in abstract classes and must be implemented by concrete subclasses. Example: abstract void display();

Q17. Which keyword makes a method thread-safe?

âœ“ Correct Answer: synchronized

The synchronized keyword ensures that only one thread can execute the method at a time. This prevents race conditions in multi-threaded applications. However, it can impact performance due to thread blocking.

Q18. Which keyword indicates a method is implemented in platform-specific code?

âœ“ Correct Answer: native

Native methods are implemented in platform-specific code (usually C/C++) using JNI (Java Native Interface). They're declared with the native keyword and have no body in Java. Example: public native void nativeMethod();

Q19. Which keyword ensures consistent floating-point calculations across platforms?

âœ“ Correct Answer: strictfp

The strictfp (strict floating-point) keyword ensures floating-point calculations produce identical results on all platforms. It can be applied to classes or methods. This is rarely used in modern Java development.

Q20. What consists of the method name and parameter list?

âœ“ Correct Answer: Method signature

A method signature consists of the method name and parameter list (types and order). The return type is NOT part of the signature. Signatures must be unique within a class for overloading to work.

Q21. What are variables declared inside a method called?

âœ“ Correct Answer: Local variables

Local variables are declared inside methods and exist only during method execution. They must be initialized before use (no default values). They're stored on the stack and destroyed when the method completes.

Q22. Do local variables have default values?

âœ“ Correct Answer: No, local variables must be initialized

Local variables do NOT have default values and must be explicitly initialized before use. Attempting to use an uninitialized local variable causes a compilation error. This is different from instance variables which do have defaults.

Q23. What is it called when a method returns 'this' to allow consecutive method calls?

âœ“ Correct Answer: Method chaining

Method chaining allows multiple method calls on the same object in a single statement. Methods return 'this' to enable chaining. Example: obj.method1().method2().method3(). This pattern is common in builder classes and fluent APIs.

Q24. What does the String[] args parameter in main method represent?

âœ“ Correct Answer: Command line arguments

The String[] args parameter contains command-line arguments passed when running the program. Example: java MyProgram arg1 arg2. These arguments are accessible as args[0], args[1], etc. Useful for passing configuration or input data.

Q25. Can a method have multiple return statements?

âœ“ Correct Answer: No, only one return statement executes

A method can have multiple return statements (e.g., in different if-else branches), but only one executes per method call. Once a return statement executes, the method immediately exits. Every path through a non-void method must have a return statement.

Q26. What happens if a non-void method doesn't return a value?

âœ“ Correct Answer: Compilation error

If a non-void method doesn't return a value on all possible execution paths, it causes a compilation error: "missing return statement". Every path through the method must either return a value or throw an exception.

Q27. What are methods used to access and modify private fields called?

âœ“ Correct Answer: Both are correct

Methods to access and modify private fields are called getters/setters or accessors/mutators. Getters retrieve values (getName()), setters modify values (setName()). This supports encapsulation by controlling access to private data.

Q28. What is a best practice for method naming?

âœ“ Correct Answer: Use descriptive names and follow naming conventions

Best practices: use descriptive verb-based names (calculateTotal, printReport), follow camelCase convention, keep methods focused on a single task, limit parameters (ideally 3 or fewer), and write clear documentation.

Q29. What principle states a method should do one thing well?

âœ“ Correct Answer: Single Responsibility Principle

The Single Responsibility Principle (SRP) states that a method should have one clear purpose and do one thing well. This makes code more maintainable, testable, and reusable. Methods should be cohesive and focused.

Q30. What is recommended for method length?

âœ“ Correct Answer: Keep methods short and focused

Keep methods short and focused (ideally under 20-30 lines). Long methods are harder to understand, test, and maintain. If a method is too long, consider breaking it into smaller, well-named helper methods.
