Q1. Which interface is the root interface of the Collection hierarchy?

âœ“ Correct Answer: Collection

The `Collection` interface is the root interface of the Collection hierarchy. `Map` is part of the Collections Framework but does not extend `Collection`.

Q2. Which class provides a dynamic array implementation in Java?

âœ“ Correct Answer: ArrayList

`ArrayList` is a resizable-array implementation of the `List` interface. It grows dynamically as elements are added.

Q3. Which collection does NOT allow duplicate elements?

âœ“ Correct Answer: HashSet

`Set` implementations like `HashSet` do not allow duplicate elements. `List` implementations allow duplicates.

Q4. Which collection stores data as key-value pairs?

âœ“ Correct Answer: HashMap

`Map` implementations like `HashMap` store data in key-value pairs. Keys must be unique.

Q5. Which class implements both List and Deque interfaces?

âœ“ Correct Answer: LinkedList

`LinkedList` implements both `List` and `Deque` interfaces, making it usable as a list, stack, or queue.

Q6. What order does LinkedHashSet maintain?

âœ“ Correct Answer: Insertion Order

`LinkedHashSet` maintains the insertion order of elements. `HashSet` does not guarantee any order, and `TreeSet` maintains sorted order.

Q7. Which collection keeps elements sorted naturally or by a comparator?

âœ“ Correct Answer: TreeSet

`TreeSet` implements `SortedSet` and keeps elements sorted according to their natural ordering or a specified `Comparator`.

Q8. Which of the following is a legacy class and is synchronized?

âœ“ Correct Answer: Vector

`Vector` is a legacy class (since Java 1.0) that is synchronized (thread-safe). `ArrayList` is not synchronized.

Q9. Which interface provides a way to access elements of a collection one by one?

âœ“ Correct Answer: Iterator

`Iterator` is an interface that provides methods to iterate over any Collection. It allows removing elements during iteration.

Q10. What exception is thrown if a collection is modified while iterating over it using an Iterator?

âœ“ Correct Answer: ConcurrentModificationException

Fail-fast iterators throw `ConcurrentModificationException` if the underlying collection is modified structurally during iteration (except through the iterator's own remove method).

Q11. Which interface is used to define the natural ordering of objects?

âœ“ Correct Answer: Comparable

The `Comparable` interface is used to define the natural ordering of a class (e.g., `compareTo` method). `Comparator` is used for custom ordering.

Q12. Which method must be implemented for the Comparator interface?

âœ“ Correct Answer: compare(Object o1, Object o2)

The `Comparator` interface requires implementing the `compare(T o1, T o2)` method. `Comparable` uses `compareTo(T o)`.

Q13. What is the time complexity of get() operation in ArrayList?

âœ“ Correct Answer: O(1)

`ArrayList` provides constant-time `O(1)` access to elements because it is backed by an array and uses index-based access.

Q14. What is the time complexity of get() operation in LinkedList?

âœ“ Correct Answer: O(n)

`LinkedList` requires traversing the list from the beginning (or end) to reach a specific index, so access time is linear `O(n)`.

Q15. Which queue implementation orders elements according to their natural ordering or a comparator?

âœ“ Correct Answer: PriorityQueue

`PriorityQueue` orders elements based on their priority (natural ordering or Comparator), not insertion order. The head is the least element.

Q16. How do you sort a List in Java?

âœ“ Correct Answer: Collections.sort()

`Collections.sort(list)` is the standard way to sort a List. `Arrays.sort()` is for arrays. (Note: Java 8 added `list.sort(comparator)` as well).

Q17. Which Map implementation does NOT allow null keys or values?

âœ“ Correct Answer: Hashtable

`Hashtable` is a legacy class that does not allow `null` keys or values. `HashMap` allows one null key and multiple null values.

Q18. Which methods are used by HashMap to store and retrieve elements?

âœ“ Correct Answer: hashCode() and equals()

`HashMap` uses `hashCode()` to determine the bucket location and `equals()` to check for key equality within the bucket (collision handling).

Q19. Which iterator allows traversing the list in both directions?

âœ“ Correct Answer: ListIterator

`ListIterator` extends `Iterator` and allows bidirectional traversal (`hasPrevious()`, `previous()`) of a List.

Q20. Which class represents a LIFO (Last-In-First-Out) stack of objects?

âœ“ Correct Answer: Stack

`Stack` is a legacy class that represents a LIFO stack. `Deque` (ArrayDeque) is preferred for new code.

Q21. How can you make a Collection read-only?

âœ“ Correct Answer: Collections.unmodifiableList()

`Collections.unmodifiableList()` (and similar methods for Set/Map) returns a read-only view of the collection. Attempts to modify it throw `UnsupportedOperationException`.

Q22. Which of the following is a thread-safe variant of ArrayList?

âœ“ Correct Answer: CopyOnWriteArrayList

`CopyOnWriteArrayList` is a thread-safe variant of `ArrayList` where all mutative operations (add, set, etc.) are implemented by making a fresh copy of the underlying array.

Q23. What is the internal data structure of TreeMap?

âœ“ Correct Answer: Red-Black Tree

`TreeMap` is based on a Red-Black tree implementation, which guarantees `O(log n)` time complexity for `containsKey`, `get`, `put` and `remove` operations.

Q24. How do you get a synchronized version of a List?

âœ“ Correct Answer: Collections.synchronizedList()

`Collections.synchronizedList(list)` returns a synchronized (thread-safe) list backed by the specified list.

Q25. Which interface introduced in Java 8 supports parallel iteration?

âœ“ Correct Answer: Spliterator

`Spliterator` (Splitable Iterator) is designed for traversing and partitioning elements of a source. It supports parallel processing of streams.

Q26. Which interface allows insertion and removal from both ends?

âœ“ Correct Answer: Deque

`Deque` (Double Ended Queue) supports element insertion and removal at both ends. Implementations include `ArrayDeque` and `LinkedList`.

Q27. Which Map implementation allows keys to be garbage collected if they are not used elsewhere?

âœ“ Correct Answer: WeakHashMap

`WeakHashMap` stores keys as `WeakReference`. If a key is no longer strongly referenced, it can be garbage collected, and the entry is removed from the map.

Q28. Which Map uses reference equality (==) instead of object equality (equals) for keys?

âœ“ Correct Answer: IdentityHashMap

`IdentityHashMap` uses reference equality (`==`) to compare keys, whereas normal maps use `equals()`.

Q29. What is the default initial capacity of a HashMap?

âœ“ Correct Answer: 16

The default initial capacity of a `HashMap` is 16, and the default load factor is 0.75.

Q30. What is the default load factor of a HashMap?

âœ“ Correct Answer: 0.75

The default load factor is 0.75, which offers a good trade-off between time and space costs.

Q31. Which class is faster than Stack when used as a stack, and faster than LinkedList when used as a queue?

âœ“ Correct Answer: ArrayDeque

`ArrayDeque` is a resizable-array implementation of the `Deque` interface. It is generally faster than `Stack` and `LinkedList`.

Q32. What happens if you try to add an element to a list returned by Arrays.asList()?

âœ“ Correct Answer: UnsupportedOperationException

`Arrays.asList()` returns a fixed-size list backed by the array. Adding or removing elements throws `UnsupportedOperationException`.

Q33. Which class extends Hashtable and is used to maintain lists of values in which the key is a String and the value is also a String?

âœ“ Correct Answer: Properties

The `Properties` class represents a persistent set of properties. It extends `Hashtable`. Keys and values are Strings.

Q34. Which Queue method retrieves, but does not remove, the head of the queue, returning null if empty?

`peek()` retrieves the head without removing it, returning `null` if empty. `element()` throws an exception if empty.

`poll()` retrieves and removes the head, returning `null` if empty. `remove()` throws an exception if empty.
