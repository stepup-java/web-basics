Q1. What is a blueprint for creating objects in Java?

âœ“ Correct Answer: class

A class is a blueprint or template for creating objects. It defines the structure (fields) and behavior (methods) that objects of that class will have. Think of it like a cookie cutter - the class is the cutter, objects are the cookies.

Q2. What is an instance of a class called?

âœ“ Correct Answer: object

An object is an instance of a class. When you create an object using the 'new' keyword, you're creating a specific instance with its own set of values for the class's fields. Multiple objects can be created from the same class.

Q3. Which keyword is used to create an object?

âœ“ Correct Answer: new

The 'new' keyword is used to create objects in Java. Example: Student s = new Student(); This allocates memory on the heap for the object and calls the constructor to initialize it.

Q4. What is a special method that initializes an object when it's created?

âœ“ Correct Answer: Constructor

A constructor is a special method that has the same name as the class and no return type. It's automatically called when an object is created using 'new'. Constructors are used to initialize object state.

Q5. What is true about a constructor?

âœ“ Correct Answer: It has the same name as the class

A constructor must have the exact same name as the class and has no return type (not even void). Constructors can have parameters (parameterized constructors) and are not static.

Q6. What is a default constructor?

âœ“ Correct Answer: A no-argument constructor provided by Java

If you don't define any constructor, Java automatically provides a default no-argument constructor. However, if you define any constructor (even with parameters), Java will NOT provide the default constructor.

Q7. Which keyword refers to the current object instance?

âœ“ Correct Answer: this

The 'this' keyword is a reference to the current object. It's commonly used to distinguish between instance variables and parameters with the same name, and to call other constructors in the same class.

Q8. Why use 'this' keyword in a constructor?

âœ“ Correct Answer: To differentiate between instance variables and parameters

When parameter names match instance variable names, 'this' is used to refer to the instance variable. Example: this.name = name; Here, this.name is the instance variable, and name is the parameter.

Q9. Which keyword creates a class-level variable shared by all instances?

âœ“ Correct Answer: static

Static variables belong to the class, not to individual objects. All instances share the same static variable. Static members can be accessed using the class name without creating an object.

Q10. What are variables declared inside a class but outside methods called?

âœ“ Correct Answer: Instance variables

Instance variables (also called fields or member variables) are declared inside a class but outside methods. Each object has its own copy of instance variables. They're initialized to default values if not explicitly initialized.

Q11. Can a class have multiple constructors?

âœ“ Correct Answer: Yes, through constructor overloading

A class can have multiple constructors with different parameter lists (constructor overloading). This allows objects to be created in different ways. The constructors must differ in the number or type of parameters.

Q12. How do you call another constructor from within a constructor?

âœ“ Correct Answer: this()

Use this() to call another constructor in the same class. This is called constructor chaining. The this() call must be the first statement in the constructor. Example: this(defaultValue);

Q13. Where are objects stored in memory?

âœ“ Correct Answer: Heap

Objects are stored in the heap memory. The reference variable (which points to the object) is stored on the stack. The heap is managed by the garbage collector, which automatically frees memory when objects are no longer referenced.

Q14. Where are object reference variables stored?

âœ“ Correct Answer: Stack

Reference variables are stored on the stack. They hold the memory address (reference) of the actual object which is stored on the heap. When the method ends, the reference variable is removed from the stack.

Q15. What automatically manages memory and removes unused objects?

âœ“ Correct Answer: Garbage Collector

The Garbage Collector (GC) automatically frees memory by removing objects that are no longer referenced. This prevents memory leaks and eliminates the need for manual memory management like in C/C++.

Q16. What value does an uninitialized object reference have?

âœ“ Correct Answer: null

Uninitialized object references default to null. Attempting to use a null reference (like calling a method on it) throws a NullPointerException. Always check for null before using object references.

Q17. What exception is thrown when you try to use a null object reference?

âœ“ Correct Answer: NullPointerException

NullPointerException (NPE) is thrown when you try to access a member (field or method) of a null object. This is one of the most common runtime exceptions in Java. Always validate objects before use.

Q18. How do you call a static method?

âœ“ Correct Answer: ClassName.staticMethod()

Static methods belong to the class, not to instances. They're called using the class name: ClassName.staticMethod(). While you can call them using an object reference, it's not recommended as it's misleading.

Q19. Can a static method access instance variables?

âœ“ Correct Answer: No, static methods cannot access instance members

Static methods cannot directly access instance variables or call instance methods because they don't have access to 'this' (no specific object context). They can only access static members or create objects to access instance members.

Q20. Can an instance method access static variables?

âœ“ Correct Answer: Yes, instance methods can access both

Instance methods can access both instance variables and static variables. They have access to the object context (this) and can also access class-level (static) members.

Q21. What is a static block?

âœ“ Correct Answer: A block of code that runs when the class is loaded

A static block (static { }) is executed once when the class is first loaded into memory. It's used to initialize static variables or perform one-time setup. Static blocks execute before the constructor.

Q22. When is a static block executed?

âœ“ Correct Answer: When the class is first loaded

Static blocks execute once when the class is loaded into memory by the JVM, before any object is created or any static method is called. This happens only once during the program's lifetime.

Q23. What is an instance initialization block?

âœ“ Correct Answer: Instance block runs before constructor, for each object

An instance initialization block ({ }) runs every time an object is created, just before the constructor. It's useful for common initialization code shared across multiple constructors.

Q24. How are objects passed to methods in Java?

âœ“ Correct Answer: Pass by value (reference value for objects)

Java is always pass-by-value. For objects, the value of the reference (memory address) is passed. This means you can modify the object's state, but you cannot change which object the original reference points to.

Q25. What happens when you assign one object reference to another?

âœ“ Correct Answer: The reference is copied, not the object

When you assign one reference to another (obj2 = obj1), both references point to the same object. Changes through either reference affect the same object. Only the reference is copied, not the actual object data.

Q26. What does the == operator do when comparing objects?

âœ“ Correct Answer: Comparing memory addresses

The == operator compares references (memory addresses), not object content. To compare object content, use the equals() method. Example: obj1.equals(obj2) compares content, obj1 == obj2 compares references.

Q27. Which method should be used to compare object content?

âœ“ Correct Answer: equals()

The equals() method is used to compare object content. You should override equals() in your classes to define what "equal" means for your objects. The default implementation in Object class uses == (reference comparison).

Q28. Which method is called by the garbage collector before destroying an object?

âœ“ Correct Answer: finalize()

The finalize() method is called by the garbage collector before destroying an object. However, it's deprecated since Java 9 and should not be used. Use try-with-resources or explicit cleanup methods instead.

Q29. What is an object created without a reference variable called?

âœ“ Correct Answer: Anonymous object

An anonymous object is created without storing its reference. Example: new Student().display(); It's used when you need an object only once. After use, it becomes eligible for garbage collection immediately.

Q30. What is an object whose state cannot be changed after creation?

âœ“ Correct Answer: Immutable object

An immutable object's state cannot be modified after creation. String is a classic example. To create immutable classes: make fields final and private, don't provide setters, and don't allow subclasses (make class final).

Q31. How can you request garbage collection (though not guaranteed)?

âœ“ Correct Answer: System.gc()

System.gc() or Runtime.getRuntime().gc() requests garbage collection, but it's not guaranteed to run immediately. The JVM decides when to actually perform GC. Relying on explicit GC calls is generally not recommended.

Q32. What's the difference between shallow copy and deep copy?

âœ“ Correct Answer: Shallow copy copies references, deep copy copies objects

Shallow copy creates a new object but copies references to nested objects (both point to same nested objects). Deep copy creates a new object and recursively copies all nested objects (completely independent copy).

Q33. Which interface must a class implement to be cloneable?

âœ“ Correct Answer: Cloneable

A class must implement the Cloneable interface and override the clone() method to support cloning. If Cloneable is not implemented, calling clone() throws CloneNotSupportedException. Note: Cloneable is a marker interface with no methods.

Q34. What is a best practice for class design?

âœ“ Correct Answer: Use meaningful class and variable names

Best practices include: use meaningful names, keep instance variables private (encapsulation), provide public getters/setters, follow single responsibility principle, and write constructors to ensure valid object state.

Q35. What OOP principle involves hiding implementation details?

âœ“ Correct Answer: Encapsulation

Encapsulation is the practice of hiding implementation details and exposing only necessary interfaces. It's achieved by making fields private and providing public getter/setter methods. This protects object integrity and allows implementation changes without affecting users.
