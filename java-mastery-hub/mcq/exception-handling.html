Q1. Which class is the superclass of all errors and exceptions in Java?

âœ“ Correct Answer: Throwable

`Throwable` is the superclass of all errors and exceptions in the Java language. `Error` and `Exception` are its direct subclasses.

Q2. Which type of exception is NOT checked by the compiler?

âœ“ Correct Answer: RuntimeException

`RuntimeException` and its subclasses are unchecked exceptions. The compiler does not force you to catch or declare them.

Q3. Which block is always executed regardless of whether an exception is thrown or not?

âœ“ Correct Answer: finally

The `finally` block is always executed (unless the JVM exits), making it useful for cleanup code like closing resources.

Q4. Which keyword is used to declare that a method might throw an exception?

âœ“ Correct Answer: throws

The `throws` keyword is used in the method signature to declare that the method might throw one or more exceptions.

Q5. Which keyword is used to explicitly throw an exception?

âœ“ Correct Answer: throw

The `throw` keyword is used to explicitly throw an exception object.

Q6. What exception is thrown when dividing an integer by zero?

âœ“ Correct Answer: ArithmeticException

`ArithmeticException` is thrown when an exceptional arithmetic condition has occurred, such as integer division by zero.

Q7. What exception is thrown when accessing a method or field of a null object?

âœ“ Correct Answer: NullPointerException

`NullPointerException` is thrown when an application attempts to use `null` in a case where an object is required.

Q8. Which interface must a resource implement to be used in a try-with-resources statement?

âœ“ Correct Answer: AutoCloseable

Any object that implements `java.lang.AutoCloseable` (or `java.io.Closeable`) can be used as a resource in a try-with-resources statement.

Q9. Which class represents serious problems that a reasonable application should not try to catch?

âœ“ Correct Answer: Error

`Error` indicates serious problems that a reasonable application should not try to catch (e.g., `OutOfMemoryError`, `StackOverflowError`).

Q10. Can a try block have multiple catch blocks?

âœ“ Correct Answer: Yes

A `try` block can be followed by multiple `catch` blocks to handle different types of exceptions separately.

Q11. In what order should catch blocks be placed?

âœ“ Correct Answer: Most specific to most general

Catch blocks must be ordered from most specific (subclass) to most general (superclass). Otherwise, the specific catch block will never be reached (unreachable code).

Q12. What exception is thrown when parsing an invalid string to a number?

âœ“ Correct Answer: NumberFormatException

`NumberFormatException` is thrown when an attempt is made to convert a string with an incorrect format to a numeric type.

Q13. In which case will the finally block NOT be executed?

âœ“ Correct Answer: System.exit(0)

The `finally` block will not execute if `System.exit()` is called, as it terminates the JVM immediately.

Q14. What is the process of an exception dropping down the call stack called?

âœ“ Correct Answer: Exception Propagation

Exception propagation is the process where an exception is passed from the method where it occurred to the method that called it, and so on, until it is caught.

Q15. How do you create a user-defined exception?

âœ“ Correct Answer: Extend Exception class

To create a custom checked exception, extend the `Exception` class. For a custom unchecked exception, extend `RuntimeException`.

Q16. What exception is thrown when accessing an array with an illegal index?

âœ“ Correct Answer: ArrayIndexOutOfBoundsException

`ArrayIndexOutOfBoundsException` is thrown when an array is accessed with an illegal index (negative or greater than or equal to size).

Q17. Which feature allows catching multiple exceptions in a single catch block (Java 7+)?

âœ“ Correct Answer: Multi-catch block

A multi-catch block allows you to catch multiple exception types in a single `catch` clause, separating them with a vertical bar (`|`).

Q18. What exception is thrown when casting an object to a subclass of which it is not an instance?

âœ“ Correct Answer: ClassCastException

`ClassCastException` is thrown to indicate that the code has attempted to cast an object to a subclass of which it is not an instance.

Q19. Which method of Throwable returns the detailed message string?

âœ“ Correct Answer: getMessage()

`getMessage()` returns the detail message string of the throwable.

Q20. Which method prints the exception and its backtrace to the standard error stream?

âœ“ Correct Answer: printStackTrace()

`printStackTrace()` prints the throwable and its backtrace to the standard error stream.

Q21. What is the concept of wrapping an exception within another exception called?

âœ“ Correct Answer: Chained Exception

Chained exceptions allow you to relate one exception to another (cause), preserving the original stack trace while throwing a higher-level exception.

Q22. What error is thrown when a recursive method calls itself too many times?

âœ“ Correct Answer: StackOverflowError

`StackOverflowError` is thrown when a stack overflow occurs because an application recurses too deeply.

Q23. Is FileNotFoundException a checked or unchecked exception?

âœ“ Correct Answer: Checked Exception

`FileNotFoundException` is a subclass of `IOException`, which is a checked exception. It must be handled or declared.

Q24. Is IllegalArgumentException a checked or unchecked exception?

âœ“ Correct Answer: Unchecked Exception

`IllegalArgumentException` is a subclass of `RuntimeException`, making it an unchecked exception.

Q25. Which statement automatically closes resources at the end of the block?

âœ“ Correct Answer: try-with-resources

The `try-with-resources` statement ensures that each resource is closed at the end of the statement.

Q26. Which method returns the cause of this throwable or null if the cause is nonexistent or unknown?

âœ“ Correct Answer: getCause()

`getCause()` returns the cause of the exception (the exception that triggered this exception) in exception chaining.

Q27. Which keyword is used to test assumptions about the program during development?

âœ“ Correct Answer: assert

The `assert` keyword is used to create assertions, which are boolean expressions that the programmer believes to be true at that point in the code.

Q28. What error is thrown if an assertion fails?

âœ“ Correct Answer: AssertionError

If an assertion evaluates to false, an `AssertionError` is thrown.

Q29. Which method is called by the garbage collector before an object is destroyed (deprecated)?

âœ“ Correct Answer: finalize()

`finalize()` is called by the garbage collector on an object when garbage collection determines that there are no more references to the object. It is deprecated since Java 9.

Q30. Is it good practice to catch the generic Exception class?

âœ“ Correct Answer: No

Catching `Exception` is generally bad practice ("pokemon exception handling") because it can hide bugs and make debugging difficult. Catch specific exceptions instead.
