Q1. What is a variable in Java?

âœ“ Correct Answer: A named memory location that stores a value of a specific data type

A variable is a named memory location that stores a value. In Java, every variable must have a data type (int, String, boolean, etc.), a unique name, and a value. The value can be changed during program execution unless marked as final.

Q2. How many primitive data types does Java have?

âœ“ Correct Answer: 8

Java has 8 primitive data types: byte, short, int, long (integer types), float, double (floating-point types), char (character type), and boolean (logical type). These are the building blocks for data manipulation in Java.

Q3. Which primitive data type is most commonly used for whole numbers?

âœ“ Correct Answer: int

The int data type is the most commonly used for whole numbers. It's 32 bits (4 bytes) and can store values from -2,147,483,648 to 2,147,483,647. It's the default choice for integer values in Java.

Q4. What is the range of the byte data type?

âœ“ Correct Answer: -128 to 127

The byte data type is 8 bits (1 byte) and can store values from -128 to 127. It's useful for small numbers, file I/O operations, and memory optimization when dealing with large arrays of small values.

Q5. Which data type should you use for precise decimal calculations like currency?

âœ“ Correct Answer: BigDecimal

For precise decimal calculations like currency, you should use BigDecimal (a reference type), not float or double. Floating-point types have precision issues due to binary representation. However, if choosing between primitives, double is more precise than float.

Q6. What values can a boolean variable hold?

âœ“ Correct Answer: true or false

A boolean variable can only hold two values: true or false. Unlike C/C++, Java does not allow 0 or 1 to represent boolean values. Boolean is used for logical operations and conditional statements.

Q7. Which data type is used to store a single character?

âœ“ Correct Answer: char

The char data type is used to store a single character. It's 16 bits (2 bytes) and uses Unicode encoding, allowing it to represent characters from various languages. Characters are enclosed in single quotes, like 'A' or '5'.

Q8. Where are primitive variables stored in memory?

âœ“ Correct Answer: Stack memory

Primitive variables are stored on the stack memory. The stack stores actual values directly and provides fast access. Each thread has its own stack, and memory is automatically cleaned up when methods return.

Q9. Where are objects stored in Java?

âœ“ Correct Answer: Heap memory

Objects are stored in heap memory. The heap is shared among all threads and is larger than the stack. Objects remain in the heap until they are garbage collected (when no references point to them).

Q10. What is the default value of an int variable?

âœ“ Correct Answer: 0

The default value of an int variable (and all numeric primitive types) is 0. This applies to instance variables and static variables. Local variables do not have default values and must be initialized before use.

Q11. What is the default value of a boolean variable?

âœ“ Correct Answer: false

The default value of a boolean variable is false. This applies to instance variables and static variables. Remember, local variables must be explicitly initialized.

Q12. What is the default value of a reference type (like String)?

âœ“ Correct Answer: null

The default value of all reference types (objects, arrays, String, etc.) is null. This means the reference doesn't point to any object in memory. Attempting to use a null reference causes a NullPointerException.

Q13. What is it called when you convert a smaller data type to a larger data type automatically?

âœ“ Correct Answer: Widening (implicit) casting

Widening or implicit casting happens automatically when converting a smaller data type to a larger one (e.g., int to long, float to double). No data loss occurs, so Java does it automatically without explicit syntax.

Q14. What type of casting requires explicit syntax because data loss may occur?

âœ“ Correct Answer: Narrowing (explicit) casting

Narrowing or explicit casting is required when converting a larger data type to a smaller one (e.g., double to int, long to short). You must use explicit syntax like (int) because data loss may occur. Example: int x = (int) 3.14;

Q15. What is the wrapper class for the primitive type int?

âœ“ Correct Answer: Integer

Integer is the wrapper class for the primitive type int. Wrapper classes convert primitives to objects, allowing them to be used in collections, provide utility methods, and support null values.

Q16. What is autoboxing in Java?

âœ“ Correct Answer: Automatic conversion between primitive and wrapper types

Autoboxing is the automatic conversion of primitive types to their corresponding wrapper classes (e.g., int to Integer). Unboxing is the reverse. Example: Integer obj = 5; (autoboxing), int num = obj; (unboxing).

Q17. Which keyword is used to create a constant variable in Java?

âœ“ Correct Answer: final

The final keyword is used to create constants in Java. Once a final variable is assigned a value, it cannot be changed. By convention, constant names are written in UPPER_SNAKE_CASE. Example: final int MAX_SIZE = 100;

Q18. What is the recommended naming convention for variables in Java?

âœ“ Correct Answer: camelCase

Variables in Java should use camelCase naming convention: first word lowercase, subsequent words capitalized. Examples: firstName, totalScore, isActive. This is a widely accepted Java coding standard.

Q19. What is the size of the long data type?

âœ“ Correct Answer: 64 bits

The long data type is 64 bits (8 bytes) and can store very large whole numbers from -9,223,372,036,854,775,808 to 9,223,372,036,854,775,807. It's used for timestamps, large file sizes, and big numbers.

Q20. What is the size of the char data type?

âœ“ Correct Answer: 16 bits (Unicode)

The char data type is 16 bits (2 bytes) and uses Unicode encoding. This allows it to represent characters from various languages and special symbols, not just ASCII characters.

Q21. What is the default data type for decimal literals in Java?

âœ“ Correct Answer: double

The default data type for decimal literals (like 3.14) is double. If you want to use float, you must append 'f' or 'F' to the literal: float x = 3.14f;

Q22. Which of the following correctly declares and initializes an integer variable?

âœ“ Correct Answer: int num = 10;

The correct syntax is: dataType variableName = value; In this case, int (lowercase) is the data type, num is the variable name, and 10 is the value. Java is case-sensitive, so 'int' must be lowercase.

Q23. What is true about local variables in Java?

âœ“ Correct Answer: Local variables must be initialized before use

Local variables (declared inside methods) do NOT have default values and must be explicitly initialized before use. They are stored on the stack and exist only during method execution. Attempting to use an uninitialized local variable causes a compilation error.

Q24. Why is String not a primitive data type?

âœ“ Correct Answer: String is a reference type, not a primitive

String is a reference type (class), not a primitive. It's an object that stores a sequence of characters. Strings are stored in the heap memory and have methods like length(), substring(), etc. The 8 primitive types are: byte, short, int, long, float, double, char, boolean.

Q25. What error occurs when the stack memory runs out of space?

âœ“ Correct Answer: StackOverflowError

StackOverflowError occurs when the stack memory runs out of space, typically due to infinite recursion or too many nested method calls. Each method call adds a frame to the stack, and if there are too many, the stack overflows.

Q26. What automatically cleans up unused objects from heap memory?

âœ“ Correct Answer: Garbage Collector

The Garbage Collector (GC) automatically cleans up heap memory by removing objects that have no references pointing to them. This automatic memory management is one of Java's key features, preventing memory leaks.

Q27. What will be the output? int x = 10 / 3; System.out.println(x);

âœ“ Correct Answer: 3

The output is 3 because integer division truncates the decimal part. When dividing two integers, the result is an integer. To get a decimal result, at least one operand must be a floating-point type: double x = 10.0 / 3;

Q28. What happens with this code? int x = 3.14;

âœ“ Correct Answer: Compilation error

This causes a compilation error: "incompatible types: possible lossy conversion from double to int". You cannot assign a double value to an int variable without explicit casting: int x = (int) 3.14;

Q29. What suffix is used to indicate a long literal?

âœ“ Correct Answer: L or l

The suffix 'L' or 'l' is used to indicate a long literal. Example: long bigNum = 9876543210L; It's recommended to use uppercase 'L' because lowercase 'l' can be confused with the digit '1'.

Q30. What suffix is used to indicate a float literal?

âœ“ Correct Answer: F or f

The suffix 'F' or 'f' is used to indicate a float literal. Example: float price = 19.99f; Without the suffix, decimal literals are treated as double by default, which would cause a compilation error when assigning to float.

Q31. How do you denote a char literal in Java?

âœ“ Correct Answer: Single quotes

Char literals are enclosed in single quotes: char letter = 'A'; Double quotes are used for String literals: String word = "Hello"; This is an important distinction in Java.

Q32. What is the main advantage of wrapper classes over primitives?

âœ“ Correct Answer: They provide utility methods and allow null values

Wrapper classes (Integer, Double, Boolean, etc.) provide utility methods (parseInt(), valueOf(), etc.), can be used in collections (which require objects), and can hold null values. However, they use more memory and are slower than primitives.

The automatic type promotion order is: byte â†’ short â†’ int â†’ long â†’ float â†’ double. Java automatically promotes smaller types to larger types in expressions to prevent data loss. Note that float comes before double even though long is 64 bits, because float can represent decimal values.

Q34. Which rule is correct for Java variable names?

âœ“ Correct Answer: It must start with a letter, $, or underscore

Java variable names must start with a letter (a-z, A-Z), dollar sign ($), or underscore (_). After the first character, they can contain letters, digits, $, or _. They cannot start with a digit, contain spaces, or be a Java keyword.

Q35. What is true about stack and heap memory in multi-threaded applications?

âœ“ Correct Answer: Each thread has its own stack, but all threads share the heap

In multi-threaded applications, each thread has its own stack memory for local variables and method calls. However, all threads share the same heap memory for objects. This is why synchronization is needed when multiple threads access shared objects.
