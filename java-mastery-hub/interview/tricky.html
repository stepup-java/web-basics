Tricky Interview Questions (Q501 - Q600)

Test your deep understanding with edge cases, confusing outputs, and tricky scenarios.

This section contains questions that often trip up candidates. They focus on operator precedence, implicit casting, string pool nuances, and exception handling edge cases.

Q501. What will be the output of `System.out.println(1.0/0.0)`?

It prints Infinity. It does not throw `ArithmeticException` because floating-point arithmetic follows IEEE 754 standard, which defines Infinity.

Q502. What will be the output of `System.out.println(0.0/0.0)`?

It prints NaN (Not a Number).

Q503. Can we override a private or static method in Java? ðŸ”¥ Most Asked

No.

  • Private: Not visible to child class, so cannot be overridden.
  • Static: Method hiding occurs, not overriding. The method called depends on the reference type, not the object type.

Q504. What does `System.exit(0)` do in a try-catch-finally block?

It terminates the JVM immediately. The `finally` block will NOT be executed.

Q505. What is the output of `Math.min(Double.MIN_VALUE, 0.0d)`?

It prints 0.0.
Reason: `Double.MIN_VALUE` is the smallest positive non-zero value (4.9E-324), unlike `Integer.MIN_VALUE` which is negative.

Q506. Can we execute a Java program without a main method?

  • JDK 6 and below: Yes, using a static block.
  • JDK 7+: No, the main method is mandatory for execution.

Q507. What happens if we put `return` in both try and finally blocks?

The value returned from the `finally` block will override the value returned from the `try` block. The `try` block's return is discarded.

Q508. Is `String s = null;` the same as `String s = "null";`?

No.

  • `s = null`: The reference points to nothing. Calling methods on it throws NPE.
  • `s = "null"`: The reference points to a String object containing the text "null".

Q509. What is the output of `long l = 1000 * 60 * 60 * 24 * 365L;` vs without 'L'?

Without 'L', the calculation is done in `int`. If the result exceeds `Integer.MAX_VALUE`, it overflows and wraps around (possibly becoming negative) before being assigned to the long variable. With 'L', calculation is done in `long`.

Q510. Can we have a try block without catch or finally?

No. A try block must be followed by either a `catch` block or a `finally` block (or both). In Java 7+, try-with-resources can exist without catch/finally.

Q511. What is the difference between `foo(Object o)` and `foo(String s)` when calling `foo(null)`?

Java chooses the most specific method. Since `String` is a subclass of `Object`, `foo(String s)` will be called.

Q512. What happens if we throw an exception inside a static block?

It causes an `ExceptionInInitializerError` and the class fails to load.

Q513. Can an Interface have a static block?

No. Interfaces cannot have static initialization blocks. They can have static methods (Java 8+), but not static blocks.

Q514. Is `Integer i = 100; Integer j = 100; (i == j)` true or false?

True. Java caches Integer objects from -128 to 127 (Integer Cache). Both point to the same object.

Q515. Is `Integer i = 200; Integer j = 200; (i == j)` true or false?

False. 200 is outside the cache range (-128 to 127), so new objects are created.

Q516. What is the output of `System.out.println('j' + 'a' + 'v' + 'a');`?

It prints the sum of ASCII values of the characters (106 + 97 + 118 + 97 = 418), not "java". To concatenate, start with a string: `"" + 'j' + ...`.

Q517. Can we declare the main method as `final`?

Yes. It doesn't affect execution because the main method is static and cannot be overridden anyway.

Q518. Can we declare the main method as `private`?

It compiles, but at runtime, the JVM will complain that the main method is not public (NoSuchMethodError or "Main method not found").

Q519. What is the difference between `ArrayList` and `Vector`?

  • Vector: Synchronized (Thread-safe), Slow, Legacy.
  • ArrayList: Not Synchronized, Fast, Modern replacement.

Q520. Why is char array preferred over String for storing passwords? ðŸ”¥ Most Asked

Strings are immutable and stay in the String Pool until GC. You cannot explicitly clear them. Char arrays can be explicitly wiped (set to 0s) immediately after use, reducing security risk.

Q521. What is the output of `System.out.println(10 + 20 + "Java");`?

`30Java`. Addition happens left-to-right. 10+20=30, then 30+"Java"="30Java".

Q522. What is the output of `System.out.println("Java" + 10 + 20);`?

`Java1020`. String concatenation happens left-to-right. "Java"+10="Java10", then "Java10"+20="Java1020".

Q523. Can we cast `null` to any type?

Yes. `(String) null` is valid. `(Integer) null` is valid.

Q524. What happens if we cast `null` to `int`?

It throws `NullPointerException` because unboxing attempts to call `intValue()` on null.

Q525. Does `finally` block execute if the thread is killed or power fails?

No. If the JVM crashes or the OS kills the process, `finally` will not execute.

Q526. What is the size of `boolean` in Java?

Not precisely defined by the spec.

  • In an array, it's usually 1 byte (same as byte).
  • As a single variable, it's often treated as an `int` (4 bytes) by the JVM.

Q527. Can we create an instance of an abstract class?

No. But we can create an instance of an anonymous subclass of the abstract class.

Q528. What is Marker Interface?

An interface with no methods. It signals the JVM/Compiler to treat the object specially. (e.g., `Serializable`, `Cloneable`, `Remote`).

Q529. Can we import the same package/class twice?

Yes, the compiler ignores duplicate imports.

Q530. What is the default value of a local variable?

Local variables do not have default values. They must be initialized before use, otherwise, it's a compile-time error.

Q531. Can we have a class named `System`?

Yes, but you will have to use the fully qualified name `java.lang.System` to access the standard System class.

Q532. What is `strictfp` keyword?

Ensures that floating-point calculations result in the same output on every platform (IEEE 754). Without it, some platforms might use higher precision.

Q533. Can we serialize a static variable?

No. Static variables belong to the class, not the object state. They are not serialized.

Q534. What is `transient` keyword?

Prevents a field from being serialized. When deserialized, it will have its default value (null/0).

Q535. Can a constructor be final, static, or abstract?

No. Constructors cannot be overridden (so final is meaningless), are not inherited (so abstract is meaningless), and are instance-creation logic (so static is meaningless).

Q536. What is the difference between `throw` and `throws`?

  • `throw`: Used to explicitly throw an exception instance.
  • `throws`: Used in method signature to declare that the method might throw exceptions.

Q537. Can we catch `Error` in Java?

Yes, you can catch `Throwable` or `Error`, but it is not recommended. Errors (like OOM, StackOverflow) usually indicate serious system problems that an app cannot recover from.

Q538. What is the parent class of all classes?

`java.lang.Object`.

Q539. Does Java support multiple inheritance?

  • Classes: No (to avoid Diamond Problem).
  • Interfaces: Yes (a class can implement multiple interfaces, an interface can extend multiple interfaces).

Q540. What is the Diamond Problem?

Ambiguity that arises when two parent classes have a method with the same signature, and a child class inherits from both. Java avoids this by not allowing multiple class inheritance.

Q541. Can we access a non-static variable from a static context?

No. Static methods belong to the class and run without an instance. Non-static variables require an instance.

Q542. What is `ClassCastException`?

Thrown when trying to cast an object to a subclass of which it is not an instance.

Q543. What is `instanceof` operator?

Used to test if an object is an instance of a specific class, subclass, or interface.

Q544. Can we make a constructor private?

Yes. This prevents instantiation from outside the class. Used in Singleton Pattern and Utility classes (like `java.lang.Math`).

Q545. What is the difference between `==` and `.equals()`? ðŸ”¥ Most Asked

  • `==`: Compares references (memory addresses).
  • `.equals()`: Compares content (if overridden). Default behavior in Object class is same as `==`.

Q546. What is the contract between `hashCode()` and `equals()`?

  • If `a.equals(b)` is true, then `a.hashCode()` MUST equal `b.hashCode()`.
  • If `a.hashCode() == b.hashCode()`, `a.equals(b)` does NOT have to be true (Collision).

Q547. What happens if you override equals() but not hashCode()?

Hash-based collections (HashMap, HashSet) will fail to work correctly. Two "equal" objects might have different hashCodes and end up in different buckets, so you won't be able to find/retrieve them.

Q548. Can we change the value of a final variable?

No. Once assigned, it cannot be reassigned.
Note: If the final variable is a reference to an object, the state of the object can be changed, but the reference cannot.

Q549. What is Blank Final Variable?

A final variable that is not initialized at the time of declaration. It must be initialized in the constructor (for instance variables) or static block (for static variables).

Q550. What is `this` keyword?

Refers to the current object instance. Used to access instance variables, methods, or call other constructors (`this()`).

Q551. What is the output of `System.out.println("Hello".replace('l', 'w'));`?

`Hewwo`. `replace(char, char)` replaces all occurrences.

Q552. What is the output of `System.out.println("Hello".replaceAll("l", "w"));`?

`Hewwo`. `replaceAll(String regex, String replacement)` also replaces all occurrences, but the first argument is a Regex.

Q553. What is the difference between `replace()` and `replaceAll()`?

  • `replace()`: Works with `char` or `CharSequence` (literal string).
  • `replaceAll()`: Works with Regex. Slower if you just want simple string replacement.

Q554. Can we override a method to throw a checked exception if the parent method doesn't?

No. If the parent method does not declare a checked exception, the overriding method cannot declare a new checked exception. It can only throw unchecked exceptions.

Q555. Can we override a method to throw a narrower (subclass) exception?

Yes. This is allowed.

Q556. Can we override a method to throw a broader (superclass) exception?

No. This would break the contract for the caller who is catching the specific exception.

Q557. What happens if we call `run()` method directly instead of `start()`? ðŸ”¥ Most Asked

The code inside `run()` will execute in the current thread (main thread), not in a new separate thread. No multithreading happens.

Q558. Can we start a thread twice?

No. It throws `IllegalThreadStateException`.

Q559. What is the difference between `wait()` and `sleep()`?

  • `wait()`: Releases the lock. Must be called from synchronized context. Object class method.
  • `sleep()`: Keeps the lock. Can be called anywhere. Thread class static method.

Q560. Can we call `wait()` without synchronized block?

No. It throws `IllegalMonitorStateException`.

Q561. What is the output of `System.out.println(Math.round(-1.5));`?

`-1`. `Math.round()` adds 0.5 and takes the floor. -1.5 + 0.5 = -1.0 -> floor is -1.

Q562. What is the output of `System.out.println(Math.round(1.5));`?

`2`. 1.5 + 0.5 = 2.0 -> floor is 2.
