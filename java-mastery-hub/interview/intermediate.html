Intermediate Interview Questions (Q151 - Q350)

  • Home
  • Interview
  • Intermediate

This section dives deeper into Java concepts, covering advanced OOPs, Collections framework, Exception handling nuances, and Java 8 features. Suitable for developers with 2-5 years of experience.

Q151. What is Association, Aggregation, and Composition? ðŸ”¥ Most Asked

  • Association: A general relationship between two classes (e.g., Teacher and Student).
  • Aggregation (HAS-A): A weak relationship where the child can exist independently of the parent (e.g., Department and Teacher).
  • Composition (Part-of): A strong relationship where the child cannot exist without the parent (e.g., House and Room).
// Aggregation
class Department { List teachers; }

// Composition
class House { Room room = new Room(); }

Q152. What is Cohesion and Coupling?

  • Cohesion: How strongly related the responsibilities of a single class are. High cohesion is good.
  • Coupling: How dependent one class is on another. Loose coupling is good.

Q153. What is Covariant Return Type?

Since Java 5, it is possible to override a method by changing its return type, provided the new return type is a subclass of the original return type.

Q154. Can we override private methods?

No. Private methods are not visible to the child class, so they cannot be overridden. If you define a method with the same name in the child class, it is a new method, not an overridden one.

Q155. What is Method Hiding?

When a child class defines a static method with the same signature as a static method in the parent class, the parent's method is "hidden", not overridden. The method called depends on the reference type, not the object type.

Q156. What is a Blank Final Variable?

A `final` variable that is not initialized at the time of declaration. It must be initialized in the constructor (for instance variables) or static block (for static variables).

Q157. Can a constructor be final?

No. Constructors cannot be inherited, so they cannot be overridden. Therefore, marking them as `final` (which prevents overriding) is meaningless and causes a compile-time error.

Q158. What is a Copy Constructor?

A constructor that creates an object using another object of the same class. Java doesn't provide a default copy constructor, but we can define one.

public Student(Student s) {
    this.name = s.name;
    this.age = s.age;
}

Q159. What is Shallow Copy vs Deep Copy? ðŸ”¥ Most Asked

  • Shallow Copy: Copies the main object, but inner objects are shared (references are copied). Default behavior of `clone()`.
  • Deep Copy: Copies the main object and recursively copies all inner objects. New memory is allocated for everything.

Q160. What is the `Cloneable` interface?

A marker interface that indicates a class allows its objects to be cloned using the `Object.clone()` method. If not implemented, `clone()` throws `CloneNotSupportedException`.

Q161. Why is multiple inheritance not supported in Java classes?

To avoid the Diamond Problem (ambiguity). If Class B and Class C extend Class A and both override a method, and Class D extends both B and C, which version of the method should D inherit? Java avoids this by disallowing multiple class inheritance.

Q162. How does Java 8 change Interfaces?

Java 8 introduced:

  • Default Methods: Methods with a body (`default` keyword).
  • Static Methods: Utility methods inside interfaces.

Q163. What is a Functional Interface?

An interface with exactly one abstract method. It can have any number of default or static methods. Used as the basis for Lambda expressions.

Q164. What are Nested Classes?

A class defined within another class. Types:

  • Static Nested Class: Associated with the outer class, not an instance.
  • Inner Class (Non-static): Associated with an instance of the outer class.
  • Local Inner Class: Defined inside a method.
  • Anonymous Inner Class: Inner class without a name.

Q165. Difference between Static Nested Class and Inner Class?

  • Static Nested: Can access only static members of the outer class. Can be instantiated without an outer class instance.
  • Inner Class: Can access both static and non-static members. Requires an outer class instance to be instantiated.

Q166. Why can Local Inner Classes access only final variables?

Because the local variable lives on the stack and disappears when the method ends, but the inner class object might survive on the heap. To ensure consistency, Java captures the value, requiring it to be `final` (or effectively final).

Q167. What is an Enum?

A special data type that represents a fixed set of constants. Enums are implicitly `public static final`. They can have fields, methods, and constructors.

Q168. Can Enums extend a class?

No, because all Enums implicitly extend `java.lang.Enum`, and Java does not support multiple inheritance.

Q169. Can Enums implement interfaces?

Yes, Enums can implement interfaces.

Q170. What are EnumSet and EnumMap?

Specialized Collection implementations for Enums. They are highly efficient and memory-optimized (using bit vectors for EnumSet).

Q171. What is the Exception Hierarchy?

`Throwable` is the root. It has two subclasses:

  • `Error` (System failures)
  • `Exception` (Application issues) -> `RuntimeException` (Unchecked) and others (Checked).

Q172. What is try-with-resources?

A `try` statement that declares one or more resources. The resource is closed automatically at the end of the statement. The resource must implement `AutoCloseable`.

try (BufferedReader br = new BufferedReader(new FileReader("test.txt"))) {
    System.out.println(br.readLine());
} catch (IOException e) {
    e.printStackTrace();
}
Q173. What is a Multi-catch block?

Introduced in Java 7, it allows catching multiple exceptions in a single catch block using the pipe `|` operator. Example: `catch (IOException | SQLException e)`. Reduces code duplication.

try {
    // code that may throw multiple exceptions
} catch (IOException | SQLException e) {
    e.printStackTrace();
}
Q174. What is Exception Propagation?

The process of dropping an exception down the call stack. If a method doesn't handle an exception, it is passed to the caller method. Unchecked exceptions are propagated automatically; checked exceptions must be declared.

Q175. What happens if `finally` block creates an exception?

If an exception is thrown in the `finally` block, the original exception (from `try` or `catch`) is lost/suppressed, and the new exception is propagated.

Q176. Difference between `ClassNotFoundException` and `NoClassDefFoundError`? ðŸ”¥ Most Asked

  • ClassNotFoundException: Checked exception. Thrown when trying to load a class by name (e.g., `Class.forName()`) and it's not found in classpath.
  • NoClassDefFoundError: Error. Thrown when a class was present at compile time but is missing at runtime.

Q177. What is String `intern()`?

A method that puts the string into the String Constant Pool. If the pool already contains an equal string, the reference to the pooled string is returned.

Q178. Why is char[] preferred over String for passwords?

Strings are immutable and stay in the pool until GC. Passwords in Strings are visible in memory dumps. `char[]` can be explicitly wiped (set to 0) immediately after use.

Q179. What is the Integer Cache?

Java caches `Integer` objects for values between -128 and 127. `Integer a = 100; Integer b = 100;` makes `a == b` true. But `Integer a = 200; Integer b = 200;` makes `a == b` false (new objects).

Q180. Difference between `BigInteger` and `BigDecimal`?

  • BigInteger: For arbitrary-precision integers.
  • BigDecimal: For arbitrary-precision decimal numbers. Essential for financial calculations to avoid floating-point errors.

Q181. What is the `transient` keyword?

Prevents a variable from being serialized. When the object is deserialized, the transient variable gets its default value.

Q182. What is `serialVersionUID`?

A unique identifier for Serializable classes. It ensures that the sender and receiver of a serialized object have loaded classes that are compatible with respect to serialization. If it doesn't match, `InvalidClassException` is thrown.

Q183. What is Externalizable interface?

It extends `Serializable` and allows custom serialization logic. The programmer must implement `writeExternal()` and `readExternal()` methods to control what gets saved.

Q184. What is the `volatile` keyword?

Guarantees visibility of changes to variables across threads. It prevents the compiler/CPU from reordering instructions or caching the variable in registers.

Q185. Can we serialize a static variable?

No. Static variables belong to the class, not the object state. Serialization saves the object state, so static variables are ignored.

Q186. What is Reflection?

An API that allows a program to inspect and modify the behavior of classes, methods, and fields at runtime. It can access private members and create objects dynamically.

Q187. What are the drawbacks of Reflection?

  • Performance: Slower than direct calls.
  • Security: Can break encapsulation (access private fields).
  • Maintenance: Code is harder to read and debug.

Q188. What is the `Class` class?

The entry point for Reflection. Instances of the class `Class` represent classes and interfaces in a running Java application.

Q189. How to create a Singleton class?

  1. Make constructor `private`.
  2. Create a `private static` instance of the class.
  3. Provide a `public static` method to return the instance.

Q190. What is the Double-Checked Locking in Singleton?

A thread-safe way to implement Singleton. It checks for null twice: once without locking and once inside a synchronized block, to avoid overhead.

Q191. What is Immutable Class?

A class whose instances cannot be modified after creation. Example: `String`, `Integer`. To create one: make class `final`, fields `private final`, no setters, and deep copy mutable fields in constructor/getters.

Q192. What is the difference between `StringBuffer` and `StringBuilder`?

  • `StringBuffer`: Synchronized (Thread-safe), slower.
  • `StringBuilder`: Not synchronized, faster. Preferred in single-threaded environments.

Q193. What is the `finalize()` method?

Called by GC before reclaiming object memory. It's unreliable and deprecated since Java 9. Use `AutoCloseable` or `Cleaner` instead.

Q194. What is a Memory Leak in Java?

When objects are no longer needed but are still referenced, preventing GC from removing them. Common causes: static collections, unclosed resources, listeners.

Q195. How to detect Memory Leaks?

Using profiling tools like VisualVM, JProfiler, or Eclipse Memory Analyzer (MAT) to analyze heap dumps.

Q196. What is the `System.gc()` method?

Suggests the JVM to run the Garbage Collector. It is not guaranteed to run.

Q197. What are the types of References in Java?

  • Strong: Normal reference (prevents GC).
  • Weak: Collected if only weakly reachable (WeakHashMap).
  • Soft: Collected only if memory is low (Caching).
  • Phantom: Enqueued when object is finalized.

Q198. What is `WeakHashMap`?

A Map implementation where keys are WeakReferences. If a key is no longer referenced elsewhere, the entry is removed from the map during GC.

Q199. What is the `StrictMath` class?

Contains mathematical methods that guarantee the same results on all platforms (bit-for-bit). `Math` class methods might vary slightly due to hardware optimizations.

Q200. What is the `native` keyword?

Indicates that a method is implemented in platform-dependent code (like C/C++) using JNI (Java Native Interface).

Q201. How does HashMap work internally? ðŸ”¥ Most Asked

HashMap works on the principle of hashing.

  • It uses `hashCode()` to calculate the bucket index.
  • It uses `equals()` to check if the key already exists in that bucket (linked list/tree).
  • If collision occurs, nodes are stored in a linked list (or Red-Black Tree after threshold 8 in Java 8).

Q202. What happens if two keys have the same hash code?

A collision occurs. Both keys are stored in the same bucket using a linked list (or tree). When retrieving, `equals()` is used to identify the correct key.

Q203. What is the difference between Fail-Fast and Fail-Safe Iterators? ðŸ”¥ Most Asked

  • Fail-Fast: Throws `ConcurrentModificationException` if collection is modified during iteration (e.g., ArrayList, HashMap).
  • Fail-Safe: Works on a clone/snapshot, so it doesn't throw exception (e.g., CopyOnWriteArrayList, ConcurrentHashMap).

Q204. How does HashSet work internally?

HashSet internally uses a HashMap. The value we insert into HashSet becomes the key in the internal HashMap, and a dummy object (`PRESENT`) is used as the value.

Q205. Difference between ArrayList and LinkedList?

  • ArrayList: Backed by dynamic array. Fast random access (O(1)). Slow insertion/deletion (O(n)) due to shifting.
  • LinkedList: Backed by doubly linked list. Slow random access (O(n)). Fast insertion/deletion (O(1)).

Q206. When to use ArrayList vs LinkedList?

  • Use ArrayList when you have more search/read operations.
  • Use LinkedList when you have more insert/delete operations.

Q207. What is `ConcurrentHashMap`?

A thread-safe implementation of Map. Unlike Hashtable (which locks the whole map), it locks only a portion of the map (Segment Locking or Bucket Locking in Java 8), allowing concurrent reads and updates.

Q208. Difference between HashMap and ConcurrentHashMap?

  • HashMap: Not thread-safe. Fast. Throws ConcurrentModificationException.
  • ConcurrentHashMap: Thread-safe. High concurrency. Does not throw ConcurrentModificationException.

Q209. What is `CopyOnWriteArrayList`?

A thread-safe variant of ArrayList. For every write operation (add, set), it makes a fresh copy of the underlying array. Efficient for read-heavy scenarios.

Q210. What is `IdentityHashMap`?

A Map implementation that uses reference equality (`==`) instead of object equality (`equals()`) for keys.

Q211. What is `WeakHashMap`?

A Map where keys are `WeakReference`. If a key is not referenced anywhere else, it is garbage collected, and the entry is removed from the map automatically.

Q212. What is `LinkedHashMap`?

Extends HashMap and maintains insertion order using a doubly linked list running through all its entries.

Q213. What is `TreeMap`?

Implements `SortedMap`. It stores keys in sorted order (natural or custom comparator). Internally uses a Red-Black Tree.

Q214. What is the difference between `poll()` and `remove()` in Queue?

  • `poll()`: Retrieves and removes the head. Returns `null` if queue is empty.
  • `remove()`: Retrieves and removes the head. Throws `NoSuchElementException` if queue is empty.

Q215. What is the difference between `peek()` and `element()` in Queue?

  • `peek()`: Retrieves but does not remove the head. Returns `null` if empty.
  • `element()`: Retrieves but does not remove the head. Throws exception if empty.

Q216. What is `BlockingQueue`?

A thread-safe queue that waits for the queue to become non-empty when retrieving, and waits for space to become available when storing. Used in Producer-Consumer problems.

Q217. What is Generics in Java?

A feature that allows types (classes and interfaces) to be parameters when defining classes, interfaces, and methods. It provides compile-time type safety and eliminates casting.

Q218. What is Type Erasure?

The process where the Java compiler removes all generic type information during compilation (replacing it with Object or bound). This ensures backward compatibility with older Java versions.

Q219. What is a Wildcard in Generics?

Represented by `?`. It stands for an unknown type.

  • ` `: Unbounded wildcard.
  • ``: Upper bounded wildcard (T or its subclass).
  • ``: Lower bounded wildcard (T or its superclass).

Q220. Can we use Generics with Arrays?

No. We cannot create generic arrays (e.g., `new List[10]` is illegal) because arrays are covariant and retain type info at runtime, while generics are invariant and erased.

Q221. What is `Collections.synchronizedList()`?

A wrapper method that returns a synchronized (thread-safe) view of the specified list. All methods are synchronized on the list object.

Q222. What is `UnsupportedOperationException`?

Thrown when a requested operation is not supported. Common when trying to modify an unmodifiable collection (e.g., adding to `Arrays.asList()`).

Q223. What is the `Iterable` interface?

The root interface of the Collection hierarchy. It allows an object to be the target of the "for-each loop". It provides the `iterator()` method.

Q224. What is `ListIterator`?

An iterator for lists that allows traversing the list in both directions (forward and backward) and modifying elements.

Q225. Difference between Iterator and ListIterator?

  • Iterator: Forward only. Works with List, Set, Queue.
  • ListIterator: Bidirectional. Works only with List.

Q226. What is `Spliterator`?

Introduced in Java 8. It stands for "Splitable Iterator". It is designed for parallel traversal of sources (like streams). It can split the iteration task.

Q227. What is `Deque`?

Double Ended Queue. Supports insertion and removal from both ends. Implementations: `ArrayDeque`, `LinkedList`.

Q228. Why is `ArrayDeque` faster than `Stack`?

`Stack` is a legacy class extending `Vector` (synchronized, slow). `ArrayDeque` is not synchronized and cache-friendly, making it faster for stack operations.

Q229. What is the default size of ArrayList and HashMap?

  • ArrayList: 10
  • HashMap: 16 (Load factor 0.75)

Q230. How to sort a List of Objects?

  • Implement `Comparable` in the class and use `Collections.sort(list)`.
  • Pass a `Comparator` to `Collections.sort(list, comparator)`.

Q231. What is the difference between `Collection` and `Collections`?

  • Collection: Interface. Root of collection hierarchy.
  • Collections: Utility class. Contains static methods to operate on collections.

Q232. What is `NavigableMap`?

An extension of `SortedMap` that provides navigation methods like `lowerEntry()`, `floorEntry()`, `ceilingEntry()`, and `higherEntry()`. Implemented by `TreeMap`.

Q233. What is `BitSet`?

A class that implements a vector of bits that grows as needed. Efficient for storing boolean flags.

Q234. What is `Properties` class?

A subclass of `Hashtable` used to maintain lists of values in which the key and value are both Strings. Commonly used for configuration files.

Q235. Can we add null to TreeSet?

No. Since Java 7, adding null to a TreeSet throws `NullPointerException` because it tries to compare the key.

Q236. What is `RandomAccess` interface?

A marker interface used by List implementations (like ArrayList) to indicate that they support fast (constant time) random access.

Q237. How to synchronize a HashMap?

  • `Collections.synchronizedMap(new HashMap<>());`
  • Use `ConcurrentHashMap` (Preferred).

Q238. What is `ThreadLocal`?

A class that provides thread-local variables. Each thread accessing the variable has its own, independently initialized copy.

Q239. What is `AtomicInteger`?

A class in `java.util.concurrent.atomic` that provides atomic operations on an int value (like `incrementAndGet()`) without using synchronization.

Q240. What is `ReentrantLock`?

A lock implementation that provides more flexibility than `synchronized` blocks (e.g., tryLock, timed lock, interruptible lock).

Q241. What is the difference between `synchronized` and `ReentrantLock`?

  • `synchronized`: Implicit lock, automatic release, less flexible.
  • `ReentrantLock`: Explicit lock, manual release (in finally), more features (fairness, tryLock).

Q242. What is `CountDownLatch`?

A synchronization aid that allows one or more threads to wait until a set of operations being performed in other threads completes.

Q243. What is `CyclicBarrier`?

A synchronization aid that allows a set of threads to all wait for each other to reach a common barrier point.

Q244. Difference between CountDownLatch and CyclicBarrier?

  • `CountDownLatch`: One-time use. Threads wait for a counter to reach zero.
  • `CyclicBarrier`: Reusable. Threads wait for each other to reach a barrier.

Q245. What is `Semaphore`?

A synchronization primitive that maintains a set of permits. Used to control access to a shared resource (e.g., limiting database connections).

Q246. What is `ExecutorService`?

A framework for asynchronous task execution. It manages a pool of threads and assigns tasks to them, decoupling task submission from execution.

Q247. What are the types of Thread Pools?

  • `FixedThreadPool`: Fixed number of threads.
  • `CachedThreadPool`: Creates new threads as needed, reuses idle ones.
  • `SingleThreadExecutor`: Single worker thread.
  • `ScheduledThreadPool`: For scheduled tasks.

Q248. What is `Future` in Java?

Represents the result of an asynchronous computation. Methods like `get()` are used to retrieve the result once the task is complete.

Q249. What is `Callable` interface?

Similar to `Runnable`, but it can return a result and throw a checked exception. Used with ExecutorService.

Q250. Difference between Runnable and Callable? ðŸ”¥ Most Asked

  • Runnable: `run()`, returns void, cannot throw checked exception.
  • Callable: `call()`, returns Object, can throw checked exception.

Q251. What is the difference between `map()` and `flatMap()`? ðŸ”¥ Most Asked

  • map(): Transforms each element into another object. One-to-one mapping. Returns `Stream`.
  • flatMap(): Transforms each element into a Stream of objects, then flattens the streams into a single stream. One-to-many mapping. Returns `Stream`.

Q252. What are Intermediate and Terminal operations in Stream?

  • Intermediate: Return a new Stream, lazy (not executed until terminal op). Examples: `filter`, `map`, `sorted`.
  • Terminal: Produce a result or side-effect, trigger the pipeline. Examples: `collect`, `forEach`, `reduce`, `count`.

Q253. What is `Predicate` interface?

A functional interface that takes one argument and returns a boolean. Method: `boolean test(T t)`. Used in `filter()`.
Predicate<String> isEmpty = s -> s.isEmpty();

Q254. What is `Consumer` interface?

A functional interface that takes one argument and returns nothing (void). Method: `void accept(T t)`. Used in `forEach()`.
Consumer<String> print = s -> System.out.println(s);

Q255. What is `Supplier` interface?

A functional interface that takes no arguments and returns a result. Method: `T get()`. Used in `Optional.orElseGet()`.
Supplier<Double> random = () -> Math.random();

Q256. What is `Function` interface?

A functional interface that takes one argument and returns a result. Method: `R apply(T t)`. Used in `map()`.
Function<String, Integer> len = s -> s.length();

Q257. What is Method Reference?

A shorthand notation of a lambda expression to call a method. Operator: `::`. Example: `System.out::println` instead of `s -> System.out.println(s)`.

Q258. What are the types of Method References?

  1. Static method: `ClassName::staticMethod`
  2. Instance method of particular object: `instance::method`
  3. Instance method of arbitrary object: `ClassName::method`
  4. Constructor: `ClassName::new`

Q259. What is the difference between `Optional.orElse()` and `orElseGet()`?

  • `orElse(T other)`: The `other` value is created/computed even if the Optional is not empty (eager).
  • `orElseGet(Supplier s)`: The supplier is executed only if the Optional is empty (lazy). Better performance.

Q260. What is the Java 8 Date-Time API?

A new, immutable, thread-safe API in `java.time` package to replace `java.util.Date` and `Calendar`. Core classes: `LocalDate`, `LocalTime`, `LocalDateTime`, `ZonedDateTime`.

Q261. Difference between LocalDate, LocalTime, and LocalDateTime?

  • `LocalDate`: Date only (YYYY-MM-DD).
  • `LocalTime`: Time only (HH:MM:SS).
  • `LocalDateTime`: Date and Time, no timezone.

Q262. What is `ZonedDateTime`?

Represents a date and time with a time-zone (e.g., "2023-10-05T10:15:30+01:00[Europe/Paris]"). Used when timezone handling is required.

Q263. How to format a date in Java 8?

Using `DateTimeFormatter`.
String s = localDate.format(DateTimeFormatter.ofPattern("dd/MM/yyyy"));

Q264. What is Period and Duration?

  • Period: Amount of time in years, months, days (Date-based).
  • Duration: Amount of time in seconds, nanoseconds (Time-based).

Q265. What is JDBC?

Java Database Connectivity. An API that defines how a client may access a database. It provides methods to query and update data in a database.

Q266. What are the steps to connect to a database?

  1. Load the driver class (`Class.forName()`).
  2. Create connection (`DriverManager.getConnection()`).
  3. Create statement (`conn.createStatement()`).
  4. Execute query (`stmt.executeQuery()`).
  5. Close connection.

Q267. Difference between Statement and PreparedStatement? ðŸ”¥ Most Asked

  • Statement: Compiles SQL every time. Vulnerable to SQL Injection.
  • PreparedStatement: Pre-compiled SQL. Faster for repeated execution. Prevents SQL Injection.

Q268. What is CallableStatement?

Used to execute stored procedures in the database. It can accept input parameters and return output parameters.

Q269. What is SQL Injection?

A security vulnerability where an attacker can interfere with the queries an application makes to its database (e.g., by inputting `' OR '1'='1`).

Q270. How to prevent SQL Injection?

By using PreparedStatement (parameterized queries) instead of concatenating strings to build SQL queries.

Q271. What is a Connection Pool?

A cache of database connections maintained so that the connections can be reused when future requests to the database are required. It improves performance by avoiding the overhead of creating new connections.

Q272. What is a Design Pattern?

A general, reusable solution to a commonly occurring problem within a given context in software design.

Q273. What are the types of Design Patterns?

  • Creational: Object creation (Singleton, Factory, Builder).
  • Structural: Class/Object composition (Adapter, Decorator, Proxy).
  • Behavioral: Object communication (Observer, Strategy, Template).

Q274. What is Singleton Pattern?

Ensures a class has only one instance and provides a global point of access to it. (e.g., Logging, Database Connection).

Q275. What is Factory Pattern?

A creational pattern that provides an interface for creating objects in a superclass, but allows subclasses to alter the type of objects that will be created.

Q276. What is Abstract Factory Pattern?

A factory of factories. It provides an interface for creating families of related or dependent objects without specifying their concrete classes.

Q277. What is Builder Pattern?

Used to construct a complex object step by step. It separates the construction of a complex object from its representation. (e.g., `StringBuilder`, `Lombok @Builder`).

Q278. What is Prototype Pattern?

Used to create a new object by copying an existing object (prototyping). Useful when object creation is expensive.

Q279. What is Adapter Pattern?

Allows objects with incompatible interfaces to collaborate. It acts as a bridge between two incompatible interfaces.

Q280. What is Decorator Pattern?

Allows adding new functionality to an existing object without altering its structure. (e.g., Java I/O streams `new BufferedReader(new FileReader(...))`).

Q281. What is Observer Pattern?

Defines a subscription mechanism to notify multiple objects about any events that happen to the object they're observing. (e.g., Event Listeners).

Q282. What is Strategy Pattern?

Defines a family of algorithms, puts each of them into a separate class, and makes their objects interchangeable. (e.g., `Collections.sort()` using different Comparators).

Q283. What are SOLID principles? ðŸ”¥ Most Asked

  • S: Single Responsibility Principle
  • O: Open/Closed Principle
  • L: Liskov Substitution Principle
  • I: Interface Segregation Principle
  • D: Dependency Inversion Principle

Q284. What is Single Responsibility Principle (SRP)?

A class should have only one reason to change, meaning it should have only one job or responsibility.

Q285. What is Open/Closed Principle?

Software entities (classes, modules, functions) should be open for extension, but closed for modification.

Q286. What is Liskov Substitution Principle (LSP)?

Objects of a superclass shall be replaceable with objects of its subclasses without breaking the application.

Q287. What is Interface Segregation Principle (ISP)?

Clients should not be forced to depend upon interfaces that they do not use. Better to have many specific interfaces than one general-purpose interface.

Q288. What is Dependency Inversion Principle (DIP)?

High-level modules should not depend on low-level modules. Both should depend on abstractions (interfaces).

Q289. What is Dependency Injection (DI)?

A design pattern where the dependencies of a class are provided from the outside (injected) rather than creating them inside the class. Used heavily in Spring Framework.

Q290. What is Inversion of Control (IoC)?

A principle where the control of flow of a program is inverted: instead of the programmer controlling the flow, an external framework controls it.

Q291. What is a Classloader?

A part of JRE that dynamically loads Java classes into the JVM. Classes are loaded on demand.

Q292. What are the types of Classloaders?

  1. Bootstrap: Loads core Java classes (rt.jar).
  2. Extension (Platform): Loads extensions (lib/ext).
  3. System (Application): Loads classes from classpath.

Q293. How does Classloader work (Delegation Model)?

When a classloader is asked to load a class, it delegates the request to its parent. Only if the parent cannot find the class does the child attempt to load it.

Q294. What is Bootstrap Classloader?

The parent of all classloaders. It is written in native code (C/C++) and loads standard JDK classes from `rt.jar`.

Q295. What is Dynamic Binding (Late Binding)?

The code to be executed is determined at runtime. Method overriding is an example of dynamic binding.

Q296. What is Static Binding (Early Binding)?

The code to be executed is determined at compile time. Method overloading, static, private, and final methods are examples.

Q297. What is the difference between Stack and Heap memory? ðŸ”¥ Most Asked

  • Stack: Stores method calls, local variables, and reference variables. LIFO order. Thread-safe (each thread has own stack).
  • Heap: Stores Objects and JRE classes. Shared by all threads. Subject to Garbage Collection.

Q298. What is JIT Compiler?

Just-In-Time Compiler. It is part of the JVM that compiles bytecodes into native machine code at runtime to improve performance.

Q299. What is HotSpot?

The name of the JVM implementation by Oracle. It identifies "hot spots" (frequently executed code) and optimizes them using JIT compilation.

Q300. What is the difference between JDK, JRE, and JVM?

  • JVM: Executes the bytecode.
  • JRE: JVM + Libraries (to run Java).
  • JDK: JRE + Dev Tools (javac, debugger) (to develop Java).

Q301. What is the Generational Garbage Collection strategy?

The Heap is divided into Young Generation (Eden, Survivor S0, S1) and Old Generation.

  • New objects are created in Eden.
  • Survivors of GC move to S0/S1, then eventually to Old Gen.
  • This improves performance because most objects die young (Minor GC is fast).

Q302. What is the difference between Minor GC and Major GC?

  • Minor GC: Cleans the Young Generation. Frequent and fast.
  • Major GC (Full GC): Cleans the Old Generation (and usually Young Gen too). Infrequent and slow. Can cause "Stop-the-World" pauses.

Q303. What is a "Stop-the-World" event?

A pause in the application execution where all application threads are suspended so that the Garbage Collector can run. Long pauses affect performance/latency.

Q304. What is the Java Memory Model (JMM)?

A specification that describes how threads interact through memory. It defines rules for Atomicity, Visibility, and Ordering (Happens-Before relationship).

Q305. What is the "Happens-Before" relationship?

A guarantee that memory writes by one specific statement are visible to another specific statement. E.g., A write to a `volatile` field happens-before every subsequent read of that field.

Q306. What is the Java Module System (Project Jigsaw)?

Introduced in Java 9 to modularize the JDK and applications. It allows strong encapsulation and explicit dependency declaration using `module-info.java`.

Q307. What is `var` in Java?

Introduced in Java 10 for Local Variable Type Inference. The compiler infers the type from the initializer.
var list = new ArrayList<String>(); // inferred as ArrayList<String>

Q308. What are the rules for using `var`?

  • Must be initialized immediately.
  • Cannot be used for class fields, method parameters, or return types.
  • Cannot be initialized with `null`.

Q309. What are Collection Factory Methods (Java 9)?

Static methods like `List.of()`, `Set.of()`, `Map.of()` to create immutable collections concisely.
List<String> list = List.of("A", "B", "C");

Q310. Difference between `Arrays.asList()` and `List.of()`?

  • `Arrays.asList()`: Returns a fixed-size list backed by the array. Mutable elements (can set values), allows nulls.
  • `List.of()`: Returns a truly immutable list. Cannot modify elements, does not allow nulls.

Q311. What are the new String methods in Java 11?

  • `isBlank()`: Checks if string is empty or contains only whitespace.
  • `lines()`: Returns a stream of lines.
  • `strip()`: Removes whitespace (Unicode aware, unlike `trim()`).
  • `repeat(n)`: Repeats the string n times.

Q312. What is the HttpClient API (Java 11)?

A modern, non-blocking API to perform HTTP requests (GET, POST, etc.). It supports HTTP/2 and WebSocket. Replaces the legacy `HttpURLConnection`.

Q313. What are Switch Expressions (Java 14)?

An enhanced switch that can return a value and uses arrow syntax (`->`) to prevent fall-through.
int days = switch(month) { case JAN, MAR -> 31; case FEB -> 28; default -> 30; };

Q314. What are Text Blocks (Java 15)?

Multi-line string literals delimited by triple quotes (`"""`). They preserve formatting and eliminate the need for escape sequences like `\n`.
String json = """ { "name": "John" } """;

Q315. What is Pattern Matching for `instanceof` (Java 16)?

Eliminates the need for explicit casting after an instanceof check.
if (obj instanceof String s) { System.out.println(s.length()); }

Q316. What are Records (Java 16)? ðŸ”¥ Most Asked

A concise way to create immutable data carrier classes. It automatically generates constructor, getters, `equals`, `hashCode`, and `toString`.
public record Point(int x, int y) {}

Q317. What are Sealed Classes (Java 17)?

Classes or interfaces that restrict which other classes may extend or implement them using the `permits` clause. It gives control over the inheritance hierarchy.

Q318. What is `CompletableFuture`?

An extension of `Future` that supports asynchronous programming. It allows chaining stages (callbacks) and combining multiple futures. It can be manually completed.

Q319. What is `ForkJoinPool`?

A specialized implementation of `ExecutorService` designed for divide-and-conquer algorithms. It uses a work-stealing algorithm where idle threads steal tasks from busy threads.

Q320. What is Parallel Stream?

A stream that processes elements in parallel using the common `ForkJoinPool`. Created using `collection.parallelStream()`. Useful for CPU-intensive tasks on large datasets.

Q321. When should you NOT use Parallel Stream?

  • When the dataset is small (overhead > benefit).
  • When operations are stateful or order-dependent.
  • When tasks are I/O intensive (blocks the common pool).

Q322. What are Private Methods in Interfaces (Java 9)?

Interfaces can have `private` methods to share common code between `default` and `static` methods within the interface, improving encapsulation.

Q323. What is the Facade Design Pattern?

A structural pattern that provides a simplified interface to a library, a framework, or any other complex set of classes. (e.g., A `CarStarter` class hiding the complexity of `Engine`, `Fuel`, `Battery`).

Q324. What is the Proxy Design Pattern?

A structural pattern that provides a surrogate or placeholder for another object to control access to it. (e.g., Lazy loading, Security access control, Spring AOP proxies).

Q325. What is the Flyweight Design Pattern?

A structural pattern that lets you fit more objects into the available amount of RAM by sharing common parts of state between multiple objects instead of keeping all of the data in each object. (e.g., String Constant Pool).

Q326. What is the Template Method Pattern?

A behavioral pattern that defines the skeleton of an algorithm in the superclass but lets subclasses override specific steps of the algorithm without changing its structure.

Q327. What is the Command Pattern?

A behavioral pattern that turns a request into a stand-alone object that contains all information about the request. This allows parameterizing methods with different requests, queuing requests, etc.

Q328. What is `readResolve()` method?

Used during deserialization. If defined, it is called after the object is deserialized. It is commonly used in Singleton classes to return the existing instance instead of the newly deserialized one.

Q329. What is `writeReplace()` method?

Used during serialization. It allows an object to nominate a replacement object to be written to the stream instead of itself.

Q330. What is a Marker Interface?

An empty interface (no methods). It signals to the JVM or compiler that the objects of the class implementing it need to be treated specially (e.g., `Serializable`, `Cloneable`, `Remote`).

Q331. What are Annotations?

Metadata that provides data about a program but is not part of the program itself. They have no direct effect on the operation of the code they annotate. Used for compile-time checks, code generation, or runtime processing.

Q332. What are Meta-Annotations?

Annotations that apply to other annotations.

  • `@Target`: Where the annotation can be used (Method, Class, Field).
  • `@Retention`: How long it is retained (Source, Class, Runtime).
  • `@Documented`: Included in Javadoc.
  • `@Inherited`: Inherited by subclasses.

Q333. How to create a Custom Annotation?

Using `@interface` keyword.
public @interface MyAnnotation { String value(); }

Q334. What is `@FunctionalInterface` annotation?

An informative annotation used to indicate that an interface type declaration is intended to be a functional interface (one abstract method). Compiler generates an error if it's not.

Q335. What is Reflection?

A feature that allows a running program to examine or "introspect" upon itself, and manipulate internal properties of the program. For example, it's possible for a Java class to obtain the names of all its members and display them.

Q336. Can Reflection access private fields?

Yes, by calling `field.setAccessible(true)`. This breaks encapsulation and is often used by frameworks like Spring and Hibernate.

Q337. What is the `Unsafe` class?

`sun.misc.Unsafe` is an internal class that provides low-level operations like direct memory access and object instantiation without constructors. It is dangerous and not recommended for standard development.

Q338. What is JNI (Java Native Interface)?

A framework that enables Java code running in a JVM to call and be called by native applications (programs specific to a hardware and OS platform) and libraries written in other languages like C, C++ and Assembly.

Q339. What is `System.nanoTime()` vs `System.currentTimeMillis()`?

  • `currentTimeMillis()`: Wall-clock time. Used for dates/logging. Can be changed by system.
  • `nanoTime()`: Monotonic time from an arbitrary origin. Used for measuring elapsed time (performance). Not related to wall-clock.

Q340. What is `Math.round()` behavior?

It rounds a floating-point number to the nearest integer. It adds 0.5 and casts to int/long (floor).

Q341. What is the difference between `Checked` and `Unchecked` exceptions?

  • Checked: Extend `Exception`. Must be handled (try-catch) or declared (throws). E.g., `IOException`.
  • Unchecked: Extend `RuntimeException`. Handling is optional. E.g., `NullPointerException`.

Q342. What is `ClassCastException`?

Thrown when trying to cast an object to a subclass of which it is not an instance.

Q343. What is `StackOverflowError`?

Thrown when the stack memory is exhausted, usually due to deep or infinite recursion.

Q344. What is `OutOfMemoryError`?

Thrown when the Java Virtual Machine cannot allocate an object because it is out of memory, and no more memory could be made available by the garbage collector.

Q345. What is `NoClassDefFoundError`?

Thrown if the Java Virtual Machine or a ClassLoader instance tries to load in the definition of a class and no definition of the class could be found. It means the class was present at compile time but missing at runtime.

Q346. What is `NullPointerException`?

Thrown when an application attempts to use `null` in a case where an object is required (calling method, accessing field, etc.).

Q347. How to avoid `NullPointerException`?

  • Use `Optional`.
  • Check for null explicitly (`if (obj != null)`).
  • Use `Objects.requireNonNull()`.
  • Return empty collections instead of null.

Q348. What is `IllegalArgumentException`?

Thrown to indicate that a method has been passed an illegal or inappropriate argument.

Q349. What is `IllegalStateException`?

Signals that a method has been invoked at an illegal or inappropriate time. The Java environment or Java application is not in an appropriate state for the requested operation.

Q350. What is the `default` keyword in Java?

  • In `switch` statement: The default block.
  • In `interface` (Java 8+): To define a method with a body.
  • In annotations: To specify a default value.
