    Advanced Interview Questions (Q351 - Q500)

    Master advanced concepts: Concurrency, JVM Internals, Performance Tuning, and System Design.

    This section is designed for Senior Developer and Tech Lead roles. It covers deep technical topics that require a solid understanding of how Java works under the hood.

    Q351. What is the Volatile keyword? ðŸ”¥ Most Asked

    `volatile` is a keyword used to indicate that a variable's value will be modified by different threads.

    • It guarantees visibility: Changes made by one thread are immediately visible to others.
    • It prevents instruction reordering by the compiler/CPU.
    • It does not guarantee atomicity (e.g., `i++` is not atomic even if `i` is volatile).

    Q352. What is the difference between `volatile` and `synchronized`?

    • `volatile`: Ensures visibility. Lightweight. No locking. Cannot guarantee atomicity for compound actions.
    • `synchronized`: Ensures visibility AND atomicity. Heavier (involves locking).

    Q353. What is AtomicReference?

    A class in `java.util.concurrent.atomic` that allows you to update an object reference atomically. It uses CAS (Compare-And-Swap) operations.

    Q354. What is CAS (Compare-And-Swap)?

    A low-level CPU instruction used for optimistic locking. It updates a variable only if it holds the expected value. If not, it retries. It avoids the overhead of traditional locking.

    Q355. What is ABA problem in Concurrency?

    A problem in CAS operations where a value is changed from A to B and back to A. A thread checking for A might think nothing changed, but side effects might have occurred. Solved using `AtomicStampedReference`.

    Q356. What is `Thread.join()`?

    A method that allows one thread to wait for the completion of another. If `t1.join()` is called in main thread, main thread waits until `t1` dies.

    Q357. What is `Thread.yield()`?

    A hint to the scheduler that the current thread is willing to yield its current use of a processor. The scheduler is free to ignore this hint.

    Q358. What is a Daemon Thread?

    A low-priority thread that runs in the background to perform tasks like Garbage Collection. The JVM exits when all user threads (non-daemon) finish, killing any remaining daemon threads.

    Q359. How to create a Daemon thread?

    Call `thread.setDaemon(true)` before starting the thread.

    Q360. What is Thread Starvation?

    A situation where a thread is unable to gain regular access to shared resources and is unable to make progress. This happens when high-priority threads constantly grab the CPU.

    Q361. What is Livelock?

    A situation where threads are not blocked, but they are too busy responding to each other to resume work. (e.g., two people trying to pass each other in a corridor, both stepping left/right repeatedly).

    Q362. What is Deadlock? ðŸ”¥ Most Asked

    A situation where two or more threads are blocked forever, waiting for each other to release a lock. (Thread A holds Lock 1 wants Lock 2; Thread B holds Lock 2 wants Lock 1).

    Q363. How to prevent Deadlock?

    • Acquire locks in a fixed order.
    • Use `tryLock()` with timeout.
    • Minimize lock scope.

    Q364. What is Race Condition?

    A condition where the system's behavior depends on the sequence or timing of other uncontrollable events (like thread scheduling). Occurs when multiple threads access shared data without synchronization.

    Q365. What is `ThreadLocalRandom`?

    A random number generator isolated to the current thread. It is more efficient than `Math.random()` or `Random` in multi-threaded environments because it avoids contention.

    Q366. What is `Phaser`?

    A reusable synchronization barrier, similar to CyclicBarrier and CountDownLatch, but more flexible. It supports dynamic number of parties.

    Q367. What is `Exchanger`?

    A synchronization point where threads can pair and swap elements within pairs. Useful for genetic algorithms or pipeline designs.

    Q368. What is `CompletableFuture.allOf()`?

    A static method that returns a new CompletableFuture that is completed when all of the given CompletableFutures complete.

    Q369. What is `CompletableFuture.anyOf()`?

    Returns a new CompletableFuture that is completed when any one of the given CompletableFutures completes.

    Q370. What is Spurious Wakeup?

    A thread might wake up from waiting (wait/condition) without being notified. This is why `wait()` should always be called inside a loop checking the condition.

    Q371. What is False Sharing?

    A performance degradation that occurs when multiple threads modify independent variables that happen to share the same cache line. This causes excessive cache invalidation. Solved using padding (e.g., `@Contended`).

    Q372. What is `@Contended` annotation?

    Introduced in Java 8 to reduce false sharing by adding padding to fields, ensuring they reside on different cache lines.

    Q373. What is Busy Spinning?

    A technique where a thread repeatedly checks a condition in a loop (consuming CPU) instead of yielding or blocking. Useful for very short waits to avoid context switch overhead.

    Q374. What is Context Switching?

    The process of storing the state of a process/thread so that it can be restored and execution resumed later. It is computationally expensive.

    Q375. What is the difference between User Thread and Kernel Thread?

    • User Thread: Managed by user-level libraries. Fast creation.
    • Kernel Thread: Managed by OS kernel. Slower creation but can utilize multiple CPUs. Java threads map 1:1 to Kernel threads (mostly).

    Q376. What are Virtual Threads (Project Loom)? ðŸ”¥ Most Asked

    Introduced in Java 19/21. Lightweight threads managed by the JVM (not OS). Millions can be created. They make blocking I/O cheap and simplify concurrent programming (Thread-per-request model).

    Q377. Difference between Platform Threads and Virtual Threads?

    • Platform Thread: OS wrapper. Expensive (1MB stack). Limited in number.
    • Virtual Thread: JVM managed. Cheap (bytes). Unlimited. Mounted on platform threads only when executing.

    Q378. What is Structured Concurrency?

    An API (incubating) to treat multiple tasks running in different threads as a single unit of work. It improves error handling and cancellation.

    Q379. What is Scoped Value?

    A modern alternative to `ThreadLocal`. It allows sharing immutable data within and across threads in a structured way, with better performance and memory footprint.

    Q380. What is Escape Analysis?

    A compiler optimization technique. If an object is allocated in a method and does not "escape" (is not returned or assigned to a global), the compiler might allocate it on the Stack instead of Heap (Stack Allocation), reducing GC pressure.

    Q381. What is Lock Elision?

    An optimization where the JVM removes locks if it detects that the object is only accessible by one thread (e.g., synchronizing on a local variable).

    Q382. What is Biased Locking?

    An optimization where a lock is "biased" towards the first thread that acquires it. Subsequent acquisitions by the same thread are very fast. (Deprecated in newer Java versions).

    Q383. What is String Deduplication?

    A GC feature (G1GC) that identifies duplicate string values on the heap and makes them point to the same char array, saving memory.

    Q384. What is Metaspace?

    Replaced PermGen in Java 8. It stores class metadata. It uses native memory (not Heap) and grows automatically.

    Q385. What is the difference between PermGen and Metaspace?

    • PermGen: Fixed size (could lead to OOM). Part of Heap (logically).
    • Metaspace: Dynamic size. Native memory. Less prone to OOM.

    Q386. What is G1 GC (Garbage First)?

    The default GC in modern Java. It divides the heap into regions. It prioritizes collecting regions with the most garbage ("Garbage First"). Good for large heaps and predictable pause times.

    Q387. What is ZGC (Z Garbage Collector)?

    A scalable low-latency GC. It performs expensive work concurrently, without stopping application threads. Pauses are typically < 1ms, regardless of heap size (even TBs).

    Q388. What is Shenandoah GC?

    Another low-latency GC (like ZGC) that compacts the heap concurrently.

    Q389. What is Class Data Sharing (CDS)?

    A feature to reduce startup time and memory footprint. It dumps loaded classes into an archive file, which can be memory-mapped by future JVM runs.

    Q390. What is AOT (Ahead-Of-Time) Compilation?

    Compiling Java classes to native code before execution (unlike JIT). Improves startup time but might have lower peak performance than JIT.

    Q391. What is GraalVM?

    A high-performance JDK distribution. It includes a polyglot VM and a Native Image builder that compiles Java apps into standalone native executables (instant startup, low memory).

    Q392. What is Native Image?

    A technology in GraalVM to compile Java code into a platform-specific binary. It does not require a JVM to run.

    Q393. What is the difference between Shallow Copy and Deep Copy?

    • Shallow: Copies the object, but fields pointing to other objects still point to the same original objects.
    • Deep: Copies the object and recursively copies all objects it references.

    Q394. How to implement Deep Copy?

    • Manually clone/copy sub-objects.
    • Serialization/Deserialization.
    • Libraries like Gson/Jackson.

    Q395. What is Serialization Proxy Pattern?

    A pattern where an inner static class (Proxy) represents the logical state of the enclosing class. The enclosing class serializes the proxy instead of itself. Improves security and flexibility.

    Q396. What is Externalizable interface?

    Extends `Serializable`. It gives the programmer complete control over the serialization logic via `writeExternal()` and `readExternal()` methods. Requires a public no-arg constructor.

    Q397. What is Double Checked Locking?

    A pattern used to reduce synchronization overhead in Singleton. It checks for null twice: once without locking, and once with locking. Requires `volatile` instance variable.

    Q398. What is the difference between `Comparable` and `Comparator`?

    • `Comparable`: Natural ordering (`compareTo()`). Implemented by the class itself.
    • `Comparator`: Custom ordering (`compare()`). Implemented by a separate class/lambda.

    Q399. What is `System.gc()`?

    A request to the JVM to run the Garbage Collector. It is not guaranteed to run.

    Q400. What is a Memory Leak in Java?

    When objects are no longer needed but are still referenced by the application (e.g., in a static Map), preventing the GC from reclaiming them. Eventually leads to OOM.

    Q401. What is Inversion of Control (IoC)? ðŸ”¥ Most Asked

    A design principle where the control of object creation and dependency management is transferred from the programmer to a container (like Spring IoC Container). It promotes loose coupling.

    Q402. What is Dependency Injection (DI)?

    A design pattern used to implement IoC. Instead of an object creating its dependencies, they are "injected" into it (via Constructor, Setter, or Field).
    @Autowired private UserService userService;

    Q403. What are the scopes of a Spring Bean?

    • Singleton: (Default) One instance per container.
    • Prototype: New instance every time it is requested.
    • Request: One instance per HTTP request.
    • Session: One instance per HTTP session.
    • Application: One instance per ServletContext.

    Q404. What is the difference between @Component, @Service, @Repository, and @Controller?

    They are all stereotypes for Spring beans.

    • @Component: Generic stereotype.
    • @Service: For service layer (business logic).
    • @Repository: For DAO layer (catches persistence exceptions).
    • @Controller: For presentation layer (Spring MVC).

    Q405. What is Spring Boot?

    An extension of the Spring Framework that simplifies the setup and development of new Spring applications. It provides defaults for code and annotation configuration to quick-start projects.

    Q406. What are the key features of Spring Boot?

    • Auto-configuration: Automatically configures beans based on classpath.
    • Starter Dependencies: Curated groups of dependencies (e.g., `spring-boot-starter-web`).
    • Embedded Server: Tomcat/Jetty embedded (no need to deploy WAR).
    • Actuator: Production-ready features (metrics, health checks).

    Q407. How does Spring Boot Auto-Configuration work? ðŸ”¥ Most Asked

    It attempts to automatically configure your Spring application based on the jar dependencies that you have added. It uses @EnableAutoConfiguration (part of @SpringBootApplication). It looks for META-INF/spring.factories (or org.springframework.boot.autoconfigure.AutoConfiguration.imports in newer versions).

    Q408. What is the difference between @RestController and @Controller?

    • @Controller: Used for traditional Spring MVC (returns views/JSP).
    • @RestController: Used for RESTful web services. It combines @Controller and @ResponseBody (returns data/JSON).

    Q409. What is @RequestMapping?

    An annotation used to map web requests to specific handler classes and/or handler methods. Can specify URL, HTTP method, headers, etc.

    Q410. What are @GetMapping, @PostMapping, etc.?

    Shortcut annotations for @RequestMapping with specific HTTP methods.

    • @GetMapping = @RequestMapping(method = RequestMethod.GET)
    • @PostMapping, @PutMapping, @DeleteMapping, @PatchMapping.

    Q411. What is @PathVariable vs @RequestParam?

    • @PathVariable: Extracts values from the URI path (e.g., /users/123 -> 123).
    • @RequestParam: Extracts values from query parameters (e.g., /users?id=123 -> 123).

    Q412. What is @RequestBody?

    Maps the HTTP request body (JSON/XML) to a Java object (deserialization). Used in POST/PUT requests.

    Q413. What is @ResponseBody?

    Indicates that the return value of a method should be written directly to the HTTP response body (serialization to JSON/XML), rather than resolving to a view.

    Q414. What is Spring Boot Actuator?

    A sub-project that provides production-ready features to help you monitor and manage your application. Endpoints: /actuator/health, /actuator/info, /actuator/metrics.

    Q415. How to define custom properties in Spring Boot?

    Define them in application.properties or application.yml. Access them using @Value("${property.name}") or bind them to a class using @ConfigurationProperties.

    Q416. What are Spring Profiles?

    A way to segregate parts of your application configuration and make it only available in certain environments (e.g., dev, test, prod).
    application-dev.properties, @Profile("dev").

    Q417. What is AOP (Aspect Oriented Programming)?

    A programming paradigm that aims to increase modularity by allowing the separation of cross-cutting concerns (logging, security, transaction management).

    • Aspect: The concern itself (e.g., Logging).
    • Advice: Action taken (Before, After, Around).
    • Pointcut: Where to apply the advice.

    Q418. What is a Microservice?

    An architectural style where an application is structured as a collection of small, loosely coupled, independently deployable services, organized around business capabilities.

    Q419. Monolithic vs Microservices Architecture? ðŸ”¥ Most Asked

    • Monolith: Single codebase, single deployment unit. Simple to develop initially, hard to scale/maintain as it grows.
    • Microservices: Distributed, independent scaling, technology agnostic. Complex to manage (network latency, distributed transactions).

    Q420. What is CAP Theorem?

    In a distributed system, you can only have 2 out of 3 guarantees:

    • Consistency (Every read receives the most recent write).
    • Availability (Every request receives a response).
    • Partition Tolerance (System continues to operate despite network failures).
    Microservices usually choose AP or CP.

    Q421. What is Service Discovery (Eureka)?

    A mechanism for microservices to find each other without hardcoding URLs. Services register with a discovery server (e.g., Netflix Eureka), and clients query the server to get the location of a service.

    Q422. What is API Gateway (Zuul / Spring Cloud Gateway)?

    A single entry point for all clients. It handles routing, load balancing, authentication, rate limiting, and SSL termination.

    Q423. What is Circuit Breaker Pattern (Hystrix / Resilience4j)?

    A pattern to prevent cascading failures. If a service is down or slow, the circuit breaker "opens" and returns a fallback response immediately, instead of waiting for the timeout.

    Q424. What is Feign Client?

    A declarative web service client. It makes writing web service clients easier. You define an interface and annotate it, and Spring creates the implementation.

    Q425. What is Distributed Tracing (Zipkin / Sleuth)?

    A way to track a request as it flows through multiple microservices. It assigns a unique Trace ID to the request chain to help debugging latency issues.

    Q426. What is Config Server?

    A centralized place to manage external properties for applications across all environments. Changes can be made without redeploying services.

    Q427. What is Idempotency in REST?

    A property where making the same request multiple times has the same effect as making it once.

    • Safe: GET, HEAD, OPTIONS (No side effects).
    • Idempotent: PUT, DELETE (State is same after N calls).
    • Non-Idempotent: POST (Creates new resource every time).

    Q428. What are HTTP Status Codes?

    • 2xx: Success (200 OK, 201 Created).
    • 3xx: Redirection.
    • 4xx: Client Error (400 Bad Request, 401 Unauthorized, 403 Forbidden, 404 Not Found).
    • 5xx: Server Error (500 Internal Server Error, 503 Service Unavailable).

    Q429. What is HATEOAS?

    Hypermedia As The Engine Of Application State. A constraint of REST where the API returns links to related resources, guiding the client on what actions are possible next.

    Q430. What is Spring Security?

    A powerful and customizable authentication and access-control framework. It is the de-facto standard for securing Spring-based applications.

    Q431. Authentication vs Authorization?

    • Authentication (Who are you?): Verifying identity (Login).
    • Authorization (What can you do?): Verifying permissions (Access Control).

    Q432. What is JWT (JSON Web Token)?

    A compact, URL-safe means of representing claims to be transferred between two parties. Commonly used for stateless authentication in microservices.

    Q433. What is OAuth2?

    An authorization framework that enables applications to obtain limited access to user accounts on an HTTP service (like Google/Facebook) without exposing user passwords.

    Q434. What is Hibernate?

    An ORM (Object-Relational Mapping) framework that maps Java objects to database tables. It simplifies data persistence.

    Q435. JPA vs Hibernate?

    • JPA (Java Persistence API): The specification (Interface).
    • Hibernate: The implementation of JPA (Provider).

    Q436. What is Entity Lifecycle in JPA?

    • Transient: New object, not in DB.
    • Persistent: Managed by Session, has DB identity.
    • Detached: Session closed, object exists but not managed.
    • Removed: Scheduled for deletion.

    Q437. What is Lazy Loading vs Eager Loading?

    • Lazy: Data is loaded only when accessed (default for OneToMany).
    • Eager: Data is loaded immediately with the parent (default for ManyToOne).

    Q438. What is N+1 Select Problem? ðŸ”¥ Most Asked

    A performance issue where 1 query fetches the parent records, and N subsequent queries fetch the child records for each parent. Solved using JOIN FETCH.

    Q439. What is @Transactional?

    A Spring annotation that ensures a method is executed within a database transaction. If an exception occurs (unchecked), the transaction is rolled back.

    Q440. What is First Level Cache vs Second Level Cache in Hibernate?

    • L1 Cache: Session level. Enabled by default. Mandatory.
    • L2 Cache: SessionFactory level. Shared across sessions. Optional (e.g., EhCache, Redis).

    Q441. What is SAGA Pattern?

    A pattern for managing distributed transactions in microservices. It breaks a transaction into a sequence of local transactions, with compensating transactions to undo changes if a step fails.

    Q442. What is CQRS (Command Query Responsibility Segregation)?

    A pattern that separates read and write operations into different models. Useful for high-performance systems where reads vastly outnumber writes.

    Q443. What is Event Sourcing?

    Storing the state of a system as a sequence of events (state changes) rather than just the current state. Allows replaying events to reconstruct state.

    Q444. What is 12-Factor App?

    A methodology for building modern, scalable, cloud-native software-as-a-service apps. Principles include Config, Backing Services, Stateless Processes, Disposability, etc.

    Q445. What is Blue-Green Deployment?

    A deployment strategy with two identical environments (Blue and Green). One is live, the other is idle. New version is deployed to idle, tested, and then traffic is switched.

    Q446. What is Canary Deployment?

    Rolling out a new version to a small subset of users first to test stability before full rollout.

    Q447. What is Docker?

    A platform for developing, shipping, and running applications in containers. Containers package code and dependencies together.

    Q448. What is Kubernetes (K8s)?

    An open-source container orchestration system for automating software deployment, scaling, and management.

    Q449. What is CI/CD?

    Continuous Integration (frequent code merges, auto-tests) and Continuous Delivery/Deployment (auto-release to production).

    Q450. What is SOLID Principles? ðŸ”¥ Most Asked

    • Single Responsibility Principle
    • Open/Closed Principle
    • Liskov Substitution Principle
    • Interface Segregation Principle
    • Dependency Inversion Principle

    Q451. What is Load Balancing?

    The process of distributing network traffic across multiple servers to ensure no single server bears too much demand. Improves responsiveness and availability.

    Q452. What are different Load Balancing algorithms?

    • Round Robin: Sequential distribution.
    • Least Connections: Sends request to server with fewest active connections.
    • IP Hash: Determines server based on client IP.

    Q453. What is Horizontal vs Vertical Scaling? ðŸ”¥ Most Asked

    • Vertical (Scale Up): Adding more power (CPU, RAM) to an existing machine. Limited by hardware.
    • Horizontal (Scale Out): Adding more machines to the pool. Unlimited scaling potential.

    Q454. What is Caching?

    Storing copies of data in a temporary storage location (cache) so that future requests for that data can be served faster. (e.g., Redis, Memcached, CDN).

    Q455. What are Cache Eviction Policies?

    Rules to decide which item to remove when the cache is full.

    • LRU (Least Recently Used): Discard items not used for the longest time.
    • LFU (Least Frequently Used): Discard items used least often.
    • FIFO (First In First Out).

    Q456. What is Database Sharding?

    A type of database partitioning that separates very large databases into smaller, faster, more easily managed parts called data shards. Horizontal partitioning.

    Q457. What is Replication?

    Storing the same data on multiple servers to improve availability and reliability. (Master-Slave, Master-Master).

    Q458. What is CDN (Content Delivery Network)?

    A geographically distributed network of proxy servers and their data centers. It delivers content (images, CSS, JS) to users based on their geographic location (lower latency).

    Q459. What is Message Queue (Kafka / RabbitMQ)?

    A form of asynchronous service-to-service communication used in serverless and microservices architectures. Messages are stored on the queue until they are processed and deleted.

    Q460. What is SQL Injection? ðŸ”¥ Most Asked

    A security vulnerability where an attacker interferes with the queries an application makes to its database.
    Prevention: Use Prepared Statements (Parameterized Queries).

    Q461. What is XSS (Cross-Site Scripting)?

    An attack where malicious scripts are injected into trusted websites. The script executes in the victim's browser.
    Prevention: Input validation and Output encoding (escaping).

    Q462. What is CSRF (Cross-Site Request Forgery)?

    An attack that forces an end user to execute unwanted actions on a web application in which they are currently authenticated.
    Prevention: Use CSRF Tokens (Synchronizer Token Pattern).

    Q463. Hashing vs Encryption?

    • Hashing: One-way function. Cannot be reversed. Used for passwords (SHA-256, BCrypt).
    • Encryption: Two-way function. Can be decrypted with a key. Used for data transmission (AES, RSA).

    Q464. What is Salt in password hashing?

    Random data added to a password before hashing it. It defends against dictionary attacks and rainbow table attacks.

    Q465. What is SSL/TLS?

    Protocols for establishing authenticated and encrypted links between networked computers. HTTPS uses TLS.

    Q466. What is Symmetric vs Asymmetric Encryption?

    • Symmetric: Same key for encryption and decryption (Faster, e.g., AES).
    • Asymmetric: Public key to encrypt, Private key to decrypt (Slower, e.g., RSA).

    Q467. What is CORS (Cross-Origin Resource Sharing)?

    A mechanism that uses additional HTTP headers to tell browsers to give a web application running at one origin, access to selected resources from a different origin.

    Q468. What is Idempotent Consumer Pattern?

    Ensures that processing the same message multiple times has the same effect as processing it once. Critical in message queues where "at-least-once" delivery is common.

    Q469. What is Rate Limiting?

    Controlling the rate of traffic sent or received by a network interface controller. Prevents DoS attacks and overuse of resources.

    Q470. What is Single Sign-On (SSO)?

    An authentication scheme that allows a user to log in with a single ID and password to any of several related, yet independent, software systems.

    Q471. What is Design Pattern?

    A general repeatable solution to a commonly occurring problem in software design.

    Q472. What is Singleton Pattern?

    Ensures a class has only one instance and provides a global point of access to it. (e.g., Logging, Database Connection).

    Q473. What is Factory Pattern?

    Defines an interface for creating an object, but let subclasses decide which class to instantiate. (e.g., `Calendar.getInstance()`).

    Q474. What is Builder Pattern?

    Separates the construction of a complex object from its representation. Allows creating objects step-by-step. (e.g., `StringBuilder`, `Stream.Builder`).

    Q475. What is Observer Pattern?

    Defines a one-to-many dependency between objects so that when one object changes state, all its dependents are notified and updated automatically. (e.g., Event Listeners).

    Q476. What is Strategy Pattern?

    Defines a family of algorithms, encapsulates each one, and makes them interchangeable. (e.g., `Collections.sort(list, comparator)`).

    Q477. What is Decorator Pattern?

    Attaches additional responsibilities to an object dynamically. Flexible alternative to subclassing. (e.g., `java.io` classes like `BufferedReader(FileReader)`).

    Q478. What is Adapter Pattern?

    Converts the interface of a class into another interface clients expect. (e.g., `Arrays.asList()`).

    Q479. What is Proxy Pattern?

    Provides a surrogate or placeholder for another object to control access to it. (e.g., Spring AOP proxies, Hibernate Lazy Loading).

    Q480. What is Facade Pattern?

    Provides a simplified interface to a library, a framework, or any other complex set of classes.

    Q481. What is Template Method Pattern?

    Defines the skeleton of an algorithm in a method, deferring some steps to subclasses. (e.g., `AbstractList`, Servlet `doGet`/`doPost`).

    Q482. What is Chain of Responsibility Pattern?

    Avoids coupling the sender of a request to its receiver by giving more than one object a chance to handle the request. (e.g., Servlet Filters, Exception Handling).

    Q483. What is Command Pattern?

    Encapsulates a request as an object, thereby letting you parameterize clients with different requests, queue or log requests, and support undoable operations. (e.g., Runnable).

    Q484. What is Iterator Pattern?

    Provides a way to access the elements of an aggregate object sequentially without exposing its underlying representation. (e.g., `java.util.Iterator`).

    Q485. What is Composite Pattern?

    Composes objects into tree structures to represent part-whole hierarchies. Lets clients treat individual objects and compositions of objects uniformly. (e.g., UI Components).

    Q486. What is Flyweight Pattern?

    Uses sharing to support large numbers of fine-grained objects efficiently. (e.g., `String` Pool, `Integer.valueOf` cache).

    Q487. What is State Pattern?

    Allows an object to alter its behavior when its internal state changes. The object will appear to change its class.

    Q488. What is Mediator Pattern?

    Defines an object that encapsulates how a set of objects interact. Promotes loose coupling by keeping objects from referring to each other explicitly.

    Q489. What is Memento Pattern?

    Captures and externalizes an object's internal state so that the object can be restored to this state later. (e.g., Undo mechanism).

    Q490. What is Visitor Pattern?

    Represents an operation to be performed on the elements of an object structure. Lets you define a new operation without changing the classes of the elements on which it operates.

    Q491. What is Prototype Pattern?

    Specifies the kinds of objects to create using a prototypical instance, and create new objects by copying this prototype. (e.g., `Cloneable`).

    Q492. What is Bridge Pattern?

    Decouples an abstraction from its implementation so that the two can vary independently. (e.g., JDBC Drivers).

    Q493. What is Interpreter Pattern?

    Given a language, defines a representation for its grammar along with an interpreter that uses the representation to interpret sentences in the language. (e.g., `java.util.regex.Pattern`).

    Q494. What is Filter Pattern (Criteria Pattern)?

    Enables developers to filter a set of objects using different criteria and chaining them in a decoupled way through logical operations.

    Q495. What is Null Object Pattern?

    Replaces check of null object instance. Instead of putting `if (obj != null)`, you provide a default object that does nothing.

    Q496. What is MVC Pattern?

    Model-View-Controller. Separates application into three interconnected parts.

    • Model: Data and logic.
    • View: UI.
    • Controller: Handles input and updates Model/View.

    Q497. What is DAO Pattern?

    Data Access Object. Abstracts and encapsulates all access to the data source. The DAO manages the connection with the data source to obtain and store data.

    Q498. What is DTO Pattern?

    Data Transfer Object. An object that carries data between processes. The motivation is to reduce the number of method calls in a distributed system.

    Q499. What is Front Controller Pattern?

    A controller that handles all requests for a website. (e.g., `DispatcherServlet` in Spring MVC).

    Q500. What is Service Locator Pattern?

    Used to locate various services using JNDI lookup. Caches the services for performance.
