Beginner Interview Questions (Q1 - Q50)

  • Home
  • Interview
  • Beginner

This section covers the fundamental concepts of Java, including JVM architecture, variables, data types, and basic OOP principles. Perfect for freshers and those brushing up on basics.

Q1. What is Java? ðŸ”¥ Most Asked

Java is a high-level, object-oriented, robust, secure, and platform-independent programming language. It was developed by Sun Microsystems (now Oracle) in 1995. Its core philosophy is "Write Once, Run Anywhere" (WORA).

Q2. What are the main features of Java?

The key features of Java are:

  • Simple: Easy to learn, no pointers.
  • Object-Oriented: Everything is an object.
  • Platform Independent: Bytecode runs on any JVM.
  • Robust: Strong memory management, exception handling.
  • Secure: No explicit pointers, runs inside sandbox.
  • Multithreaded: Supports concurrent execution.

Q3. What is the difference between JDK, JRE, and JVM? ðŸ”¥ Most Asked

  • JVM (Java Virtual Machine): An abstract machine that executes Java bytecode. It is platform-dependent.
  • JRE (Java Runtime Environment): Contains JVM + Class Libraries. It is used to run Java applications.
  • JDK (Java Development Kit): Contains JRE + Development Tools (javac, debugger). It is used to develop Java applications.

Q4. Why is Java platform independent?

Java is platform independent because the compiler (`javac`) converts source code into bytecode (`.class` file), which is not specific to any machine. This bytecode can be executed by the JVM on any operating system (Windows, Mac, Linux), making the code portable.

Q5. Is JVM platform independent?

No, the JVM is platform-dependent. You need a different JVM for Windows, Linux, and macOS because it acts as an interface between the bytecode and the specific underlying hardware/OS.

Q6. What is the JIT compiler?

JIT (Just-In-Time) compiler is a component of the JVM that improves performance. It compiles bytecode into native machine code at runtime for frequently executed methods ("hotspots"), so the JVM doesn't have to interpret them every time.

Q7. What are the memory areas in JVM?

  • Heap Area: Stores objects and instance variables.
  • Stack Area: Stores method calls, local variables, and reference variables.
  • Method Area: Stores class structures (metadata), static variables, constants.
  • PC Register: Stores the address of the current instruction.
  • Native Method Stack: Stores native method information.

Q8. What is a ClassLoader?

The ClassLoader is a subsystem of JVM responsible for loading class files (`.class`) into memory dynamically when needed. There are three built-in class loaders: Bootstrap, Extension (Platform), and System (Application).

Q9. What is the difference between C++ and Java?

  • Platform: C++ is platform-dependent; Java is platform-independent.
  • Pointers: C++ supports pointers; Java does not (for security).
  • Inheritance: C++ supports multiple inheritance; Java does not (supports via Interfaces).
  • Memory: C++ requires manual memory management; Java has automatic Garbage Collection.

Q10. What is `public static void main(String[] args)`? ðŸ”¥ Most Asked

It is the entry point of any Java application.

  • public: Accessible by JVM from anywhere.
  • static: Can be called without creating an object of the class.
  • void: Returns nothing.
  • main: Name of the method JVM looks for.
  • String[] args: Accepts command-line arguments.

Q11. Can we execute a Java program without a main method?

Prior to JDK 7, it was possible using a static block. However, from JDK 7 onwards, the JVM explicitly checks for the `main` method before initializing the class, so it is not possible to execute a standard application without `main`.

Q12. What happens if we declare main method as private?

The code will compile successfully, but it will throw a runtime error: `Main method not found in class...`, because the JVM cannot access a private method.

Q13. Can we overload the main method?

Yes, we can overload the main method by defining other methods named `main` with different parameters. However, the JVM will always call the one with `String[] args` signature as the entry point.

Q14. What are Java identifiers?

Identifiers are names given to classes, methods, variables, and packages. They must start with a letter, `$`, or `_`, and are case-sensitive. They cannot be keywords.

Q15. What are the primitive data types in Java?

There are 8 primitive data types:

  • byte (8-bit)
  • short (16-bit)
  • int (32-bit)
  • long (64-bit)
  • float (32-bit)
  • double (64-bit)
  • char (16-bit, Unicode)
  • boolean (true/false)
int age = 25;
double price = 99.99;
boolean isJavaFun = true;
char grade = 'A';

Q16. What is the default value of local variables? ðŸ”¥ Most Asked

Local variables (declared inside methods) do not have a default value. They must be initialized before use, otherwise, the compiler will throw an error (`variable might not have been initialized`).

Q17. What is the difference between instance and static variables?

  • Instance Variable: Belongs to an object. Each object has its own copy. Created when object is created.
  • Static Variable: Belongs to the class. Shared among all objects. Created when class is loaded.

Q18. What is type casting?

Converting one data type into another.

  • Implicit (Widening): Automatic, smaller to larger type (e.g., `int` to `double`).
  • Explicit (Narrowing): Manual, larger to smaller type (e.g., `double` to `int`), may lose data.
// Implicit
int num = 10;
double d = num; // 10.0

// Explicit
double pi = 3.14;
int p = (int) pi; // 3

Q19. What is a wrapper class?

A class that wraps a primitive data type into an object. Examples: `Integer` for `int`, `Double` for `double`. They are used in Collections (which store objects only) and provide utility methods.

Q20. What is Autoboxing and Unboxing?

  • Autoboxing: Automatic conversion of primitive to wrapper object (e.g., `int` to `Integer`).
  • Unboxing: Automatic conversion of wrapper object to primitive (e.g., `Integer` to `int`).

Q21. What is the difference between `==` and `equals()`? ðŸ”¥ Most Asked

  • `==` Operator: Compares references (memory addresses) for objects, and values for primitives.
  • `equals()` Method: Compares content (values) of objects (if overridden). Default implementation in `Object` class behaves like `==`.

Q22. Why are Strings immutable in Java?

String immutability ensures:

  • Security: Strings are used for passwords, URLs, db connections. Changing them could compromise security.
  • Thread Safety: Immutable objects are inherently thread-safe.
  • String Constant Pool: Saves memory by sharing identical strings.

Q23. What is the String Constant Pool?

A special memory area in the Heap where String literals are stored. If a string literal is created that already exists in the pool, the reference to the existing string is returned instead of creating a new object.

Q24. What is the difference between String, StringBuffer, and StringBuilder?

  • String: Immutable, slow for concatenation.
  • StringBuffer: Mutable, synchronized (thread-safe), slower than StringBuilder.
  • StringBuilder: Mutable, not synchronized (not thread-safe), fastest.

Q25. What is an Array?

An object that stores a fixed-size collection of elements of the same type. Arrays are index-based (starting from 0) and stored in contiguous memory locations. 

int[] numbers = {1, 2, 3, 4, 5};
String[] names = new String[5];
names[0] = "Alice";
Q26. How do you find the length of an array vs a string?

  • Array: Use the `.length` property (e.g., `arr.length`).
  • String: Use the `.length()` method (e.g., `str.length()`).

Q27. What is a multi-dimensional array?

An array of arrays. The most common is a 2D array (matrix). Example: `int[][] matrix = new int[3][3];`.

Q28. What is Object-Oriented Programming (OOP)?

A programming paradigm based on "objects" containing data (fields) and code (methods). The four main pillars are:

  • Encapsulation
  • Inheritance
  • Polymorphism
  • Abstraction

Q29. What is a Class and an Object?

  • Class: A blueprint or template that defines the properties and behaviors.
  • Object: An instance of a class. It has state (variables) and behavior (methods).

Q30. What is a Constructor?

A special method used to initialize objects. It has the same name as the class and no return type. It is called automatically when an object is created.

class Car {
    Car() {
        System.out.println("Car created");
    }
}
Car myCar = new Car(); // Output: Car created
Q31. What are the types of constructors?

  • Default Constructor: No-args constructor created by compiler if none is defined.
  • No-Arg Constructor: Explicitly defined with no arguments.
  • Parameterized Constructor: Accepts arguments to initialize fields with specific values.

Q32. Can a constructor be private?

Yes. If a constructor is private, you cannot create an instance of the class from outside. This is used in the Singleton Design Pattern.

Q33. What is the `this` keyword?

`this` is a reference variable that refers to the current object. It is used to resolve naming conflicts between instance variables and parameters, or to call another constructor (`this()`).

Q34. What is Inheritance?

A mechanism where one class (Child/Subclass) acquires the properties and behaviors of another class (Parent/Superclass). It promotes code reusability. Used via `extends` keyword.

Q35. Does Java support multiple inheritance? ðŸ”¥ Most Asked

Java does not support multiple inheritance with classes (to avoid the "Diamond Problem"). However, it supports multiple inheritance through Interfaces.

Q36. What is the `super` keyword?

`super` is a reference variable used to refer to the immediate parent class object. It is used to access parent class methods, variables, or constructors (`super()`).

Q37. What is Polymorphism?

Polymorphism means "many forms". It allows a single action to behave differently based on the object performing it.

  • Compile-time (Static): Method Overloading.
  • Run-time (Dynamic): Method Overriding.

Q38. What is Method Overloading?

Defining multiple methods in the same class with the same name but different parameters (type, number, or order). Return type doesn't matter.

Q39. What is Method Overriding?

Defining a method in the child class that already exists in the parent class with the same signature (name, parameters, return type). Used to provide specific implementation.

Q40. Can we override static methods?

No. Static methods belong to the class, not the object. If you define a static method with the same signature in the child class, it is called Method Hiding, not overriding.

Q41. What is Encapsulation?

Encapsulation is wrapping data (variables) and code (methods) together into a single unit (class). It is achieved by making variables private and providing public getter and setter methods.

Q42. What is Abstraction?

Abstraction is hiding internal implementation details and showing only the functionality to the user. It is achieved using Abstract Classes (0-100% abstraction) and Interfaces (100% abstraction).

Q43. What is an Abstract Class?

A class declared with `abstract` keyword. It can have both abstract (no body) and non-abstract methods. It cannot be instantiated.

Q44. What is an Interface?

A blueprint of a class that contains static constants and abstract methods (before Java 8). It supports multiple inheritance and 100% abstraction. From Java 8, it can have default and static methods.

Q45. Difference between Abstract Class and Interface? ðŸ”¥ Most Asked

  • Abstract Class: Can have state (instance variables), constructors, and non-abstract methods. Used for "is-a" relationship.
  • Interface: Cannot have state (only constants), no constructors. Used for "can-do" capability and multiple inheritance.

Q46. What is the `final` keyword?

  • Variable: Value cannot be changed (constant).
  • Method: Cannot be overridden.
  • Class: Cannot be inherited (e.g., String class).

Q47. What is the `static` keyword?

Used to manage memory. It belongs to the class rather than an instance. We can have static variables, methods, blocks, and nested classes.

Q48. What is a Package?

A namespace that organizes a set of related classes and interfaces. It prevents naming conflicts and controls access. Built-in packages include `java.lang`, `java.util`, etc.

Q49. What are Access Modifiers?

They control the visibility of classes, methods, and variables.

  • private: Within class only.
  • default: Within package only.
  • protected: Within package + subclasses outside package.
  • public: Everywhere.

Q50. What is the `instanceof` operator?

It is used to test whether an object is an instance of a specific class or interface. It returns `true` or `false`. Example: `if (s instanceof String) ...`

Q51. Is Java Pass-by-Value or Pass-by-Reference? ðŸ”¥ Most Asked

Java is strictly Pass-by-Value.

  • For primitives, the actual value is passed.
  • For objects, the value of the reference (memory address) is passed, not the object itself.

Q52. What is the `Object` class?

The `Object` class is the root class of the Java class hierarchy. Every class in Java directly or indirectly inherits from the `Object` class.

Q53. What are the important methods of the Object class?

  • `toString()`: Returns string representation.
  • `equals(Object obj)`: Compares objects.
  • `hashCode()`: Returns hash code.
  • `getClass()`: Returns runtime class.
  • `clone()`: Creates object copy.
  • `finalize()`: Called by GC (deprecated).
  • `wait()`, `notify()`, `notifyAll()`: For threading.

Q54. What is the purpose of `toString()` method?

It returns a string representation of the object. By default, it returns `ClassName@HashCode`. We override it to provide a meaningful string representation (e.g., printing object state).

Q55. What is the contract between `equals()` and `hashCode()`?

  • If two objects are equal according to `equals()`, they must have the same `hashCode()`.
  • If two objects have the same `hashCode()`, they are not necessarily equal.

Q56. What is a Marker Interface?

An interface with no methods or constants. It provides run-time type information to the JVM. Examples: `Serializable`, `Cloneable`, `Remote`.

Q57. What is the difference between `break` and `continue`?

  • break: Terminates the loop or switch statement immediately.
  • continue: Skips the current iteration and jumps to the next iteration of the loop.

Q58. What is the `return` keyword?

It is used to exit from a method, with or without a value. If the method has a return type, it must return a value of that type.

Q59. What is an Exception?

An event that disrupts the normal flow of the program's execution. It is an object thrown at runtime.

Q60. What is the difference between Error and Exception?

  • Error: Serious problems that a reasonable application should not try to catch (e.g., `OutOfMemoryError`, `StackOverflowError`). Usually caused by the environment.
  • Exception: Conditions that a reasonable application might want to catch (e.g., `NullPointerException`, `IOException`).

Q61. What are Checked and Unchecked Exceptions? ðŸ”¥ Most Asked

  • Checked: Checked at compile-time. Must be handled using try-catch or throws (e.g., `IOException`, `SQLException`).
  • Unchecked (Runtime): Checked at runtime. Handling is optional (e.g., `NullPointerException`, `ArrayIndexOutOfBoundsException`).

Q62. What are the keywords used in Exception Handling?

  • try: Block of code to monitor for exceptions.
  • catch: Block to handle the exception.
  • finally: Block that always executes (cleanup code).
  • throw: Used to throw an exception explicitly.
  • throws: Declares exceptions a method can throw.

Q63. Can we have a try block without catch?

Yes, but it must be followed by a `finally` block. A `try` block cannot exist alone.

Q64. What is the difference between `throw` and `throws`?

  • throw: Used inside a method to throw an exception object explicitly.
  • throws: Used in method signature to declare that the method might throw exceptions.

Q65. What is `finally` block?

A block that executes regardless of whether an exception is handled or not. It is typically used for resource cleanup (closing files, DB connections).

Q66. Does `finally` block always execute? ðŸ”¥ Most Asked

Yes, except in a few cases:

  • If `System.exit()` is called.
  • If the JVM crashes.
  • If the thread executing the code is killed.

Q67. What is a Collection in Java?

A framework that provides an architecture to store and manipulate a group of objects. It includes interfaces (List, Set, Map) and classes (ArrayList, HashSet, HashMap).

Q68. What is the difference between Array and ArrayList?

  • Array: Fixed size, supports primitives and objects, faster.
  • ArrayList: Dynamic size, supports only objects, slower (due to resizing).

Q69. What is the difference between List and Set?

  • List: Ordered collection, allows duplicates (e.g., ArrayList, LinkedList).
  • Set: Unordered collection, does not allow duplicates (e.g., HashSet, TreeSet).

Q70. What is a Map?

An object that maps keys to values. It cannot contain duplicate keys. Each key can map to at most one value. It is not a subtype of the `Collection` interface.

Q71. What is `static` block?

A block of code used to initialize static variables. It is executed once when the class is loaded into memory, even before the main method.

Q72. Can we execute code before main method?

Yes, using a static block. The static block executes when the class is loaded, which happens before the JVM calls the `main` method.

Q73. What is an Inner Class?

A class defined inside another class. It can access private members of the outer class. Used to logically group classes used in one place.

Q74. What is an Anonymous Inner Class?

An inner class without a name, used to instantiate an object with certain "extras" (like overriding methods) in a single step. Often used for event handling or listeners.

Q75. What is the `final` class?

A class that cannot be subclassed (inherited). Example: `java.lang.String`, `java.lang.Math`. This is done for security and immutability.

Q76. What is the `transient` keyword?

Used in serialization. If a variable is declared as `transient`, it is not serialized (its value is not saved). During deserialization, it gets the default value.

Q77. What is the `volatile` keyword?

Indicates that a variable's value will be modified by different threads. It ensures that the value is always read from the main memory, not from the thread's local cache, guaranteeing visibility.

Q78. What is Serialization?

The process of converting the state of an object into a byte stream to save it to a file or send it over a network. The reverse process is called Deserialization.

Q79. How do you make a class Serializable?

By implementing the `java.io.Serializable` interface. It is a marker interface (no methods).

Q80. What is Garbage Collection (GC)?

The process of automatically reclaiming memory occupied by objects that are no longer reachable (referenced) in the program. It prevents memory leaks.

Q81. Can we force Garbage Collection?

No. We can request it using `System.gc()` or `Runtime.getRuntime().gc()`, but the JVM does not guarantee that GC will run immediately.

Q82. What is the `finalize()` method?

A method in the `Object` class called by the Garbage Collector before an object is destroyed. It is used for cleanup code. (Note: Deprecated in Java 9).

Q83. What is a Constructor Chaining?

Calling one constructor from another constructor within the same class (using `this()`) or from the parent class (using `super()`).

Q84. What is the default value of a boolean?

`false`.

Q85. What is the default value of an object reference?

`null`.

Q86. What is a Literal?

A source code representation of a fixed value. Examples: `100` (int literal), `'A'` (char literal), `"Hello"` (String literal), `true` (boolean literal).

Q87. What is Type Promotion?

When evaluating an expression, Java automatically promotes smaller data types to larger ones (e.g., `byte`, `short`, `char` are promoted to `int`).

Q88. What is the difference between `while` and `do-while` loop?

  • while: Entry-controlled. Condition checked before execution. May run 0 times.
  • do-while: Exit-controlled. Condition checked after execution. Runs at least once.
// while
while(x > 0) { x--; }

// do-while
do {
    System.out.println("Runs at least once");
} while(false);

Q89. What is an Enhanced for-loop (for-each)?

A simplified loop used to iterate over arrays or collections. Syntax: `for (Type var : array) { ... }`. It is cleaner but cannot access the index.

Q90. What is a Switch statement?

A control flow statement that executes one block of code from multiple options based on a variable's value. Supports `int`, `char`, `String`, `enum`.

int day = 3;
switch(day) {
    case 1: System.out.println("Monday"); break;
    case 2: System.out.println("Tuesday"); break;
    default: System.out.println("Other");
}
Q91. What is the `default` case in switch?

The block of code executed if none of the `case` values match the switch expression. It is optional.

Q92. Can we use Strings in switch case?

Yes, starting from Java 7, we can use String objects in switch statements.

Q93. What is the `Math` class?

A utility class in `java.lang` containing static methods for mathematical operations like `Math.max()`, `Math.sqrt()`, `Math.random()`, etc.

Q94. How to generate a random number?

Using `Math.random()` (returns double between 0.0 and 1.0) or `java.util.Random` class.

Q95. What is a recursive method?

A method that calls itself. It must have a base condition to stop the recursion, otherwise it leads to `StackOverflowError`.

Q96. What is the difference between `void` and `null`?

  • void: A return type indicating the method returns nothing.
  • null: A literal representing the absence of an object reference.

Q97. What is Command Line Argument?

Arguments passed to the `main` method at the time of running the application. They are stored in the `String[] args` array.

Q98. What is the `Scanner` class?

A class in `java.util` used to parse primitive types and strings using regular expressions. Commonly used for reading input from the console (`System.in`).

Q99. What is `System.out.println()`?

  • System: A class in `java.lang`.
  • out: A static final variable (PrintStream) in System class.
  • println(): A method of PrintStream class to print text with a newline.

Q100. What is the difference between `print()` and `println()`?

`print()` outputs text and keeps the cursor on the same line. `println()` outputs text and moves the cursor to the next line.

Q101. What is an Array?

An object that stores a collection of elements of the same type. It has a fixed size.

int[] numbers = {1, 2, 3, 4, 5};
String[] names = new String[5];
Q102. What is ArrayIndexOutOfBoundsException?

A runtime exception thrown when trying to access an array index that is negative or greater than or equal to the array size.

int[] arr = {1, 2, 3};
System.out.println(arr[5]); // Throws Exception
Q103. Can we change the size of an array after creation?

No, arrays have a fixed size once initialized. To have a dynamic size, use ArrayList.

Q104. What is a Multi-dimensional Array?

An array of arrays. The most common is a 2D array (matrix).

int[][] matrix = {
    {1, 2, 3},
    {4, 5, 6}
};
Q105. What is a String in Java?

An object that represents a sequence of characters. It is immutable and stored in the String Constant Pool (if created as a literal).

Q106. Why is String immutable?

For security, synchronization, caching (hashcode), and performance (String Pool).

Q107. What is the String Pool?

A special memory region in the Heap where String literals are stored. If a string already exists, the reference is returned instead of creating a new object.

String s1 = "Hello";
String s2 = "Hello";
// s1 == s2 is true (same reference)
Q108. Difference between String, StringBuilder, and StringBuffer?
  • String: Immutable.
  • StringBuilder: Mutable, not thread-safe (Fast).
  • StringBuffer: Mutable, thread-safe (Slow).
Q109. How to compare two Strings?

Use .equals() for content comparison. Do NOT use == as it checks reference equality.

String s1 = new String("Hello");
String s2 = new String("Hello");
System.out.println(s1 == s2);       // false
System.out.println(s1.equals(s2));  // true
Q110. What is the `toString()` method?

A method in Object class that returns the string representation of an object. Default implementation returns ClassName@HashCode.

Q111. What is a Wrapper Class?

A class that wraps a primitive data type into an object. Example: Integer for int, Double for double.

Q112. What is Autoboxing and Unboxing?

Autoboxing: Automatic conversion of primitive to wrapper (e.g., int to Integer).

Unboxing: Automatic conversion of wrapper to primitive.

Integer num = 10; // Autoboxing
int n = num;      // Unboxing
Q113. What is the `super` keyword?

A reference variable used to refer to the immediate parent class object. Used to call parent constructor or methods.

super(); // Calls parent constructor
super.display(); // Calls parent method
Q114. What is the `this` keyword?

A reference variable that refers to the current object.

this.x = x; // Distinguishes instance variable from parameter
Q115. Can `this` be used in a static context?

No, because static methods belong to the class, not an instance. this refers to an instance.

Q116. What is Method Overloading?

Multiple methods with the same name but different parameters (compile-time polymorphism).

void add(int a, int b) { ... }
void add(double a, double b) { ... }
Q117. What is Method Overriding?

Providing a specific implementation in the subclass for a method already defined in the parent class (runtime polymorphism).

Q118. Can we override a static method?

No. If a subclass defines a static method with the same signature, it is called Method Hiding, not overriding.

Q119. Can we override a private method?

No, private methods are not visible to subclasses, so they cannot be overridden.

Q120. What is Covariant Return Type?

Since Java 5, an overriding method can return a subtype of the return type declared in the parent class method.

class Parent { Object get() { ... } }
class Child extends Parent { String get() { ... } } // Valid
Q121. What is an Abstract Class?

A class declared with `abstract` keyword. It cannot be instantiated and can have both abstract (no body) and concrete methods.

Q122. What is an Interface?

A blueprint of a class that contains only abstract methods (before Java 8) and constants. It supports multiple inheritance.

Q123. Difference between Abstract Class and Interface?
  • Abstract Class: Can have state (fields), constructors, and non-abstract methods.
  • Interface: Only constants (public static final), no constructors.
Q124. Can an interface have a constructor?

No, because interfaces cannot be instantiated.

Q125. Can an interface have fields?

Yes, but they are implicitly public static final (constants).

Q126. What are Default Methods in Interface?

Methods in an interface with a body, marked with the default keyword. Introduced in Java 8 to allow adding new methods without breaking existing implementations.

interface MyInterface {
    default void show() {
        System.out.println("Default Method");
    }
}
Q127. What are Static Methods in Interface?

Static methods in interfaces (Java 8+) are utility methods that belong to the interface, not the implementation class.

interface MathUtil {
    static int add(int a, int b) {
        return a + b;
    }
}
Q128. What is Multiple Inheritance? Does Java support it?

Inheriting from more than one class. Java does not support multiple inheritance with classes to avoid ambiguity (Diamond Problem), but supports it through interfaces.

Q129. What is the Diamond Problem?

Ambiguity that arises when two parent classes have a method with the same name, and a child class inherits from both. Java avoids this by not allowing multiple class inheritance.

Q130. What is a Package?

A namespace that organizes a set of related classes and interfaces. It helps avoid name collisions and provides access control.

Q131. What are Access Modifiers?

Keywords that set the accessibility of classes, methods, and variables.

  • public: Accessible everywhere.
  • protected: Accessible in same package and subclasses.
  • default (no keyword): Accessible only in same package.
  • private: Accessible only in same class.
Q132. What is the default access modifier?

If no modifier is specified, it is package-private (default). Accessible only within the same package.

Q133. What is `protected` access modifier?

Accessible within the same package and by subclasses in other packages.

Q134. What is Encapsulation?

Wrapping data (variables) and code (methods) together as a single unit. It restricts direct access to some components.

Q135. How to achieve Encapsulation?

By declaring variables as private and providing public getter and setter methods. 

class Person {
    private String name;
    public String getName() { return name; }
    public void setName(String name) { this.name = name; }
}
Q136. What is Polymorphism?

The ability of an object to take on many forms. "Poly" means many, "morph" means forms.

Q137. What is Compile-time Polymorphism?

Achieved through Method Overloading. The compiler determines which method to call based on method signature.

Q138. What is Runtime Polymorphism?

Achieved through Method Overriding. The JVM determines which method to call at runtime based on the actual object type.

Q139. What is Dynamic Method Dispatch?

The mechanism by which a call to an overridden method is resolved at runtime rather than compile-time.

Parent obj = new Child();
obj.show(); // Calls Child's show()
Q140. What is `instanceof` operator?

Used to test if an object is an instance of a specific class or interface.

if (obj instanceof String) { ... }
Q141. What is Casting? (Upcasting vs Downcasting)

Upcasting: Casting a subclass object to a superclass reference (Implicit).

Downcasting: Casting a superclass reference to a subclass object (Explicit). 

Parent p = new Child(); // Upcasting
Child c = (Child) p;    // Downcasting
Q142. What is ClassCastException?

Thrown when trying to cast an object to a subclass of which it is not an instance.

Q143. What is Object Cloning?

Creating an exact copy of an object using the clone() method.

Q144. What is Deep Copy vs Shallow Copy?
  • Shallow Copy: Copies references. Changes to the new object affect the old one.
  • Deep Copy: Copies values (creates new objects). Changes do not affect the original.
Q145. What is the `Cloneable` interface?

A marker interface that indicates a class allows its objects to be cloned. If not implemented, clone() throws CloneNotSupportedException.

Q146. What is Singleton Pattern?

A design pattern that ensures a class has only one instance and provides a global point of access to it.

Q147. How to create a Singleton class?

Make the constructor private and provide a public static method to return the instance.

class Singleton {
    private static Singleton instance = new Singleton();
    private Singleton() {}
    public static Singleton getInstance() { return instance; }
}
Q148. Difference between `==` and `.equals()`?
  • == compares references (memory address).
  • .equals() compares content (if overridden).
Q149. What is the `final` keyword?
  • Variable: Constant (cannot be changed).
  • Method: Cannot be overridden.
  • Class: Cannot be inherited.
Q150. What is the difference between `throw` and `throws`?
  • throw: Used to explicitly throw an exception.
  • throws: Used in method signature to declare exceptions.
void method() throws Exception {
    throw new Exception("Error");
}
