üßµ Introduction to Strings

Level 1: Beginner-Friendly Explanation (Like Explaining to a Baby)

Imagine you have a label maker. You type a name like "Alice" and print a sticker. Once that sticker is printed, you can't change the letters on it! If you want "Alicia", you have to print a whole new sticker.

A String in Java is exactly like that sticker. It's a sequence of characters (letters, numbers, symbols) used to make words or sentences. And just like the sticker, once you create it, you can't change it (we call this immutable).

Level 2: Real-World Analogy

Think of a String like a printed book page.

  • Once the ink is dry, you can't rearrange the letters on that page.
  • If you want to correct a typo, you have to print a completely new page.
  • This is different from a whiteboard (StringBuilder), where you can erase and rewrite easily.

Level 3: Technical Explanation

In Java, String is a class in the java.lang package. It represents a sequence of characters.

  • Object, not Primitive: Unlike int or char, String is an object.
  • Immutable: The internal state of a String object cannot be modified after creation.
  • Storage: Strings are stored in a special memory area called the String Constant Pool (inside the Heap) to save memory.
  • Final: The String class is final, meaning it cannot be subclassed.

üèä String Pool and Memory Management

The String Constant Pool

Java uses a special memory optimization called the String Pool. When you create a String literal, Java checks if that text already exists in the pool.

  • If it exists, Java returns a reference to the existing one (saving memory!).
  • If it doesn't exist, Java creates a new one in the pool.
üí° EXAMPLE: Literal vs New Keyword

String s1 = "Java";              // Goes to String Pool
String s2 = "Java";              // Refers to same object in Pool
String s3 = new String("Java");  // Forces creation of NEW object in Heap (outside pool)

System.out.println(s1 == s2);    // true (Same reference)
System.out.println(s1 == s3);    // false (Different references)
üìù TIP - intern() Method

You can move a String created with new into the pool using the intern() method.
s3.intern() would return the reference from the pool (same as s1).

üîí String Immutability

Immutability means "unchangeable". But why did Java make Strings immutable?

Why Immutable?

  1. Security: Strings are used for passwords, database URLs, and file paths. If they were mutable, a hacker could change a file path after security checks passed.
  2. Thread Safety: Since they can't change, multiple threads can safely share the same String object without synchronization.
  3. String Pool: Pooling is only possible because Strings are immutable. If one reference changed the content, it would affect all other references pointing to the same text!
‚ö†Ô∏è WARNING - Memory Waste

Because Strings are immutable, every time you modify one (e.g., s = s + " World"), a NEW object is created. Doing this in a loop can create thousands of garbage objects!

‚öîÔ∏è String vs StringBuilder vs StringBuffer

Feature String StringBuilder StringBuffer
Mutability Immutable (Cannot change) Mutable (Can change) Mutable (Can change)
Thread Safety Yes (Implicitly) No (Not synchronized) Yes (Synchronized)
Performance Slow for modifications Fastest Slower than StringBuilder

‚ûï String Concatenation

You can join strings in multiple ways:

1. Using + Operator

String s = "Hello" + " " + "World"; // "Hello World"

Note: The compiler internally converts this to StringBuilder for efficiency in simple cases.

2. Using concat() Method

String s = "Hello".concat(" World");

3. Using String.join() (Java 8+)

String s = String.join("-", "2023", "10", "15"); // "2023-10-15"

üöÄ String with Loops & Performance

‚ö†Ô∏è BAD PRACTICE

// NEVER DO THIS!
String s = "";
for (int i = 0; i < 1000; i++) {
    s = s + i; // Creates 1000 temporary String objects!
}
‚úÖ BEST PRACTICE

// DO THIS INSTEAD
StringBuilder sb = new StringBuilder();
for (int i = 0; i < 1000; i++) {
    sb.append(i); // Modifies the same object. Fast!
}
String s = sb.toString();

üß© String and Regex

Strings have built-in support for Regular Expressions (Regex).

  • matches(regex): Checks if string matches pattern.
  • replaceAll(regex, replacement): Replaces matching parts.
  • split(regex): Splits string by pattern.

String email = "test@example.com";
boolean isValid = email.matches("^[\\w-\\.]+@([\\w-]+\\.)+[\\w-]{2,4}$");

String text = "apple,banana,orange";
String[] fruits = text.split(","); // ["apple", "banana", "orange"]

‚ùå Common Mistakes

  • Using == instead of equals(): == compares references (memory address), equals() compares actual text.
  • NullPointerException: Calling methods on a null String. Always check for null!
  • Modifying Strings in a loop: As mentioned, use StringBuilder.

‚úÖ Best Practices

  • Use StringBuilder for dynamic string manipulation.
  • Always use .equals() for comparison.
  • Check for null or use "Constant".equals(variable) to avoid NPE.
  • Use isEmpty() or isBlank() (Java 11+) to check for empty strings.
