Control Flow Statements & Loops

üéØ What are Control Flow Statements?

Level 1: Beginner-Friendly Explanation (Like Explaining to a Baby)

Imagine you're playing a video game. You don't just walk straight forward forever, right? Sometimes you need to:

  • Make decisions: "If I see a monster, run away. Otherwise, keep walking."
  • Repeat actions: "Jump 5 times to reach the platform."
  • Skip things: "If I already have this item, don't pick it up again."

Control flow statements are like the rules that tell your program what to do next. They help your code make decisions, repeat tasks, and skip unnecessary steps - just like you do in real life!

üìù NOTE

Without control flow, programs would just execute line by line from top to bottom, like reading a book. Control flow lets programs be SMART and make choices!

Level 2: Real-World Analogy

Think of control flow like traffic signals and road signs:

  • Traffic Light (if-else): If light is green, GO. If light is red, STOP.
  • Roundabout (loop): Keep going around until you find your exit.
  • Detour Sign (break): Stop your current route and take a different path.
  • Speed Bump (continue): Slow down for this one, then keep going.

Just like traffic rules control how cars move, control flow statements control how your program executes!

Level 3: Technical Deep-Dive

Control flow statements alter the sequential execution of a program. By default, Java executes statements from top to bottom, one after another. Control flow mechanisms allow:

  • Conditional Execution: Execute code only if certain conditions are met
  • Iterative Execution: Repeat code blocks multiple times
  • Branching: Jump to different parts of code or exit loops early
üí° EXAMPLE - Real-World Usage

Login System: "If password is correct, grant access. Else, show error and allow 3 attempts (loop). If 3 failed attempts, lock account (break)."

Why Are They Essential?

  • Decision Making: Programs need to respond differently based on input
  • Automation: Repeat tasks without writing duplicate code
  • Efficiency: Skip unnecessary operations
  • User Interaction: Respond to user choices dynamically

üîÄ Decision Making Statements

Level 1: Beginner-Friendly Explanation

Decision making is like choosing what to wear based on the weather:

  • "If it's raining, take an umbrella."
  • "If it's sunny, wear sunglasses. Otherwise, wear a jacket."
  • "If it's freezing, wear a heavy coat. If it's cold, wear a sweater. If it's warm, wear a t-shirt."

In Java, we use if, else, and switch to make these kinds of decisions!

Level 2: Real-World Analogy

Think of decision statements like a restaurant menu:

  • if: "If you're vegetarian, here are your options."
  • if-else: "If you want spicy, order this. Otherwise, order that."
  • if-else-if ladder: "If you want breakfast, see page 1. If lunch, see page 2. If dinner, see page 3."
  • switch-case: "Pick a number from 1-5, and I'll tell you what meal that is."

Level 3: Technical Breakdown

1. if Statement

Executes a block of code ONLY if the condition is true. 

if (condition) {
    // Code executes only if condition is true
}

// Example:
int age = 18;
if (age >= 18) {
    System.out.println("You can vote!");
}

2. if-else Statement

Provides an alternative path when the condition is false. 

if (condition) {
    // Executes if condition is true
} else {
    // Executes if condition is false
}

// Example:
int marks = 65;
if (marks >= 60) {
    System.out.println("Pass");
} else {
    System.out.println("Fail");
}

3. if-else-if Ladder

Tests multiple conditions in sequence. First true condition executes. 

if (condition1) {
    // Executes if condition1 is true
} else if (condition2) {
    // Executes if condition2 is true
} else if (condition3) {
    // Executes if condition3 is true
} else {
    // Executes if all conditions are false
}

// Example: Grade Calculator
int score = 85;
if (score >= 90) {
    System.out.println("Grade: A");
} else if (score >= 80) {
    System.out.println("Grade: B");
} else if (score >= 70) {
    System.out.println("Grade: C");
} else if (score >= 60) {
    System.out.println("Grade: D");
} else {
    System.out.println("Grade: F");
}

4. switch-case Statement

Efficient for checking a variable against multiple constant values. 

switch (variable) {
    case value1:
        // Code for value1
        break;
    case value2:
        // Code for value2
        break;
    default:
        // Code if no case matches
}

// Example: Day of Week
int day = 3;
switch (day) {
    case 1:
        System.out.println("Monday");
        break;
    case 2:
        System.out.println("Tuesday");
        break;
    case 3:
        System.out.println("Wednesday");
        break;
    default:
        System.out.println("Invalid day");
}
‚ö†Ô∏è WARNING - Missing break

Without break, execution "falls through" to the next case! This is usually a bug.

Example: If day = 1 and you forget break, it will print "Monday", "Tuesday", "Wednesday", etc.

üí° TIP - When to Use What

Use if-else: For boolean conditions or range checks (age > 18, score >= 60)
Use switch-case: For checking exact constant values (day of week, menu options, status codes)

üîÅ Looping Statements

Level 1: Beginner-Friendly Explanation

Loops are like doing homework problems:

  • for loop: "Solve problems 1 through 10" (you know exactly how many)
  • while loop: "Keep solving problems until you understand" (you don't know how many)
  • do-while loop: "Solve at least one problem, then keep going if needed" (guaranteed to do it once)

Loops save you from writing the same code over and over. Instead of writing "print 1, print 2, print 3..." 100 times, you write ONE loop!

Level 2: Real-World Analogy

Think of loops like climbing stairs:

  • for loop: "Climb 10 stairs" - You know the exact number
  • while loop: "Climb until you reach the top" - You check before each step
  • do-while loop: "Take one step, then check if you need more" - You act first, check later

Level 3: Technical Breakdown

1. for Loop

Best when you know EXACTLY how many times to repeat. Has three parts: initialization, condition, update. 

for (initialization; condition; update) {
    // Code to repeat
}

// Example: Print 1 to 5
for (int i = 1; i <= 5; i++) {
    System.out.println(i);
}

// Breakdown:
// 1. int i = 1     ‚Üí Start at 1
// 2. i <= 5        ‚Üí Continue while i is 5 or less
// 3. i++           ‚Üí Increase i by 1 after each iteration
üí° EXAMPLE - for Loop Execution

Iteration 1: i=1, check 1<=5 (true), print 1, i becomes 2
Iteration 2: i=2, check 2<=5 (true), print 2, i becomes 3
...
Iteration 5: i=5, check 5<=5 (true), print 5, i becomes 6
Iteration 6: i=6, check 6<=5 (false), STOP

2. while Loop

Best when you DON'T know how many times to repeat. Checks condition BEFORE each iteration. 

while (condition) {
    // Code to repeat
    // Don't forget to update the condition variable!
}

// Example: Print 1 to 5
int i = 1;
while (i <= 5) {
    System.out.println(i);
    i++;  // CRITICAL: Must update i, or infinite loop!
}
‚ö†Ô∏è WARNING - Infinite Loop

If you forget to update the loop variable, the condition will ALWAYS be true, creating an infinite loop!

while (i <= 5) { System.out.println(i); } ‚ùå Infinite loop! i never changes!

3. do-while Loop

Guarantees AT LEAST ONE execution. Checks condition AFTER each iteration. 

do {
    // Code to repeat (executes at least once)
} while (condition);

// Example: Menu system
int choice;
do {
    System.out.println("1. Play");
    System.out.println("2. Settings");
    System.out.println("3. Exit");
    choice = scanner.nextInt();
} while (choice != 3);  // Repeat until user chooses Exit
üìù NOTE - while vs do-while

while: Checks condition FIRST. May execute 0 times.
do-while: Checks condition LAST. Always executes at least 1 time.

4. Enhanced for Loop (for-each)

Simplified loop for iterating through arrays and collections. No index needed! 

for (dataType variable : array/collection) {
    // Use variable (each element)
}

// Example: Print all elements
int[] numbers = {10, 20, 30, 40, 50};
for (int num : numbers) {
    System.out.println(num);
}

// Equivalent to:
for (int i = 0; i < numbers.length; i++) {
    System.out.println(numbers[i]);
}
üí° TIP - When to Use Enhanced for Loop

Use enhanced for loop when:
‚úÖ You need to access every element
‚úÖ You don't need the index
‚úÖ You're not modifying the array

Use regular for loop when:
‚úÖ You need the index
‚úÖ You're modifying array elements
‚úÖ You need to iterate in reverse or skip elements

üöÄ Branching Statements

Level 1: Beginner-Friendly Explanation

Branching statements are like shortcuts or emergency exits:

  • break: "Stop everything and get out NOW!" (exit the loop)
  • continue: "Skip this one and move to the next" (skip current iteration)
  • return: "I'm done here, going back!" (exit the method)

Level 2: Real-World Analogy

Think of branching like watching TV episodes:

  • break: "This show is boring, I'm turning off the TV" (stop watching entirely)
  • continue: "I'll skip this episode and watch the next one" (skip one, keep watching)
  • return: "I found what I was looking for, no need to watch more" (exit with result)

Level 3: Technical Breakdown

1. break Statement

Immediately exits the current loop or switch statement. Execution continues after the loop. 

// Example: Find first even number
for (int i = 1; i <= 10; i++) {
    if (i % 2 == 0) {
        System.out.println("First even number: " + i);
        break;  // Exit loop immediately
    }
}
// Output: First even number: 2
// Loop stops, doesn't check 3, 4, 5, etc.

2. continue Statement

Skips the rest of the current iteration and moves to the next iteration. 

// Example: Print only odd numbers
for (int i = 1; i <= 5; i++) {
    if (i % 2 == 0) {
        continue;  // Skip even numbers
    }
    System.out.println(i);
}
// Output: 1, 3, 5
// When i=2 or i=4, continue skips the print statement

3. return Statement

Exits the entire method immediately. Can optionally return a value. 

// Example: Search for a value
public static int findIndex(int[] arr, int target) {
    for (int i = 0; i < arr.length; i++) {
        if (arr[i] == target) {
            return i;  // Found it! Exit method with index
        }
    }
    return -1;  // Not found
}

// Usage:
int[] numbers = {10, 20, 30, 40};
int index = findIndex(numbers, 30);
System.out.println("Index: " + index);  // Output: Index: 2
üí° EXAMPLE - break vs continue vs return

break: Exits the loop, continues with code after loop
continue: Skips to next iteration of the same loop
return: Exits the entire method immediately

‚ö†Ô∏è Common Mistakes with Control Flow

1. Using = Instead of == in Conditions

// ‚ùå WRONG - Assignment, not comparison
if (x = 5) {  // This assigns 5 to x, not comparing!
    System.out.println("x is 5");
}

// ‚úÖ CORRECT - Comparison
if (x == 5) {
    System.out.println("x is 5");
}

2. Infinite Loops (Forgetting to Update Loop Variable)

// ‚ùå WRONG - Infinite loop
int i = 1;
while (i <= 5) {
    System.out.println(i);
    // Forgot i++! i is always 1, condition always true!
}

// ‚úÖ CORRECT
int i = 1;
while (i <= 5) {
    System.out.println(i);
    i++;  // Update the variable!
}

3. Missing break in switch-case

// ‚ùå WRONG - Fall-through (usually unintended)
int day = 1;
switch (day) {
    case 1:
        System.out.println("Monday");
        // Missing break! Falls through to Tuesday!
    case 2:
        System.out.println("Tuesday");
        break;
}
// Output: Monday\nTuesday (both print!)

// ‚úÖ CORRECT
switch (day) {
    case 1:
        System.out.println("Monday");
        break;  // Exit switch
    case 2:
        System.out.println("Tuesday");
        break;
}

4. Off-by-One Errors

// ‚ùå WRONG - Prints 0 to 9 (10 numbers, but not 1-10)
for (int i = 0; i < 10; i++) {
    System.out.println(i);
}

// ‚úÖ CORRECT - Prints 1 to 10
for (int i = 1; i <= 10; i++) {
    System.out.println(i);
}

// Or if you want 0-9 for array indexing:
for (int i = 0; i < array.length; i++) {
    System.out.println(array[i]);
}

5. Semicolon After Loop Header

// ‚ùå WRONG - Semicolon creates empty loop body
for (int i = 0; i < 5; i++);  // ‚Üê Semicolon here!
{
    System.out.println(i);  // This is NOT part of the loop!
}

// ‚úÖ CORRECT
for (int i = 0; i < 5; i++) {
    System.out.println(i);
}

6. Logical Operator Confusion (&& vs ||)

// ‚ùå WRONG - Using OR when you mean AND
int age = 25;
if (age > 18 || age < 65) {  // This is ALWAYS true!
    System.out.println("Working age");
}
// age > 18 OR age < 65 means: if age is either above 18 OR below 65
// Every number satisfies at least one condition!

// ‚úÖ CORRECT - Using AND
if (age > 18 && age < 65) {
    System.out.println("Working age");
}

7. Modifying Collection While Iterating

// ‚ùå WRONG - ConcurrentModificationException
List numbers = new ArrayList<>(Arrays.asList(1, 2, 3, 4, 5));
for (int num : numbers) {
    if (num % 2 == 0) {
        numbers.remove(num);  // Modifying while iterating!
    }
}

// ‚úÖ CORRECT - Use Iterator
Iterator it = numbers.iterator();
while (it.hasNext()) {
    if (it.next() % 2 == 0) {
        it.remove();  // Safe removal
    }
}

‚ú® Best Practices for Control Flow

1. Always Use Braces (Even for Single Statements)

// ‚ùå Avoid - Easy to make mistakes
if (x > 0)
    System.out.println("Positive");
    System.out.println("This always prints!");  // Not part of if!

// ‚úÖ Recommended - Clear and safe
if (x > 0) {
    System.out.println("Positive");
}

2. Prefer switch-case for Multiple Constant Checks

// ‚ùå Less efficient and harder to read
if (status == 200) {
    handleSuccess();
} else if (status == 404) {
    handleNotFound();
} else if (status == 500) {
    handleServerError();
}

// ‚úÖ Better - More efficient and readable
switch (status) {
    case 200:
        handleSuccess();
        break;
    case 404:
        handleNotFound();
        break;
    case 500:
        handleServerError();
        break;
    default:
        handleUnknownError();
}

3. Avoid Deeply Nested Loops

// ‚ùå Hard to read and maintain
for (int i = 0; i < n; i++) {
    for (int j = 0; j < m; j++) {
        for (int k = 0; k < p; k++) {
            // Too deep!
        }
    }
}

// ‚úÖ Better - Extract to methods
for (int i = 0; i < n; i++) {
    processRow(i);
}

private void processRow(int row) {
    for (int j = 0; j < m; j++) {
        processCell(row, j);
    }
}

4. Use Enhanced for Loop When Possible

// ‚ùå Verbose
int[] numbers = {1, 2, 3, 4, 5};
for (int i = 0; i < numbers.length; i++) {
    System.out.println(numbers[i]);
}

// ‚úÖ Cleaner and less error-prone
for (int num : numbers) {
    System.out.println(num);
}

5. Initialize Loop Variables Close to Usage

// ‚ùå Variable declared far from usage
int i;
// ... 50 lines of code ...
for (i = 0; i < 10; i++) {
    // Use i
}

// ‚úÖ Declare where used
for (int i = 0; i < 10; i++) {
    // Use i
}

6. Use Meaningful Variable Names

// ‚ùå Unclear
for (int i = 0; i < students.length; i++) {
    for (int j = 0; j < subjects.length; j++) {
        // What are i and j?
    }
}

// ‚úÖ Clear and self-documenting
for (int studentIndex = 0; studentIndex < students.length; studentIndex++) {
    for (int subjectIndex = 0; subjectIndex < subjects.length; subjectIndex++) {
        // Clear what each index represents
    }
}

7. Validate Input in Loops

// ‚úÖ Good practice - Validate user input
Scanner scanner = new Scanner(System.in);
int age;
do {
    System.out.print("Enter age (1-120): ");
    while (!scanner.hasNextInt()) {
        System.out.print("Invalid input! Enter a number: ");
        scanner.next();
    }
    age = scanner.nextInt();
} while (age < 1 || age > 120);
üí° TIP - Performance Consideration

Cache array length: If using a regular for loop with arrays, cache the length to avoid repeated lookups:

for (int i = 0, len = array.length; i < len; i++)

However, modern JVMs optimize this automatically, so prioritize readability over micro-optimizations.

üìù NOTE - Industry Standards

Google Java Style Guide: Always use braces, even for single-line blocks
Effective Java: Prefer for-each loops over traditional for loops
Clean Code: Keep loops simple and extract complex logic to methods
